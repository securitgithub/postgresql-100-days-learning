# Day 072: JSONB 高级操作 — 路径查询、修改与聚合

## 学习目标

- 掌握 `jsonb_path_query` 系列函数进行 SQL/JSON Path 查询
- 学会使用 `@?` 和 `@@` 操作符进行路径匹配与过滤
- 熟练使用 `jsonb_set` 和 `jsonb_insert` 修改 JSONB 数据
- 掌握 `jsonb_agg` 和 `jsonb_object_agg` 进行 JSON 聚合
- 能够在实际场景中灵活运用 JSONB 高级功能

## 理论讲解

### 1. SQL/JSON Path 语言

PostgreSQL 12 引入了 SQL/JSON Path 语言，这是 SQL:2016 标准的一部分。它提供了一种声明式的方式来查询 JSON 数据，类似于 XML 的 XPath。

Path 表达式的基本语法：
- `$`：根元素
- `.key`：访问对象的键
- `[n]`：访问数组的第 n 个元素
- `[*]`：数组的所有元素
- `.*`：对象的所有值
- `?(@.key > 10)`：过滤表达式
- `.**`：递归访问所有层级

### 2. 路径查询函数

| 函数 | 返回类型 | 说明 |
|------|---------|------|
| `jsonb_path_query(target, path)` | setof jsonb | 返回所有匹配的元素（多行） |
| `jsonb_path_query_array(target, path)` | jsonb | 返回匹配元素的数组 |
| `jsonb_path_query_first(target, path)` | jsonb | 返回第一个匹配元素 |
| `jsonb_path_exists(target, path)` | boolean | 是否存在匹配 |
| `jsonb_path_match(target, path)` | boolean | 路径表达式是否为真 |

### 3. 路径查询操作符

- `@?`：等价于 `jsonb_path_exists`，用于 WHERE 子句过滤
- `@@`：等价于 `jsonb_path_match`，用于布尔谓词匹配

这两个操作符可以使用 GIN 索引加速查询。

### 4. JSONB 修改函数

PostgreSQL 的 JSONB 是不可变类型，"修改"实际上是创建新值：

- `jsonb_set(target, path, new_value, create_missing)`：设置指定路径的值
- `jsonb_insert(target, path, new_value, insert_after)`：在数组中插入元素
- `||`：合并两个 JSONB 值
- `-`：删除键或数组元素
- `#-`：删除指定路径的元素

### 5. JSONB 聚合函数

- `jsonb_agg(expression)`：将多行值聚合为 JSON 数组
- `jsonb_object_agg(key, value)`：将键值对聚合为 JSON 对象

这些聚合函数在构建 API 响应和数据转换中极为有用。

### 示例1：jsonb_path_query 基础用法

```sql
-- 准备测试数据
CREATE TABLE products (
    id SERIAL PRIMARY KEY,
    name TEXT,
    data JSONB
);

INSERT INTO products (name, data) VALUES

    "brand": "ThinkPad",
    "price": 6999,
    "specs": {"cpu": "i7-12700H", "ram": "16GB", "storage": "512GB SSD"},
    "tags": ["办公", "编程", "轻薄"],
    "reviews": [
        {"user": "张三", "score": 5, "comment": "非常好用"},
        {"user": "李四", "score": 4, "comment": "性价比高"},
        {"user": "王五", "score": 3, "comment": "一般般"}
    ]
}'),
('机械键盘', '{
    "brand": "HHKB",
    "price": 2199,
    "specs": {"type": "静电容", "layout": "60%", "weight": "530g"},
    "tags": ["编程", "静音", "便携"],
    "reviews": [
        {"user": "赵六", "score": 5, "comment": "手感极佳"},
        {"user": "钱七", "score": 5, "comment": "值得拥有"}
    ]
}'),
('显示器', '{
    "brand": "Dell",
    "price": 3299,
    "specs": {"size": "27寸", "resolution": "4K", "panel": "IPS"},
    "tags": ["设计", "编程", "护眼"],
    "reviews": [
        {"user": "孙八", "score": 4, "comment": "色彩准确"},
        {"user": "周九", "score": 5, "comment": "4K很清晰"}
    ]
}');

-- 查询所有产品的品牌
SELECT jsonb_path_query(data, '$.brand') AS brand
FROM products;

-- 查询所有评论的评分
SELECT name, jsonb_path_query(data, '$.reviews[*].score') AS score
FROM products;

-- 查询所有标签（展开为行）
SELECT name, jsonb_path_query(data, '$.tags[*]') AS tag
FROM products;

-- 以数组形式返回所有标签
SELECT name, jsonb_path_query_array(data, '$.tags[*]') AS all_tags
FROM products;

-- 只取第一条评论
SELECT name, jsonb_path_query_first(data, '$.reviews[0]') AS first_review
FROM products;
```

### 示例2：带过滤条件的路径查询

```sql
-- 查询评分大于等于 5 的评论
SELECT
    name,
    jsonb_path_query(data, '$.reviews[*] ? (@.score >= 5)') AS top_reviews
FROM products;

-- 查询价格超过 3000 的产品数据
SELECT name, data
FROM products
WHERE jsonb_path_exists(data, '$ ? (@.price > 3000)');

-- 查询包含"编程"标签的产品
SELECT name
FROM products
WHERE jsonb_path_exists(data, '$.tags[*] ? (@ == "编程")');

-- 使用变量传参
SELECT name, jsonb_path_query(
    data,
    '$.reviews[*] ? (@.score >= $min_score)',
    '{"min_score": 4}'  -- 变量参数
) AS good_reviews
FROM products;

-- 多条件过滤
SELECT name, jsonb_path_query(
    data,
    '$.reviews[*] ? (@.score >= 4 && @.comment like_regex "好")'
) AS filtered_reviews
FROM products;
```

### 示例3：@? 和 @@ 操作符

```sql
-- @? 操作符：检查路径是否存在匹配
-- 等价于 jsonb_path_exists
SELECT name FROM products
WHERE data @? '$.tags[*] ? (@ == "编程")';

-- @@ 操作符：路径表达式求值为布尔值
-- 等价于 jsonb_path_match
SELECT name FROM products
WHERE data @@ '$.price > 3000';

-- 创建 GIN 索引加速 @? 和 @@ 查询
CREATE INDEX idx_products_data ON products USING gin(data jsonb_path_ops);

-- 利用索引加速的查询
SELECT name FROM products
WHERE data @? '$.specs.cpu';

-- 组合使用
SELECT name, data->'price' AS price
FROM products
WHERE data @? '$.tags[*] ? (@ == "编程")'
  AND data @@ '$.price < 5000';
```

### 示例4：jsonb_set 和 jsonb_insert 修改数据

```sql
-- jsonb_set：修改已有字段
-- 修改价格
UPDATE products
SET data = jsonb_set(data, '{price}', '5999')
WHERE name = '笔记本电脑';

-- 修改嵌套字段
UPDATE products
SET data = jsonb_set(data, '{specs,ram}', '"32GB"')
WHERE name = '笔记本电脑';

-- 添加新字段（create_missing 默认为 true）
UPDATE products
SET data = jsonb_set(data, '{discount}', '0.85')
WHERE name = '笔记本电脑';

-- 不创建新字段（只修改已存在的）
UPDATE products
SET data = jsonb_set(data, '{warranty}', '"3年"', false)
WHERE name = '笔记本电脑';
-- warranty 不存在，不会被创建

-- jsonb_insert：在数组中插入元素
-- 在标签数组开头插入
UPDATE products
SET data = jsonb_insert(data, '{tags,0}', '"热卖"')
WHERE name = '机械键盘';

-- 在标签数组末尾插入（insert_after = true）
UPDATE products
SET data = jsonb_insert(data, '{tags,-1}', '"限量"', true)
WHERE name = '机械键盘';

-- 添加新评论到评论数组末尾
UPDATE products
SET data = jsonb_insert(
    data,
    '{reviews,-1}',
    '{"user": "新用户", "score": 5, "comment": "推荐购买"}',
    true
)
WHERE name = '显示器';

-- 使用 || 合并
UPDATE products
SET data = data || '{"on_sale": true, "sale_price": 1999}'::jsonb
WHERE name = '机械键盘';

-- 删除键
UPDATE products
SET data = data - 'on_sale'
WHERE name = '机械键盘';

-- 删除嵌套路径
UPDATE products
SET data = data #- '{specs,weight}'
WHERE name = '机械键盘';

-- 删除数组中的元素（按索引）
UPDATE products
SET data = data #- '{tags,0}'
WHERE name = '机械键盘';
```

### 示例5：jsonb_agg 聚合

```sql
-- 准备订单数据
CREATE TABLE orders (
    id SERIAL PRIMARY KEY,
    customer_name TEXT,
    product TEXT,
    amount NUMERIC,
    order_date DATE
);

INSERT INTO orders VALUES
(1, '张三', '笔记本', 6999, '2024-01-15'),
(2, '张三', '鼠标', 199, '2024-01-15'),
(3, '李四', '键盘', 599, '2024-01-16'),
(4, '李四', '显示器', 3299, '2024-01-16'),
(5, '李四', '耳机', 899, '2024-01-17'),
(6, '王五', '笔记本', 8999, '2024-01-17');

-- jsonb_agg：将每个客户的产品聚合为 JSON 数组
SELECT
    customer_name,
    jsonb_agg(product) AS products
FROM orders
GROUP BY customer_name;

-- 聚合为对象数组
SELECT
    customer_name,
    jsonb_agg(
        jsonb_build_object(
            'product', product,
            'amount', amount,
            'date', order_date
        )
    ) AS order_details
FROM orders
GROUP BY customer_name;

-- jsonb_agg 配合 ORDER BY
SELECT
    customer_name,
    jsonb_agg(product ORDER BY amount DESC) AS products_by_price
FROM orders
GROUP BY customer_name;

-- jsonb_agg 过滤 NULL
SELECT
    customer_name,
    jsonb_agg(product) FILTER (WHERE amount > 1000) AS expensive_products
FROM orders
GROUP BY customer_name;
```

### 示例6：jsonb_object_agg 聚合

```sql
-- 将产品和金额聚合为 JSON 对象
SELECT
    customer_name,
    jsonb_object_agg(product, amount) AS product_amounts
FROM orders
GROUP BY customer_name;

-- 构建配置风格的 JSON
CREATE TABLE settings (
    id SERIAL PRIMARY KEY,
    category TEXT,
    key TEXT,
    value TEXT
);

INSERT INTO settings (category, key, value) VALUES
('display', 'theme', 'dark'),
('display', 'language', 'zh-CN'),
('display', 'font_size', '14'),
('notification', 'email', 'true'),
('notification', 'sms', 'false'),
('notification', 'push', 'true');

-- 按类别聚合为嵌套 JSON
SELECT jsonb_object_agg(
    category,
    category_settings
) AS full_config
FROM (
    SELECT
        category,
        jsonb_object_agg(key, value) AS category_settings
    FROM settings
    GROUP BY category
) sub;

-- 结果：{"display": {"theme": "dark", "language": "zh-CN", ...}, "notification": {...}}
```

### 示例7：综合实战 — 构建 API 响应

```sql
-- 模拟构建一个 RESTful API 的 JSON 响应
WITH customer_orders AS (
    SELECT
        customer_name,
        count(*) AS order_count,
        sum(amount) AS total_amount,
        jsonb_agg(
            jsonb_build_object(
                'id', id,
                'product', product,
                'amount', amount,
                'date', order_date
            ) ORDER BY order_date
        ) AS orders
    FROM orders
    GROUP BY customer_name
)
SELECT jsonb_build_object(
    'status', 'success',
    'data', jsonb_agg(
        jsonb_build_object(
            'customer', customer_name,
            'summary', jsonb_build_object(
                'order_count', order_count,
                'total_amount', total_amount
            ),
            'orders', orders
        )
    ),
    'meta', jsonb_build_object(
        'total_customers', count(*),
        'generated_at', now()
    )
) AS api_response
FROM customer_orders;
```

## 常见问题

### Q1: jsonb_path_query 和传统的 -> / ->> 操作符相比有什么优势？

**A:** `jsonb_path_query` 的主要优势在于：
1. **声明式语法**：路径表达式更简洁，特别是处理嵌套数组和过滤时。
2. **内置过滤**：`?()` 过滤表达式可以在路径中直接过滤，而 `->` 需要额外的 WHERE 条件。
3. **变量支持**：可以通过第三个参数传入变量，便于参数化查询。
4. **递归查询**：`.**` 可以递归搜索所有层级。
5. **标准兼容**：遵循 SQL:2016 标准，可移植性更好。
对于简单的键值访问，`->` 和 `->>` 更直观；对于复杂查询，推荐使用 Path 语法。

### Q2: jsonb_set 多次嵌套修改时如何避免代码过于复杂？

**A:** 当需要同时修改多个字段时，有几种优雅的方式：
1. 使用 `||` 合并操作符批量更新顶层字段。
2. 使用嵌套的 `jsonb_set` 链式调用。
3. 在应用层构建完整的新 JSON，直接赋值。
4. 创建 PL/pgSQL 函数封装复杂的修改逻辑。
推荐根据修改的复杂程度选择策略：少量修改用 `jsonb_set`，大量修改考虑应用层构建后整体替换。

### Q3: jsonb_agg 结果中包含 NULL 怎么处理？

**A:** `jsonb_agg` 默认会将 NULL 值包含在结果数组中（作为 JSON null）。处理方式：
1. 使用 `FILTER (WHERE expr IS NOT NULL)` 子句过滤掉 NULL。
2. 使用 `jsonb_agg(expr) - 'null'` 不适用，因为 `-` 是删除键不是删除值。
3. 使用 `coalesce` 提供默认值。
推荐方式是 `FILTER` 子句：`jsonb_agg(col) FILTER (WHERE col IS NOT NULL)`。
