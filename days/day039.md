# Day 039: JSON 与 JSONB 基础

## 学习目标

- 理解 JSON 与 JSONB 的区别及各自适用场景
- 掌握 JSONB 数据的插入与查询方法
- 熟练使用 `->`、`->>`、`#>`、`#>>` 操作符
- 掌握常用的 JSONB 函数
- 能在实际项目中合理使用 JSONB 存储半结构化数据

## 理论讲解

### JSON vs JSONB

PostgreSQL 提供两种 JSON 数据类型：

| 特性 | JSON | JSONB |
|------|------|-------|
| 存储格式 | 原始文本 | 解析后的二进制格式 |
| 写入速度 | 快（直接存储） | 稍慢（需要解析） |
| 查询速度 | 慢（每次查询需解析） | 快（已预解析） |
| 保留格式 | 保留空格、键顺序、重复键 | 不保留空格、键按排序存储、去除重复键 |
| 索引支持 | 不支持 | 支持 GIN 索引 |
| 操作符 | 基本操作符 | 丰富的操作符和函数 |

**结论：绝大多数场景应选择 JSONB。** 仅在需要保留原始 JSON 格式时使用 JSON。

### JSONB 操作符

#### 取值操作符

| 操作符 | 说明 | 返回类型 | 示例 |
|--------|------|----------|------|
| `->` | 按键名取值 | JSONB | `data -> 'name'` |
| `->>` | 按键名取值 | TEXT | `data ->> 'name'` |
| `->` | 按索引取值（数组） | JSONB | `data -> 0` |
| `->>` | 按索引取值（数组） | TEXT | `data ->> 0` |
| `#>` | 按路径取值 | JSONB | `data #> '{addr,city}'` |
| `#>>` | 按路径取值 | TEXT | `data #>> '{addr,city}'` |

**关键区别：** `->` 返回 JSONB 类型（可以继续链式操作），`->>` 返回 TEXT 类型（用于最终取值、比较、WHERE 条件）。

#### 包含与存在操作符

| 操作符 | 说明 | 示例 |
|--------|------|------|
| `@>` | 左边包含右边 | `data @> '{"status":"active"}'` |
| `<@` | 左边被右边包含 | `'{"a":1}' <@ data` |
| `?` | 是否存在某个键 | `data ? 'email'` |
| `?\|` | 是否存在任意一个键 | `data ?\| array['email','phone']` |
| `?&` | 是否同时存在所有键 | `data ?& array['email','phone']` |

#### 修改操作符（JSONB 独有）

| 操作符 | 说明 | 示例 |
|--------|------|------|
| `\|\|` | 合并/追加 | `data \|\| '{"new_key":"value"}'` |
| `-` | 删除键 | `data - 'key_to_remove'` |
| `-` | 删除数组元素（按索引） | `data - 2` |
| `#-` | 按路径删除 | `data #- '{nested,key}'` |

### 常用 JSONB 函数

| 函数 | 说明 |
|------|------|
| `jsonb_build_object(k1,v1,k2,v2,...)` | 构建 JSONB 对象 |
| `jsonb_build_array(v1,v2,...)` | 构建 JSONB 数组 |
| `jsonb_object_keys(jsonb)` | 返回顶层所有键 |
| `jsonb_each(jsonb)` | 展开为键值对（值为 JSONB） |
| `jsonb_each_text(jsonb)` | 展开为键值对（值为 TEXT） |
| `jsonb_array_elements(jsonb)` | 展开 JSONB 数组为多行 |
| `jsonb_array_length(jsonb)` | 返回 JSONB 数组长度 |
| `jsonb_typeof(jsonb)` | 返回值的类型 |
| `jsonb_set(target, path, value)` | 设置指定路径的值 |
| `jsonb_insert(target, path, value)` | 在指定路径插入值 |
| `jsonb_strip_nulls(jsonb)` | 移除所有 NULL 值的键 |
| `jsonb_pretty(jsonb)` | 格式化输出 |
| `to_jsonb(record)` | 将行转换为 JSONB |
| `jsonb_agg(expr)` | 聚合为 JSONB 数组 |
| `jsonb_object_agg(key, value)` | 聚合为 JSONB 对象 |
| `jsonb_path_query(target, path)` | SQL/JSON 路径查询 |

## 代码示例

### 示例 1：JSONB 基本插入与查询

```sql
-- 创建包含 JSONB 列的表
CREATE TABLE products (
    id       SERIAL PRIMARY KEY,
    name     VARCHAR(200) NOT NULL,
    metadata JSONB NOT NULL DEFAULT '{}',
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 插入 JSONB 数据
INSERT INTO products (name, metadata) VALUES
```

### 示例 2：取值操作符详解

```sql
-- -> 返回 JSONB 类型
SELECT name,
       metadata -> 'brand' AS brand_jsonb      -- 返回 "Apple"（带引号，JSONB类型）
FROM products;

-- ->> 返回 TEXT 类型
SELECT name,
       metadata ->> 'brand' AS brand_text      -- 返回 Apple（无引号，TEXT类型）
FROM products;

-- -> 链式访问嵌套对象
SELECT name,
       metadata -> 'specs' -> 'cpu' AS cpu_jsonb,
       metadata -> 'specs' ->> 'cpu' AS cpu_text   -- 最后一步用 ->> 取文本
FROM products;

-- -> 访问数组元素（从0开始）
SELECT name,
       metadata -> 'colors' -> 0 AS first_color,
       metadata -> 'colors' ->> 1 AS second_color
FROM products;

-- #> 按路径取值（JSONB类型）
SELECT name,
       metadata #> '{specs,screen}' AS screen_jsonb
FROM products;

-- #>> 按路径取值（TEXT类型）
SELECT name,
       metadata #>> '{specs,screen}' AS screen_text,
       metadata #>> '{specs,storage,0}' AS base_storage
FROM products;

-- 在 WHERE 中使用（注意用 ->> 取文本值比较）
SELECT name, metadata ->> 'brand' AS brand
FROM products
WHERE metadata ->> 'brand' = 'Apple';

-- 数值比较需要类型转换
SELECT name, (metadata ->> 'price')::NUMERIC AS price
FROM products
WHERE (metadata ->> 'price')::NUMERIC < 6000;
```

### 示例 3：包含与存在操作符

```sql
-- @> 包含查询：查找品牌为 Apple 的产品
SELECT name FROM products
WHERE metadata @> '{"brand": "Apple"}';

-- @> 嵌套包含：查找屏幕为 6.2 英寸的产品
SELECT name FROM products
WHERE metadata @> '{"specs": {"screen": "6.2英寸"}}';

-- @> 数组包含：查找有黑色款的产品
SELECT name FROM products
WHERE metadata @> '{"colors": ["黑色"]}';

-- ? 键是否存在
SELECT name FROM products
WHERE metadata ? 'brand';

-- ?| 任意一个键存在
SELECT name FROM products
WHERE metadata ?| array['discount', 'brand'];

-- ?& 所有键都存在
SELECT name FROM products
WHERE metadata ?& array['brand', 'price', 'in_stock'];

-- NOT 结合使用
SELECT name FROM products
WHERE NOT (metadata @> '{"in_stock": true}');
```

### 示例 4：修改 JSONB 数据

```sql
-- || 合并（添加或更新键）
UPDATE products
SET metadata = metadata || '{"discount": 0.1, "warranty": "1年"}'
WHERE id = 1;

-- || 更新嵌套值（注意：只能替换顶层键）
UPDATE products
SET metadata = metadata || '{"price": 7499}'
WHERE id = 1;

-- jsonb_set 更新指定路径的值
UPDATE products
SET metadata = jsonb_set(metadata, '{specs,cpu}', '"A17 Pro"')
WHERE id = 1;

-- jsonb_set 更新嵌套数组中的元素
UPDATE products
SET metadata = jsonb_set(metadata, '{colors,0}', '"深空黑"')
WHERE id = 1;

-- jsonb_set 添加新的嵌套键
UPDATE products
SET metadata = jsonb_set(metadata, '{specs,weight}', '"187g"', true)
WHERE id = 1;
-- 第四个参数 true 表示如果路径不存在则创建

-- - 删除键
UPDATE products
SET metadata = metadata - 'discount'
WHERE id = 1;

-- #- 删除嵌套路径
UPDATE products
SET metadata = metadata #- '{specs,weight}'
WHERE id = 1;

-- 向 JSONB 数组追加元素
UPDATE products
SET metadata = jsonb_set(
    metadata,
    '{colors}',
    (metadata -> 'colors') || '"粉色"'
)
WHERE id = 1;

-- 查看修改结果（格式化输出）
SELECT name, jsonb_pretty(metadata)
FROM products WHERE id = 1;
```

### 示例 5：JSONB 函数实战

```sql
-- jsonb_object_keys：获取所有顶层键
SELECT jsonb_object_keys(metadata) AS key
FROM products WHERE id = 1;

-- jsonb_each_text：展开为键值对
SELECT key, value
FROM products, jsonb_each_text(metadata)
WHERE id = 1;

-- jsonb_array_elements：展开 JSONB 数组
SELECT name, color.value AS color
FROM products,
     jsonb_array_elements_text(metadata -> 'colors') AS color
WHERE id = 1;

-- jsonb_typeof：检查值类型
SELECT
    jsonb_typeof(metadata -> 'brand') AS brand_type,     -- string
    jsonb_typeof(metadata -> 'price') AS price_type,     -- number
    jsonb_typeof(metadata -> 'colors') AS colors_type,   -- array
    jsonb_typeof(metadata -> 'specs') AS specs_type,     -- object
    jsonb_typeof(metadata -> 'in_stock') AS stock_type   -- boolean
FROM products WHERE id = 1;

-- jsonb_build_object：动态构建 JSONB
SELECT jsonb_build_object(
    'product', name,
    'brand', metadata ->> 'brand',
    'price', (metadata ->> 'price')::NUMERIC,
    'available', (metadata ->> 'in_stock')::BOOLEAN
) AS summary
FROM products;

-- jsonb_agg：聚合为 JSONB 数组
SELECT jsonb_agg(
    jsonb_build_object('name', name, 'brand', metadata ->> 'brand')
) AS all_products
FROM products;

-- jsonb_object_agg：聚合为 JSONB 对象（以品牌为键）
SELECT jsonb_object_agg(
    metadata ->> 'brand',
    jsonb_build_object('name', name, 'price', metadata -> 'price')
) AS brand_map
FROM products;

-- jsonb_strip_nulls：移除 NULL 值
SELECT jsonb_strip_nulls('{"a": 1, "b": null, "c": {"d": null, "e": 2}}'::JSONB);
-- 结果：{"a": 1, "c": {"e": 2}}
```

### 示例 6：JSONB 索引

```sql
-- 方式1：GIN 索引（支持 @>, ?, ?|, ?& 操作符）
CREATE INDEX idx_products_metadata ON products USING GIN(metadata);

-- 以下查询会使用 GIN 索引
SELECT * FROM products WHERE metadata @> '{"brand": "Apple"}';
SELECT * FROM products WHERE metadata ? 'discount';

-- 方式2：GIN 索引（jsonb_path_ops，更小更快，仅支持 @>）
CREATE INDEX idx_products_metadata_path
    ON products USING GIN(metadata jsonb_path_ops);

-- 方式3：B-tree 索引（针对特定键的精确查询）
CREATE INDEX idx_products_brand ON products((metadata ->> 'brand'));
CREATE INDEX idx_products_price ON products(((metadata ->> 'price')::NUMERIC));

-- 以下查询使用 B-tree 索引
SELECT * FROM products WHERE metadata ->> 'brand' = 'Apple';
SELECT * FROM products WHERE (metadata ->> 'price')::NUMERIC < 6000;

-- 验证索引使用
EXPLAIN ANALYZE
SELECT * FROM products WHERE metadata @> '{"brand": "Apple"}';
```

### 示例 7：SQL/JSON 路径查询（PostgreSQL 12+）

```sql
-- jsonb_path_query：使用 SQL/JSON 路径表达式
-- 查找所有品牌名
SELECT jsonb_path_query(metadata, '$.brand') AS brand
FROM products;

-- 查找所有颜色
SELECT jsonb_path_query(metadata, '$.colors[*]') AS color
FROM products;

-- 带过滤条件的路径查询
SELECT name,
       jsonb_path_query(metadata, '$.specs.storage[*]') AS storage
FROM products
WHERE jsonb_path_exists(metadata, '$.price ? (@ < 6000)');

-- jsonb_path_query_first：只返回第一个匹配
SELECT name,
       jsonb_path_query_first(metadata, '$.colors[0]') AS primary_color
FROM products;

-- jsonb_path_exists：检查路径是否存在匹配
SELECT name
FROM products
WHERE jsonb_path_exists(metadata, '$.specs.storage[*] ? (@ == "512GB")');
```

## 常见问题

### Q1：JSONB 列中的数值比较要注意什么？

`->>` 返回的是 TEXT 类型，直接比较会按字符串排序（'9' > '80'）。必须显式转换类型：

```sql
-- 错误：字符串比较
SELECT * FROM products
WHERE metadata ->> 'price' > '5000';  -- '7999' > '5000' 但 '999' < '5000'

-- 正确：转换为数值
SELECT * FROM products
WHERE (metadata ->> 'price')::NUMERIC > 5000;

-- 或者使用 @> 进行精确匹配
SELECT * FROM products
WHERE metadata @> '{"price": 7999}';
```

### Q2：JSONB 适合替代所有关系表设计吗？

不适合。JSONB 适用于：
- **半结构化数据**：不同记录的属性可能不同
- **扩展属性**：核心字段用列存储，灵活属性用 JSONB
- **配置信息**：用户设置、系统配置等
- **第三方数据**：API 返回的不确定结构

不适合用 JSONB 的场景：
- 需要外键约束的关联关系
- 需要频繁更新 JSONB 中的单个字段（每次更新是整个文档重写）
- 所有记录结构完全一致的数据

### Q3：如何高效地更新 JSONB 中深层嵌套的值？

使用 `jsonb_set` 可以更新指定路径的值，但对于复杂的多字段更新，可能需要多次调用：

```sql
-- 更新多个嵌套字段
UPDATE products
SET metadata = jsonb_set(
    jsonb_set(metadata, '{specs,cpu}', '"A17 Pro"'),
    '{specs,screen}', '"6.3英寸"'
)
WHERE id = 1;

-- 或者使用 || 合并整个嵌套对象
UPDATE products
SET metadata = metadata || '{"specs": {"cpu": "A17 Pro", "screen": "6.3英寸"}}'::jsonb
WHERE id = 1;
-- 注意：这会替换整个 specs 对象，而非合并！

-- 深层合并可以使用 jsonb_set 逐个字段操作
-- 或者编写自定义的深度合并函数
```
