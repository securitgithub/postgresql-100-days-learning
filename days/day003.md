## 理论讲解

### 1. psql 连接方式

psql 支持多种连接方式，核心连接参数包括：

| 参数 | 短选项 | 环境变量 | 说明 |
|------|--------|---------|------|
| --host | -h | PGHOST | 主机地址 |
| --port | -p | PGPORT | 端口号 |
| --username | -U | PGUSER | 用户名 |
| --dbname | -d | PGDATABASE | 数据库名 |
| --password | -W | PGPASSWORD | 强制提示输入密码 |
| --no-password | -w | - | 不提示密码 |

连接字符串也可以使用 URI 格式：
```
postgresql://用户名:密码@主机:端口/数据库名
```

### 2. 元命令（Meta-Commands）分类

psql 的元命令以反斜杠 `\` 开头，不需要分号结尾。可以分为以下几类：

- **信息查看类**：查看数据库对象信息
- **格式控制类**：控制输出格式
- **输入输出类**：导入导出数据
- **连接管理类**：切换数据库、用户
- **执行控制类**：定时、执行文件

### 3. pgAdmin 简介

pgAdmin 是 PostgreSQL 最流行的图形化管理工具：
- 基于 Web 的界面，支持浏览器访问
- 提供可视化的数据库管理和查询功能
- 支持仪表盘监控、备份恢复、ERD 图等功能
- 随 PostgreSQL Windows 安装程序一起提供
- 也可以独立安装：https://www.pgadmin.org/

## 代码示例

### 连接数据库的多种方式

```bash
# 方式1：默认连接（本地 postgres 用户）
sudo -u postgres psql

# 方式2：指定主机、端口、用户、数据库
psql -h localhost -p 5432 -U myuser -d mydb

# 方式3：使用 URI 连接字符串
psql "postgresql://myuser:mypassword@localhost:5432/mydb"

# 方式4：使用环境变量
export PGHOST=localhost
export PGPORT=5432
export PGUSER=myuser
export PGDATABASE=mydb
psql

# 方式5：使用 .pgpass 文件免密码连接
# 在 ~/.pgpass 中添加：hostname:port:database:username:password
# 设置权限：chmod 600 ~/.pgpass

# 方式6：使用 service 文件
# 在 ~/.pg_service.conf 中配置
# [myservice]
# host=localhost
# port=5432
# dbname=mydb
# user=myuser
psql "service=myservice"
```

### 常用信息查看元命令

```sql
-- ========== 数据库级别 ==========

-- 列出所有数据库
\l
\l+    -- 显示更多细节（大小、表空间、描述）

-- 切换数据库
\c mydb
\connect mydb

-- 显示当前连接信息
\conninfo

-- ========== 表与视图 ==========

-- 列出当前数据库中的所有表
\dt
\dt+         -- 包含大小和描述
\dt public.* -- 指定 schema
\dt *.my*    -- 模式匹配

-- 列出所有视图
\dv

-- 列出所有物化视图
\dm

-- ========== 表结构 ==========

-- 查看表的列定义
\d tablename
\d+ tablename  -- 包含存储类型、描述等额外信息

-- ========== 索引 ==========

-- 列出所有索引
\di
\di+           -- 包含大小信息
\di tablename  -- 特定表的索引

-- ========== 序列 ==========
\ds            -- 列出所有序列

-- ========== 函数 ==========
\df            -- 列出所有函数
\df+           -- 包含函数源码
\df *count*    -- 搜索函数名

-- ========== Schema ==========
\dn            -- 列出所有 schema
\dn+           -- 包含权限信息

-- ========== 用户与角色 ==========
\du            -- 列出所有用户/角色
\du+           -- 包含额外信息

-- ========== 表空间 ==========
\db            -- 列出所有表空间

-- ========== 类型 ==========
\dT            -- 列出所有数据类型
\dT+           -- 包含详细信息

-- ========== 权限 ==========
\dp tablename  -- 查看表的访问权限
\z tablename   -- 同 \dp
```

### 格式与输出控制

```sql
-- 切换扩展显示模式（竖排显示，适合宽表）
\x
\x auto  -- 自动判断是否使用扩展模式

-- 示例：普通模式 vs 扩展模式
SELECT * FROM pg_database LIMIT 2;
\x
SELECT * FROM pg_database LIMIT 2;
\x  -- 切回普通模式

-- 设置输出格式
\pset format aligned    -- 对齐格式（默认）
\pset format unaligned  -- 无对齐
\pset format csv        -- CSV 格式
\pset format html       -- HTML 表格格式
\pset format latex      -- LaTeX 格式

-- 设置边框
\pset border 0  -- 无边框
\pset border 1  -- 内边框（默认）
\pset border 2  -- 完整边框

-- 设置 NULL 值的显示
\pset null '(NULL)'

-- 显示查询执行时间
\timing on
SELECT count(*) FROM pg_class;
\timing off

-- 开启/关闭查询结果输出
\pset tuples_only on  -- 只显示数据行
\pset tuples_only off
```

### 输入输出重定向

```sql
-- ========== 执行 SQL 文件 ==========

-- 方式1：在 psql 中执行文件
\i /path/to/script.sql
\include /path/to/script.sql

-- 方式2：从命令行执行文件
-- psql -U myuser -d mydb -f /path/to/script.sql

-- ========== 输出到文件 ==========

-- 将后续所有输出重定向到文件
\o /tmp/output.txt
SELECT * FROM pg_database;
SELECT * FROM pg_tables LIMIT 10;
\o  -- 取消重定向，恢复屏幕输出

-- ========== 使用 COPY 导入导出 ==========

-- 将查询结果导出为 CSV
\copy (SELECT * FROM pg_database) TO '/tmp/databases.csv' WITH CSV HEADER

-- 从 CSV 文件导入数据
-- \copy mytable FROM '/tmp/data.csv' WITH CSV HEADER

-- ========== 编辑与执行 ==========

-- 用外部编辑器编辑查询
\e                    -- 打开编辑器编写新查询
\e /tmp/myquery.sql   -- 编辑指定文件

-- 编辑上一条查询
\e

-- 查看并编辑函数定义
\ef function_name

-- 查看并编辑视图定义
\ev view_name
```

### psql 变量与条件执行

```sql
-- 设置 psql 变量
\set myvar 'Hello PostgreSQL'
\echo :myvar

-- 使用变量进行查询
\set table_name 'pg_database'
SELECT * FROM :table_name LIMIT 3;

-- 设置常用提示符
\set PROMPT1 '%n@%/%R%# '
-- %n = 用户名, %/ = 数据库名, %R = 连接状态, %# = 超级用户标识

-- 查看所有已设置变量
\set

-- 取消设置变量
\unset myvar

-- 查看命令历史
\s
\s /tmp/history.txt  -- 保存历史到文件

-- 执行 shell 命令
\! ls -la /tmp
\! date

-- 显示帮助
\?           -- psql 元命令帮助
\h           -- SQL 命令帮助
\h SELECT    -- 特定 SQL 命令的帮助
\h CREATE TABLE
```

### .psqlrc 配置文件

```sql
-- 在 ~/.psqlrc 中添加以下内容自定义 psql 行为：

-- 设置友好的提示符
\set PROMPT1 '%n@%/%R%#%x '
\set PROMPT2 '%R%# '

-- NULL 值显示为可见文本
\pset null '(NULL)'

-- 显示查询执行时间
\timing on

-- 自动使用扩展显示
\x auto

-- 历史文件大小
\set HISTSIZE 5000

-- 忽略重复的历史命令
\set HISTCONTROL ignoredups

-- 自动补全使用大写关键字
\set COMP_KEYWORD_CASE upper

-- 输出提示信息
\echo '欢迎使用 PostgreSQL！当前时间：' `date`
\echo ''
```

## 常见问题

### Q1: psql 连接时提示 "connection refused" 怎么办？

**A:** 可能的原因：
1. PostgreSQL 服务未启动 —— 执行 `sudo systemctl status postgresql` 检查
2. 端口不正确 —— 确认 `postgresql.conf` 中的 `port` 设置
3. 连接地址不在 `listen_addresses` 范围 —— 默认只监听 localhost
4. 防火墙阻止 —— 检查 iptables/firewalld 规则

### Q2: 如何在 psql 中执行多行 SQL 语句？

**A:** psql 会自动识别多行输入，直到遇到分号 `;` 才执行。你可以直接换行输入：
```sql
SELECT
    datname,
    pg_size_pretty(pg_database_size(datname))
FROM
    pg_database
ORDER BY
    pg_database_size(datname) DESC;
```
提示符会从 `=#` 变为 `-#` 表示正在等待输入。如果想取消当前输入，按 `Ctrl+C`。

### Q3: psql 如何实现自动补全？

**A:** psql 内置了 Tab 键自动补全功能：
- 输入 `SEL` 按 Tab → 自动补全为 `SELECT`
- 输入 `\d` 按 Tab → 显示所有以 `\d` 开头的元命令
- 输入表名前几个字母按 Tab → 自动补全表名
- 在 `.psqlrc` 中设置 `\set COMP_KEYWORD_CASE upper` 可让关键字自动大写
