## 理论讲解

### 1. CREATE TABLE 语法结构

```sql
CREATE [ TEMPORARY | UNLOGGED ] TABLE [ IF NOT EXISTS ] table_name (
    column_name data_type [ column_constraint [...] ],
    column_name data_type [ column_constraint [...] ],
    ...
    [ table_constraint, ... ]
)
[ TABLESPACE tablespace_name ];
```

### 2. 列约束（Column Constraints）

| 约束 | 说明 | 级别 |
|------|------|------|
| NOT NULL | 不允许 NULL 值 | 列级 |
| NULL | 允许 NULL（默认行为） | 列级 |
| DEFAULT expr | 默认值表达式 | 列级 |
| CHECK (expr) | 检查约束（布尔表达式） | 列级或表级 |
| UNIQUE | 唯一约束 | 列级或表级 |
| PRIMARY KEY | 主键（= UNIQUE + NOT NULL） | 列级或表级 |
| REFERENCES | 外键引用 | 列级或表级 |

### 3. 表约束 vs 列约束

- **列约束**：定义在列声明之后，只涉及单列
- **表约束**：定义在所有列声明之后，可以涉及多列

实际上，除了 NOT NULL 之外的所有列约束都可以改写为表约束。多列 UNIQUE 或 CHECK 必须用表约束。

### 4. ALTER TABLE 操作分类

ALTER TABLE 是最常用的 DDL 命令之一，可以：
- 增加/删除/修改列
- 增加/删除约束
- 重命名表或列
- 修改列的数据类型
- 设置/删除默认值
- 修改表的所有者、Schema

### 5. DROP vs TRUNCATE

| 操作 | 效果 | 速度 | 可回滚 | 触发器 |
|------|------|------|--------|--------|
| DROP TABLE | 删除表及全部数据和结构 | 快 | 可以(在事务中) | 不触发行级触发器 |
| TRUNCATE TABLE | 删除全部数据，保留结构 | 极快 | 可以(在事务中) | 不触发行级触发器 |
| DELETE FROM | 逐行删除数据 | 慢（大量数据时） | 可以 | 触发行级触发器 |

## 代码示例

### 基础建表

```sql
-- 最简单的建表
CREATE TABLE students (
    id integer,
    name text,
    age integer
);

-- 使用 IF NOT EXISTS 避免重复创建错误
CREATE TABLE IF NOT EXISTS students (
    id integer,
    name text,
    age integer
);

-- 带完整约束的建表
CREATE TABLE employees (
    id integer GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    employee_no varchar(10) NOT NULL UNIQUE,
    first_name varchar(50) NOT NULL,
    last_name varchar(50) NOT NULL,
    email varchar(100) UNIQUE,
    hire_date date NOT NULL DEFAULT current_date,
    salary numeric(10,2) CHECK (salary > 0),
    department varchar(50) DEFAULT '未分配',
    is_active boolean NOT NULL DEFAULT true,
    created_at timestamptz NOT NULL DEFAULT now(),
    updated_at timestamptz NOT NULL DEFAULT now()
);

-- 查看表结构
\d employees
\d+ employees  -- 包含存储信息
```

### NOT NULL 约束

```sql
CREATE TABLE orders (
    id integer GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    customer_name text NOT NULL,          -- 客户名必填
    order_date date NOT NULL DEFAULT current_date,  -- 必填但有默认值
    notes text                            -- 备注可以为 NULL
);

-- 成功插入
INSERT INTO orders (customer_name) VALUES ('张三');

-- 失败：违反 NOT NULL
INSERT INTO orders (customer_name) VALUES (NULL);
-- ERROR: null value in column "customer_name" violates not-null constraint

-- 注意：空字符串 '' 不是 NULL
INSERT INTO orders (customer_name) VALUES ('');  -- 这可以成功
```

### DEFAULT 默认值

```sql
CREATE TABLE articles (
    id integer GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    title text NOT NULL,
    content text NOT NULL DEFAULT '',
    status varchar(20) NOT NULL DEFAULT 'draft',
    view_count integer NOT NULL DEFAULT 0,
    created_at timestamptz NOT NULL DEFAULT now(),
    tags text[] DEFAULT '{}'::text[],       -- 数组类型的默认值
    metadata jsonb DEFAULT '{}'::jsonb      -- JSONB 的默认值
);

-- 只提供必填字段，其余使用默认值
INSERT INTO articles (title) VALUES ('我的第一篇文章');
SELECT * FROM articles;

-- DEFAULT 还可以使用函数表达式
CREATE TABLE audit_log (
    id integer GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    action text NOT NULL,
    performed_by text NOT NULL DEFAULT current_user,  -- 当前用户
    performed_at timestamptz NOT NULL DEFAULT now(),   -- 当前时间
    ip_address inet DEFAULT inet_client_addr()         -- 客户端IP
);
```

### CHECK 约束

```sql
-- 列级 CHECK
CREATE TABLE products (
    id integer GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    name text NOT NULL,
    price numeric(10,2) NOT NULL CHECK (price >= 0),
    discount numeric(3,2) CHECK (discount >= 0 AND discount <= 1),
    stock integer NOT NULL DEFAULT 0 CHECK (stock >= 0),
    weight numeric(8,3) CHECK (weight > 0),
    sku varchar(20) NOT NULL CHECK (length(sku) >= 3)
);

-- 表级 CHECK（可以涉及多列）
CREATE TABLE events (
    id integer GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    name text NOT NULL,
    start_date date NOT NULL,
    end_date date NOT NULL,
    min_attendees integer DEFAULT 1,
    max_attendees integer,
    -- 表级约束：结束日期必须晚于开始日期
    CONSTRAINT valid_dates CHECK (end_date > start_date),
    -- 表级约束：最大人数必须大于最小人数
    CONSTRAINT valid_attendees CHECK (max_attendees >= min_attendees)
);

-- 测试约束
INSERT INTO events (name, start_date, end_date, min_attendees, max_attendees)
VALUES ('PG大会', '2025-08-01', '2025-07-31', 10, 100);
-- ERROR: new row violates check constraint "valid_dates"

-- 命名约束的好处：错误信息更清晰
INSERT INTO products (name, price, stock, sku) VALUES ('测试', -10, 0, 'ABC');
-- ERROR: new row violates check constraint "products_price_check"
```

### UNIQUE 约束

```sql
-- 单列唯一
CREATE TABLE users (
    id integer GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    username varchar(30) NOT NULL UNIQUE,
    email varchar(100) UNIQUE,
    phone varchar(15) UNIQUE
);

-- 多列组合唯一（表级约束）
CREATE TABLE course_enrollment (
    id integer GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    student_id integer NOT NULL,
    course_id integer NOT NULL,
    enrolled_at date NOT NULL DEFAULT current_date,
    -- 同一学生不能重复选同一门课
    CONSTRAINT unique_enrollment UNIQUE (student_id, course_id)
);

INSERT INTO course_enrollment (student_id, course_id) VALUES (1, 101);
INSERT INTO course_enrollment (student_id, course_id) VALUES (1, 102);  -- OK，不同课程
INSERT INTO course_enrollment (student_id, course_id) VALUES (2, 101);  -- OK，不同学生
INSERT INTO course_enrollment (student_id, course_id) VALUES (1, 101);  -- ERROR！重复
-- ERROR: duplicate key value violates unique constraint "unique_enrollment"

-- UNIQUE 允许多个 NULL（NULL != NULL）
INSERT INTO users (username) VALUES ('user1');  -- email 和 phone 为 NULL
INSERT INTO users (username) VALUES ('user2');  -- 同样 NULL，不冲突
```

### ALTER TABLE 操作

```sql
-- ========== 列操作 ==========

-- 添加列
ALTER TABLE employees ADD COLUMN middle_name varchar(50);
ALTER TABLE employees ADD COLUMN birth_date date;

-- 添加列并设置默认值（大表也很快，不会重写表）
ALTER TABLE employees ADD COLUMN status varchar(20) NOT NULL DEFAULT 'active';

-- 删除列
ALTER TABLE employees DROP COLUMN middle_name;
ALTER TABLE employees DROP COLUMN IF EXISTS middle_name;  -- 不存在也不报错

-- 修改列名
ALTER TABLE employees RENAME COLUMN first_name TO given_name;
ALTER TABLE employees RENAME COLUMN given_name TO first_name;  -- 改回来

-- 修改数据类型
ALTER TABLE employees ALTER COLUMN department TYPE text;

-- 修改类型并转换数据（使用 USING）
ALTER TABLE employees ALTER COLUMN salary TYPE integer USING salary::integer;
ALTER TABLE employees ALTER COLUMN salary TYPE numeric(10,2);  -- 改回来

-- 设置/删除默认值
ALTER TABLE employees ALTER COLUMN department SET DEFAULT '技术部';
ALTER TABLE employees ALTER COLUMN department DROP DEFAULT;

-- 设置/删除 NOT NULL
ALTER TABLE employees ALTER COLUMN email SET NOT NULL;
ALTER TABLE employees ALTER COLUMN email DROP NOT NULL;

-- ========== 约束操作 ==========

-- 添加 CHECK 约束
ALTER TABLE employees ADD CONSTRAINT salary_positive CHECK (salary > 0);

-- 添加 UNIQUE 约束
ALTER TABLE employees ADD CONSTRAINT unique_email UNIQUE (email);

-- 删除约束
ALTER TABLE employees DROP CONSTRAINT salary_positive;
ALTER TABLE employees DROP CONSTRAINT IF EXISTS salary_positive;

-- ========== 表操作 ==========

-- 重命名表
ALTER TABLE employees RENAME TO staff;
ALTER TABLE staff RENAME TO employees;

-- 更改所有者
ALTER TABLE employees OWNER TO postgres;

-- 移动到另一个 Schema
ALTER TABLE employees SET SCHEMA hr;
ALTER TABLE hr.employees SET SCHEMA public;
```

### DROP 与 TRUNCATE

```sql
-- DROP TABLE：删除表的全部（结构 + 数据 + 索引 + 约束）
DROP TABLE students;
DROP TABLE IF EXISTS students;           -- 不存在也不报错
DROP TABLE orders CASCADE;                -- 级联删除依赖此表的对象（如外键引用）
DROP TABLE events, products;              -- 一次删除多张表

-- TRUNCATE：清空数据，保留表结构
TRUNCATE TABLE articles;
TRUNCATE TABLE articles RESTART IDENTITY;  -- 同时重置自增序列
TRUNCATE TABLE articles CASCADE;           -- 级联清空有外键引用的相关表

-- 在事务中使用 TRUNCATE（可回滚）
BEGIN;
    TRUNCATE TABLE articles RESTART IDENTITY;
    -- 如果发现错误，可以回滚
ROLLBACK;  -- 数据恢复

-- TRUNCATE vs DELETE 性能对比（大表）
-- DELETE: 逐行删除，生成大量 WAL，触发触发器
-- TRUNCATE: 直接截断文件，极快，不触发行级触发器
```

### 临时表与 UNLOGGED 表

```sql
-- 临时表：仅在当前会话中存在，会话结束自动删除
CREATE TEMPORARY TABLE temp_results (
    id integer,
    score numeric
);

-- 也可以指定事务结束时的行为
CREATE TEMP TABLE temp_data (
    val text
) ON COMMIT DELETE ROWS;  -- 每次事务提交后清空数据

CREATE TEMP TABLE temp_data2 (
    val text
) ON COMMIT DROP;  -- 事务提交后删除整个表

-- UNLOGGED 表：不写 WAL 日志，崩溃后数据丢失，但写入速度快
CREATE UNLOGGED TABLE cache_data (
    key text PRIMARY KEY,
    value jsonb,
    expires_at timestamptz
);
-- 适用场景：缓存数据、临时ETL数据、可重新生成的数据

-- 从查询结果创建表 (CREATE TABLE AS)
CREATE TABLE top_earners AS
    SELECT * FROM employees WHERE salary > 50000;

-- 创建与已有表结构相同的空表
CREATE TABLE employees_backup (LIKE employees INCLUDING ALL);
-- INCLUDING ALL 包括：约束、默认值、索引、注释等
```

### 为表和列添加注释

```sql
-- 为表添加注释
COMMENT ON TABLE employees IS '员工信息主表，存储公司全部在职和离职员工';

-- 为列添加注释
COMMENT ON COLUMN employees.salary IS '月薪（税前），单位：元';
COMMENT ON COLUMN employees.is_active IS '是否在职：true=在职, false=离职';

-- 查看注释
\d+ employees

-- 使用 SQL 查看注释
SELECT
    c.column_name,
    c.data_type,
    pgd.description AS comment
FROM information_schema.columns c
LEFT JOIN pg_catalog.pg_statio_all_tables st
    ON c.table_schema = st.schemaname AND c.table_name = st.relname
LEFT JOIN pg_catalog.pg_description pgd
    ON pgd.objoid = st.relid AND pgd.objsubid = c.ordinal_position
WHERE c.table_name = 'employees';
```

## 常见问题

### Q1: ALTER TABLE 修改列类型会锁表吗？

**A:** 大多数情况下会。具体取决于修改的类型：
- **不需要重写表的操作**（几乎瞬间完成）：添加有默认值的列（PG 11+）、删除列
- **需要重写表的操作**（可能耗时很长）：修改列的数据类型、添加 NOT NULL 到已有列
- 在生产环境对大表操作时，建议在低峰期进行，或使用 `pg_repack` 等工具避免长时间锁表

### Q2: TRUNCATE 和 DELETE FROM 哪个更快？

**A:** TRUNCATE 几乎总是更快，因为：
- TRUNCATE 直接释放表的存储页面，不逐行删除
- TRUNCATE 生成很少的 WAL 日志
- TRUNCATE 可以同时重置 IDENTITY/SERIAL 序列
- 但 TRUNCATE 不触发行级触发器，如果依赖触发器则需用 DELETE

### Q3: 一张表最多可以有多少列？

**A:** PostgreSQL 的硬限制是每张表最多 1600 列（受限于行的最大大小 8KB，对于 TOAST 可变长度列稍有不同）。但在实践中，如果一张表需要这么多列，通常意味着需要重新设计——考虑垂直拆分或使用 JSONB 存储动态属性。
