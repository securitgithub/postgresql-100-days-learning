## 理论讲解

### 常用数学函数

| 函数 | 说明 | 示例 |
|------|------|------|
| `abs(x)` | 绝对值 | `abs(-5) → 5` |
| `ceil(x)` / `ceiling(x)` | 向上取整 | `ceil(4.2) → 5` |
| `floor(x)` | 向下取整 | `floor(4.8) → 4` |
| `round(x)` | 四舍五入到整数 | `round(4.5) → 5` |
| `round(x, n)` | 四舍五入到 n 位小数 | `round(3.1415, 2) → 3.14` |
| `trunc(x, n)` | 截断到 n 位小数（不四舍五入） | `trunc(3.1499, 2) → 3.14` |
| `mod(x, y)` | 取模（余数） | `mod(10, 3) → 1` |
| `power(x, y)` | x 的 y 次方 | `power(2, 10) → 1024` |
| `sqrt(x)` | 平方根 | `sqrt(144) → 12` |
| `sign(x)` | 符号（-1/0/1） | `sign(-5) → -1` |
| `div(x, y)` | 整数除法 | `div(7, 2) → 3` |
| `log(x)` | 以 10 为底的对数 | `log(100) → 2` |
| `ln(x)` | 自然对数 | `ln(2.718) → ~1` |
| `pi()` | 圆周率 | `pi() → 3.14159...` |

### random() 函数

`random()` 返回 0 到 1 之间（含 0 不含 1）的随机浮点数。

```sql
-- 生成 0-1 随机数
SELECT random();

-- 生成 1-100 的随机整数
SELECT floor(random() * 100 + 1)::INT;

-- 使用 setseed() 设置随机种子以获得可重复结果
SELECT setseed(0.42);
SELECT random();  -- 每次设置相同种子后结果相同
```

### generate_series() 函数

`generate_series()` 是一个集合返回函数（SRF），可以生成一系列连续的值：

```sql
generate_series(start, stop)            -- 整数序列，步长为 1
generate_series(start, stop, step)      -- 整数序列，自定义步长
generate_series(start, stop, interval)  -- 时间序列
```

### 类型转换

PostgreSQL 提供两种显式类型转换方式：

```sql
-- SQL 标准语法
CAST(expression AS target_type)

-- PostgreSQL 特有语法（更简洁）
expression::target_type
```

常见的类型转换场景：

| 源类型 | 目标类型 | 示例 |
|--------|----------|------|
| 字符串 → 整数 | `'123'::INT` | `CAST('123' AS INTEGER)` |
| 整数 → 浮点 | `5::NUMERIC(10,2)` | `CAST(5 AS NUMERIC(10,2))` |
| 字符串 → 日期 | `'2025-06-15'::DATE` | `CAST('2025-06-15' AS DATE)` |
| 时间戳 → 日期 | `now()::DATE` | `CAST(now() AS DATE)` |
| 数值 → 字符串 | `42::TEXT` | `CAST(42 AS TEXT)` |
| 布尔 → 整数 | `TRUE::INT` | 结果为 1 |

### 隐式类型转换

PostgreSQL 会在某些场景下自动进行类型转换：
- `integer + numeric` → numeric
- 字符串与日期比较时自动解析
- 函数参数类型不完全匹配时

> **注意**：过度依赖隐式转换可能导致意外结果或性能问题，建议在关键场景中使用显式转换。

### 示例一：基本数学函数

```sql
SELECT
    name,
    price,
    cost,
    -- 利润率（保留两位小数）
    round((price - cost) / price * 100, 2) AS margin_pct,
    -- 向上取整到整元
    ceil(price) AS price_ceil,
    -- 向下取整
    floor(price) AS price_floor,
    -- 截断到一位小数（不四舍五入）
    trunc(price, 1) AS price_trunc
FROM products;
```

### 示例二：abs、mod、sign

```sql
-- 计算价格调整幅度
SELECT
    name,
    price,
    cost,
    price - cost AS profit,
    abs(price - cost) AS abs_profit,
    sign(price - cost) AS profit_sign,  -- 1=盈利, 0=持平, -1=亏损
    mod(quantity, 12) AS remainder_dozen  -- 除以12的余数（装箱余量）
FROM products;
```

### 示例三：power、sqrt、log

```sql
-- 复利计算：1000 元本金，年利率 5%，10 年后
SELECT
    round(1000 * power(1.05, 10), 2) AS future_value,
    sqrt(144) AS square_root,
    log(1000) AS log10_val,
    ln(exp(1.0)) AS natural_log_e;
```

### 示例四：random() 应用

```sql
-- 随机抽取 2 个产品做质检
SELECT name, price
FROM products
ORDER BY random()
LIMIT 2;

-- 生成随机折扣价（8折到9.5折之间）
SELECT
    name,
    price,
    round(price * (0.80 + random() * 0.15), 2) AS discount_price
FROM products;

-- 生成 10 个 1-100 的随机整数
SELECT floor(random() * 100 + 1)::INT AS random_num
FROM generate_series(1, 10);
```

### 示例五：generate_series() 整数序列

```sql
-- 生成 1 到 10 的序列
SELECT * FROM generate_series(1, 10);

-- 生成偶数序列：2, 4, 6, ..., 20
SELECT * FROM generate_series(2, 20, 2);

-- 倒序序列
SELECT * FROM generate_series(10, 1, -1);

-- 使用 generate_series 创建编号
SELECT
    'PROD-' || lpad(gs::TEXT, 4, '0') AS product_code
FROM generate_series(1, 20) AS gs;
```

### 示例六：generate_series() 日期序列

```sql
-- 生成近 7 天的日期
SELECT d::DATE AS date_val
FROM generate_series(
    current_date - INTERVAL '6 days',
    current_date,
    INTERVAL '1 day'
) AS d;

-- 生成 2025 年每月第一天
SELECT d::DATE AS month_start
FROM generate_series(
    '2025-01-01'::DATE,
    '2025-12-01'::DATE,
    INTERVAL '1 month'
) AS d;

-- 生成每小时时间点（用于时间轴报表）
SELECT ts
FROM generate_series(
    '2025-06-15 00:00:00'::TIMESTAMP,
    '2025-06-15 23:00:00'::TIMESTAMP,
    INTERVAL '1 hour'
) AS ts;
```

### 示例七：CAST 与 :: 类型转换

```sql
-- 各种类型转换示例
SELECT
    -- 字符串 → 数值
    CAST('3.14' AS NUMERIC) AS str_to_num,
    '42'::INTEGER AS str_to_int,

    -- 数值 → 字符串
    CAST(99.9 AS TEXT) AS num_to_str,
    price::TEXT || ' 元' AS price_text,

    -- 时间戳 → 日期
    now()::DATE AS today,

    -- 布尔转换
    CAST(1 AS BOOLEAN) AS int_to_bool,    -- true
    CAST('yes' AS BOOLEAN) AS str_to_bool  -- true
FROM products
LIMIT 1;
```

### 示例八：综合应用 — 销售数据分析

```sql
-- 综合分析产品数据
SELECT
    name,
    price,
    quantity,
    -- 库存金额（格式化为万元）
    round(price * quantity / 10000.0, 2) AS stock_value_wan,
    -- 每公斤单价
    round(price / NULLIF(weight_kg, 0), 2) AS price_per_kg,
    -- 库存等级
    CASE
        WHEN quantity > 5000 THEN '充足'
        WHEN quantity > 500 THEN '正常'
        ELSE '偏低'
    END AS stock_level,
    -- 用 generate_series 模拟月度预计销量
    round(quantity / 12.0) AS monthly_estimate
FROM products
ORDER BY price * quantity DESC;
```

### 示例九：数据填充与左连接

```sql
-- 使用 generate_series 生成日期范围，左连接实际数据以填充空白日期
WITH date_range AS (
    SELECT d::DATE AS report_date
    FROM generate_series('2025-06-01'::DATE, '2025-06-07'::DATE, '1 day') AS d
),
sample_orders AS (
    SELECT '2025-06-01'::DATE AS order_date, 5 AS cnt
    UNION ALL SELECT '2025-06-03', 8
    UNION ALL SELECT '2025-06-05', 3
    UNION ALL SELECT '2025-06-07', 12
)
SELECT
    dr.report_date,
    COALESCE(so.cnt, 0) AS order_count
FROM date_range dr
LEFT JOIN sample_orders so ON dr.report_date = so.order_date
ORDER BY dr.report_date;
```

## 常见问题

### Q1：round() 的四舍五入规则是什么？
**A**：PostgreSQL 的 `round()` 使用"银行家舍入法"（四舍六入五成双），即当要舍弃的部分恰好为 5 时，向最近的偶数方向舍入。例如 `round(2.5) → 2`，`round(3.5) → 4`。这与一些编程语言的行为不同。

### Q2：`::` 和 CAST 哪种写法更好？
**A**：`::` 是 PostgreSQL 特有语法，更简洁；`CAST` 是 SQL 标准语法，可移植性更好。如果你的 SQL 只在 PostgreSQL 中运行，`::` 更方便；如果需要跨数据库兼容，使用 `CAST`。

### Q3：generate_series 会不会占用大量内存？
**A**：`generate_series` 是一个集合返回函数，PostgreSQL 会以流式方式逐行产生数据，不会一次性将所有结果加载到内存中。但如果在子查询中物化了整个序列，仍可能消耗较多内存。对于非常大的序列，建议配合 `LIMIT` 使用。
