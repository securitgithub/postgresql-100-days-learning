# Day 042: 权限深入

## 学习目标

- 掌握 PostgreSQL 表级权限的精细控制（SELECT/INSERT/UPDATE/DELETE/TRUNCATE 等）
- 理解并使用列级权限实现字段粒度的访问控制
- 掌握 Schema 级权限的管理，理解 USAGE 与 CREATE 权限的作用
- 熟练使用 DEFAULT PRIVILEGES 自动为新对象设置权限
- 能够设计多层次的权限方案满足实际业务需求

## 理论讲解

### 1. PostgreSQL 权限体系概览

PostgreSQL 的权限体系是层次化的，从粗粒度到细粒度依次为：

```
集群 → 数据库 → Schema → 表/视图/序列/函数 → 列
```

每一层都有对应的权限控制。对象的所有者（Owner）默认拥有该对象的所有权限，并且可以将权限授予其他角色。

### 2. 表级权限详解

PostgreSQL 支持以下表级权限：

| 权限 | 说明 | 适用对象 |
|------|------|----------|
| `SELECT` | 读取表数据 | 表、视图、序列 |
| `INSERT` | 插入数据 | 表 |
| `UPDATE` | 更新数据 | 表、序列 |
| `DELETE` | 删除数据 | 表 |
| `TRUNCATE` | 清空表 | 表 |
| `REFERENCES` | 创建外键引用 | 表 |
| `TRIGGER` | 创建触发器 | 表 |

权限可以组合授予，也可以使用 `ALL PRIVILEGES` 授予全部权限。

### 3. 列级权限

当需要更精细的控制时，可以将 SELECT、INSERT、UPDATE、REFERENCES 权限限定到特定列：

- 列级权限仅支持这四种操作
- 如果同时存在表级和列级权限，以更宽松的为准
- 列级权限在 `information_schema.column_privileges` 中可查

### 4. Schema 级权限

Schema 是对象的命名空间，其权限包括：

- **USAGE**：允许访问 Schema 中的对象（前提条件）
- **CREATE**：允许在 Schema 中创建新对象

重要规则：即使用户拥有某张表的 SELECT 权限，如果没有该 Schema 的 USAGE 权限，也无法访问该表。

### 5. DEFAULT PRIVILEGES

`ALTER DEFAULT PRIVILEGES` 用于为将来创建的对象预设权限。这解决了一个常见问题：每次创建新表都需要手动 GRANT 权限。

关键点：
- DEFAULT PRIVILEGES 只影响**之后**创建的对象，不影响已存在的对象
- 可以针对特定 Schema 设置
- 可以指定由特定角色创建的对象才应用默认权限
- 支持表、序列、函数、类型、Schema 等对象类型

## 代码示例

### 示例 1：表级权限管理

```sql
-- 创建测试环境
CREATE TABLE employees (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(200),
    salary NUMERIC(12,2),
    department VARCHAR(50),
    hire_date DATE DEFAULT CURRENT_DATE
);

INSERT INTO employees (name, email, salary, department) VALUES
    ('张三', 'zhang@example.com', 15000.00, '技术部'),
    ('李四', 'li@example.com', 12000.00, '市场部'),
    ('王五', 'wang@example.com', 18000.00, '技术部');

-- 创建不同权限的角色
CREATE ROLE hr_reader NOLOGIN;
CREATE ROLE hr_editor NOLOGIN;
CREATE ROLE hr_admin NOLOGIN;

-- 授予只读权限
GRANT SELECT ON employees TO hr_reader;

-- 授予读写权限（不含删除和清空）
GRANT SELECT, INSERT, UPDATE ON employees TO hr_editor;
GRANT USAGE, SELECT ON SEQUENCE employees_id_seq TO hr_editor;

-- 授予全部权限
GRANT ALL PRIVILEGES ON employees TO hr_admin;
GRANT ALL PRIVILEGES ON SEQUENCE employees_id_seq TO hr_admin;

-- WITH GRANT OPTION：允许被授权者将权限再授予他人
GRANT SELECT ON employees TO hr_admin WITH GRANT OPTION;
```

### 示例 2：列级权限控制

```sql
-- 场景：普通员工可以查看姓名和部门，但不能查看薪资
CREATE ROLE employee_self NOLOGIN;

-- 授予特定列的 SELECT 权限（不包含 salary 列）
GRANT SELECT (id, name, email, department, hire_date) ON employees TO employee_self;

-- 测试（以 employee_self 身份）：
-- 以下查询会成功：
-- SELECT id, name, department FROM employees;

-- 以下查询会失败（没有 salary 列的权限）：
-- SELECT * FROM employees;
-- SELECT salary FROM employees;

-- 授予 HR 编辑人员可以更新特定列
CREATE ROLE hr_basic_editor NOLOGIN;
GRANT SELECT ON employees TO hr_basic_editor;
GRANT UPDATE (email, department) ON employees TO hr_basic_editor;

-- hr_basic_editor 可以更新 email 和 department，但不能更新 salary
-- UPDATE employees SET email = 'new@example.com' WHERE id = 1;  -- 成功
-- UPDATE employees SET salary = 20000 WHERE id = 1;              -- 失败

-- 列级 INSERT 权限
CREATE ROLE data_entry NOLOGIN;
GRANT INSERT (name, email, department) ON employees TO data_entry;
-- data_entry 只能插入指定列的值，其他列使用默认值
```

### 示例 3：Schema 级权限管理

```sql
-- 创建业务 Schema
CREATE SCHEMA sales;
CREATE SCHEMA inventory;

-- 在 Schema 中创建表
CREATE TABLE sales.orders (
    id SERIAL PRIMARY KEY,
    customer_name VARCHAR(100),
    total NUMERIC(12,2),
    order_date TIMESTAMP DEFAULT NOW()
);

CREATE TABLE inventory.products (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100),
    stock INT DEFAULT 0,
    price NUMERIC(10,2)
);

-- 创建部门角色
CREATE ROLE sales_team NOLOGIN;
CREATE ROLE inventory_team NOLOGIN;

-- 授予 Schema USAGE 权限（必须的第一步）
GRANT USAGE ON SCHEMA sales TO sales_team;
GRANT USAGE ON SCHEMA inventory TO inventory_team;

-- 授予表权限
GRANT SELECT, INSERT, UPDATE ON ALL TABLES IN SCHEMA sales TO sales_team;
GRANT USAGE, SELECT ON ALL SEQUENCES IN SCHEMA sales TO sales_team;

GRANT SELECT, INSERT, UPDATE ON ALL TABLES IN SCHEMA inventory TO inventory_team;
GRANT USAGE, SELECT ON ALL SEQUENCES IN SCHEMA inventory TO inventory_team;

-- 允许 sales_team 也能查看库存（只读）
GRANT USAGE ON SCHEMA inventory TO sales_team;
GRANT SELECT ON ALL TABLES IN SCHEMA inventory TO sales_team;

-- 授予 CREATE 权限允许在 Schema 中创建新对象
GRANT CREATE ON SCHEMA sales TO sales_team;
```

### 示例 4：DEFAULT PRIVILEGES 自动权限设置

```sql
-- 场景：DBA 角色 admin_dba 经常在 sales Schema 中创建表
-- 希望 sales_team 自动获得新表的 SELECT, INSERT, UPDATE 权限

-- 设置默认权限：admin_dba 在 sales Schema 中创建的表自动授权
ALTER DEFAULT PRIVILEGES FOR ROLE admin_dba IN SCHEMA sales
    GRANT SELECT, INSERT, UPDATE ON TABLES TO sales_team;

-- 设置默认权限：序列自动授予 USAGE 和 SELECT
ALTER DEFAULT PRIVILEGES FOR ROLE admin_dba IN SCHEMA sales
    GRANT USAGE, SELECT ON SEQUENCES TO sales_team;

-- 设置默认权限：函数自动授予 EXECUTE
ALTER DEFAULT PRIVILEGES FOR ROLE admin_dba IN SCHEMA sales
    GRANT EXECUTE ON FUNCTIONS TO sales_team;

-- 现在，admin_dba 创建的新表会自动被 sales_team 访问
-- 验证：以 admin_dba 身份创建新表
SET ROLE admin_dba;
CREATE TABLE sales.promotions (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100),
    discount NUMERIC(5,2)
);
RESET ROLE;

-- sales_team 成员无需额外 GRANT 即可访问 sales.promotions

-- 查看当前的默认权限设置
SELECT * FROM pg_default_acl;

-- 撤销默认权限
ALTER DEFAULT PRIVILEGES FOR ROLE admin_dba IN SCHEMA sales
    REVOKE INSERT, UPDATE ON TABLES FROM sales_team;
```

### 示例 5：权限查询与审计

```sql
-- 查看表级权限（ACL 格式）
SELECT schemaname, tablename, tableowner,
       array_to_string(ARRAY(
           SELECT unnest(relacl)::text
       ), E'\n') AS permissions
FROM pg_tables
JOIN pg_class ON relname = tablename
WHERE schemaname = 'public';

-- 查看详细的表权限（information_schema）
SELECT grantor, grantee, table_schema, table_name,
       privilege_type, is_grantable
FROM information_schema.table_privileges
WHERE table_schema = 'public'
ORDER BY table_name, grantee;

-- 查看列级权限
SELECT grantor, grantee, table_name, column_name,
       privilege_type, is_grantable
FROM information_schema.column_privileges
WHERE table_schema = 'public'
ORDER BY table_name, column_name;

-- 查看 Schema 权限
SELECT nspname AS schema_name,
       nspacl AS access_privileges
FROM pg_namespace
WHERE nspname NOT LIKE 'pg_%'
  AND nspname != 'information_schema';

-- 查看默认权限
SELECT pg_get_userbyid(d.defaclrole) AS owner,
       n.nspname AS schema,
       CASE d.defaclobjtype
           WHEN 'r' THEN 'table'
           WHEN 'S' THEN 'sequence'
           WHEN 'f' THEN 'function'
           WHEN 'T' THEN 'type'
           WHEN 'n' THEN 'schema'
       END AS object_type,
       d.defaclacl AS default_acl
FROM pg_default_acl d
LEFT JOIN pg_namespace n ON n.oid = d.defaclnamespace;
```

## 常见问题

### Q1：GRANT ALL TABLES 只对当前已有的表生效，新建的表需要重新 GRANT 吗？

**A：** 是的。`GRANT ... ON ALL TABLES IN SCHEMA xxx` 只对执行时已存在的表生效。对于之后创建的新表，需要：
1. 重新执行 GRANT 语句，或
2. 使用 `ALTER DEFAULT PRIVILEGES` 设置默认权限（推荐）

这是初学者最容易忽略的问题。最佳实践是同时执行两步：
```sql
-- 对已有表授权
GRANT SELECT ON ALL TABLES IN SCHEMA myschema TO reader;
-- 对未来新表设置默认权限
ALTER DEFAULT PRIVILEGES IN SCHEMA myschema GRANT SELECT ON TABLES TO reader;
```

### Q2：如何理解 PostgreSQL 的 ACL 字符串格式？

**A：** PostgreSQL 用 ACL（Access Control List）字符串表示权限，格式为 `grantee=privileges/grantor`。权限字母含义如下：

| 字母 | 权限 |
|------|------|
| `r` | SELECT (read) |
| `a` | INSERT (append) |
| `w` | UPDATE (write) |
| `d` | DELETE |
| `D` | TRUNCATE |
| `x` | REFERENCES |
| `t` | TRIGGER |
| `X` | EXECUTE |
| `U` | USAGE |
| `C` | CREATE |
| `c` | CONNECT |
| `T` | TEMPORARY |

例如 `alice=arwd/postgres` 表示 postgres 授予 alice SELECT、INSERT、UPDATE、DELETE 权限。

### Q3：撤销权限时，如何处理通过 WITH GRANT OPTION 传递出去的权限？

**A：** 使用 `REVOKE ... CASCADE` 可以级联撤销：
```sql
-- 假设 admin 授予 alice WITH GRANT OPTION，alice 又授予了 bob
-- 从 alice 级联撤销，bob 的权限也会被撤销
REVOKE SELECT ON employees FROM alice CASCADE;

-- 如果只想撤销 GRANT OPTION 而保留权限本身：
REVOKE GRANT OPTION FOR SELECT ON employees FROM alice CASCADE;
```
