## 理论讲解

### 1. 安装方式概览

PostgreSQL 提供多种安装方式：

| 方式 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| 包管理器（apt/yum） | 生产环境 Linux | 自动管理依赖、易于升级 | 版本可能不是最新 |
| 官方仓库 | 需要最新版本 | 版本最新、官方维护 | 需添加仓库源 |
| 源码编译 | 自定义需求 | 完全控制编译选项 | 复杂、耗时 |
| Docker | 开发与测试 | 隔离环境、快速部署 | 有一定学习成本 |
| Homebrew | macOS 开发 | 简单方便 | 仅适用于 macOS |
| 图形化安装器 | Windows/初学者 | 界面友好 | 灵活性较低 |

### 2. 目录结构

安装完成后，PostgreSQL 的关键目录包括：

```
/usr/lib/postgresql/16/       -- 程序文件（二进制）
/etc/postgresql/16/main/      -- 配置文件目录
/var/lib/postgresql/16/main/  -- 数据目录 (PGDATA)
    ├── base/                  -- 数据库文件
    ├── global/                -- 集群全局表
    ├── pg_wal/                -- WAL 预写日志
    ├── pg_xact/               -- 事务提交状态
    ├── pg_stat_tmp/           -- 统计临时文件
    ├── postgresql.conf        -- 主配置文件
    ├── pg_hba.conf            -- 客户端认证配置
    ├── pg_ident.conf          -- 用户名映射
    └── postmaster.pid         -- 进程ID文件
```

### 3. 三大配置文件详解

**3.1 postgresql.conf — 主配置文件**

这是 PostgreSQL 最重要的配置文件，控制数据库的运行行为。关键参数分类如下：

| 参数类别 | 关键参数 | 说明 |
|---------|---------|------|
| 连接设置 | listen_addresses | 监听地址，默认 localhost |
| 连接设置 | port | 端口号，默认 5432 |
| 连接设置 | max_connections | 最大连接数，默认 100 |
| 内存设置 | shared_buffers | 共享缓冲区，建议为内存的 25% |
| 内存设置 | work_mem | 排序/哈希操作内存 |
| 内存设置 | maintenance_work_mem | 维护操作内存（VACUUM等） |
| WAL 设置 | wal_level | WAL 级别（replica/logical） |
| 日志设置 | logging_collector | 是否启用日志收集器 |
| 日志设置 | log_directory | 日志文件目录 |
| 日志设置 | log_statement | 记录哪些 SQL 语句 |

**3.2 pg_hba.conf — 客户端认证配置**

HBA 即 Host-Based Authentication，用于控制谁可以从哪里以什么方式连接。每行的格式为：

```
# TYPE    DATABASE    USER    ADDRESS         METHOD
local     all         all                     peer
host      all         all     127.0.0.1/32    scram-sha-256
host      all         all     ::1/128         scram-sha-256
host      all         all     0.0.0.0/0       scram-sha-256
```

认证方法说明：

| 方法 | 说明 |
|------|------|
| trust | 无条件允许连接（不推荐在生产环境使用） |
| peer | 使用操作系统用户名认证（仅限本地连接） |
| scram-sha-256 | 密码认证（推荐） |
| md5 | MD5 密码认证（较老但常见） |
| reject | 拒绝连接 |
| cert | SSL 证书认证 |

**3.3 pg_ident.conf — 用户名映射**

用于将操作系统用户名映射为数据库用户名：

```
# MAPNAME    SYSTEM-USERNAME    PG-USERNAME
mymap        root               postgres
mymap        www-data           webuser
```

## 代码示例

### Linux (Ubuntu/Debian) 安装

```sql
-- 以下为终端命令，非 SQL

-- 1. 添加 PostgreSQL 官方仓库
sudo sh -c 'echo "deb http://apt.postgresql.org/pub/repos/apt $(lsb_release -cs)-pgdg main" > /etc/apt/sources.list.d/pgdg.list'

-- 2. 导入仓库签名密钥
wget --quiet -O - https://www.postgresql.org/media/keys/ACCC4CF8.asc | sudo apt-key add -

-- 3. 更新包列表并安装
sudo apt-get update
sudo apt-get install -y postgresql-16

-- 4. 验证安装
psql --version
sudo systemctl status postgresql
```

### Linux (CentOS/RHEL) 安装

```bash
# 1. 安装官方仓库 RPM
sudo dnf install -y https://download.postgresql.org/pub/repos/yum/reporpms/EL-9-x86_64/pgdg-redhat-repo-latest.noarch.rpm

# 2. 禁用内置 PostgreSQL 模块
sudo dnf -qy module disable postgresql

# 3. 安装 PostgreSQL 16
sudo dnf install -y postgresql16-server

# 4. 初始化数据库
sudo /usr/pgsql-16/bin/postgresql-16-setup initdb

# 5. 启动并设为开机自启
sudo systemctl start postgresql-16
sudo systemctl enable postgresql-16
```

### macOS 安装 (Homebrew)

```bash
# 1. 安装
brew install postgresql@16

# 2. 启动服务
brew services start postgresql@16

# 3. 验证
psql postgres -c "SELECT version();"
```

### Docker 安装（推荐用于学习）

```bash
# 1. 拉取镜像
docker pull postgres:16

# 2. 启动容器
docker run --name pg-learn \
  -e POSTGRES_PASSWORD=mypassword \
  -e POSTGRES_USER=learner \
  -e POSTGRES_DB=learndb \
  -p 5432:5432 \
  -v pgdata:/var/lib/postgresql/data \
  -d postgres:16

# 3. 连接到数据库
docker exec -it pg-learn psql -U learner -d learndb

# 4. 查看日志
docker logs pg-learn
```

### Windows 安装

```text
1. 下载安装程序：https://www.enterprisedb.com/downloads/postgres-postgresql-downloads
2. 运行安装向导，选择安装目录
3. 设置 postgres 用户密码
4. 选择端口号（默认 5432）
5. 选择区域设置（建议 Chinese, China 或 Default locale）
6. 完成安装后会自动启动服务
```

### 服务管理命令

```bash
# === systemd 管理（Ubuntu/Debian/CentOS） ===

# 启动
sudo systemctl start postgresql

# 停止
sudo systemctl stop postgresql

# 重启
sudo systemctl restart postgresql

# 重新加载配置（不中断连接）
sudo systemctl reload postgresql

# 查看状态
sudo systemctl status postgresql

# 设为开机自启
sudo systemctl enable postgresql

# === pg_ctl 命令（通用） ===

# 启动
pg_ctl -D /var/lib/postgresql/16/main start

# 停止（三种模式）
pg_ctl -D /var/lib/postgresql/16/main stop -m smart    # 等待所有连接关闭
pg_ctl -D /var/lib/postgresql/16/main stop -m fast     # 断开连接并正常关闭
pg_ctl -D /var/lib/postgresql/16/main stop -m immediate # 立即关闭（可能需恢复）

# 重新加载配置
pg_ctl -D /var/lib/postgresql/16/main reload

# 查看状态
pg_ctl -D /var/lib/postgresql/16/main status
```

### 修改配置并验证

```sql
-- 在 psql 中查看当前配置
SHOW listen_addresses;
SHOW port;
SHOW max_connections;
SHOW shared_buffers;

-- 查看配置文件位置
SHOW config_file;
SHOW hba_file;
SHOW data_directory;

-- 查看哪些配置需要重启才能生效
SELECT name, setting, context
FROM pg_settings
WHERE context = 'postmaster'
ORDER BY name;

-- 查看修改配置后是否需要重启
SELECT pg_reload_conf();  -- 重新加载配置（无需重启）
```

## 动手练习

1. **安装 PostgreSQL**：选择适合你操作系统的方式安装 PostgreSQL 16（推荐 Docker 方式），确保能成功执行 `psql --version`。

2. **首次连接**：使用 psql 连接到默认数据库，执行 `SELECT version();` 并截图记录输出。

3. **查找配置文件**：连接数据库后，使用 `SHOW config_file;` 和 `SHOW hba_file;` 找到配置文件位置，用文本编辑器打开查看内容。

4. **修改端口号**：将 PostgreSQL 监听端口从 5432 改为 5433，重启服务后验证连接。完成后改回默认端口。

5. **配置远程访问**：
   - 修改 `postgresql.conf` 的 `listen_addresses` 为 `'*'`
   - 在 `pg_hba.conf` 中添加一行允许特定 IP 段访问
   - 重新加载配置并验证

## 常见问题

### Q1: 安装后无法启动服务怎么办？

**A:** 常见原因和解决方法：
1. **端口冲突**：检查 5432 端口是否被占用，使用 `sudo lsof -i :5432` 或 `netstat -tlnp | grep 5432`
2. **权限问题**：确保数据目录的所有者是 postgres 用户，`sudo chown -R postgres:postgres /var/lib/postgresql/`
3. **查看日志**：检查 `/var/log/postgresql/` 下的日志文件获取详细错误信息
4. **数据目录未初始化**：运行 `initdb -D /path/to/data`

### Q2: peer 认证失败 "Peer authentication failed" 怎么处理？

**A:** 这是因为 pg_hba.conf 中本地连接使用了 `peer` 认证方式，要求操作系统用户名与数据库用户名一致。解决方法：
- 切换到 postgres 系统用户：`sudo -u postgres psql`
- 或修改 pg_hba.conf 将 `peer` 改为 `scram-sha-256`，然后设置密码

### Q3: postgresql.conf 修改后不生效？

**A:** 部分参数需要重启服务才能生效（context = 'postmaster'），而部分参数只需要 reload 即可。使用以下查询确认：
```sql
SELECT name, context FROM pg_settings WHERE name = '你的参数名';
```
如果 context 是 `postmaster`，则需要重启；如果是 `sighup` 或 `user`，则 reload 或重新连接即可。
