## 理论讲解

### MERGE 语句（PostgreSQL 15+）

MERGE 语句将 INSERT、UPDATE、DELETE 操作合并为一个原子操作，根据源数据与目标表的匹配情况执行不同的动作：

```sql
MERGE INTO target_table AS t
USING source_table AS s
ON t.key = s.key
WHEN MATCHED AND condition THEN
    UPDATE SET ...
WHEN MATCHED THEN
    DELETE
WHEN NOT MATCHED THEN
    INSERT (...) VALUES (...);
```

MERGE 的优势：
- 原子操作，无需多条 DML
- 语义清晰，表达"如果存在则更新，否则插入"
- SQL 标准语法，可移植性好

### INSERT ON CONFLICT（UPSERT）

PostgreSQL 9.5+ 引入的 UPSERT 机制，比 MERGE 更早可用：

```sql
INSERT INTO table (columns)
VALUES (values)
ON CONFLICT (conflict_target)
DO UPDATE SET column = EXCLUDED.column
-- 或
DO NOTHING;
```

- `conflict_target`：可以是列名、约束名或 WHERE 条件
- `EXCLUDED`：特殊表名，引用试图插入的行
- `DO UPDATE`：冲突时执行更新
- `DO NOTHING`：冲突时跳过

### RETURNING 子句

RETURNING 可以附加在 INSERT、UPDATE、DELETE 语句后，返回被操作的行：

```sql
INSERT INTO ... RETURNING *;
UPDATE ... RETURNING id, old_value, new_value;
DELETE FROM ... RETURNING *;
```

这避免了"先操作再查询"的额外往返，对性能和原子性都有好处。

### COPY 命令

COPY 是 PostgreSQL 最高效的批量数据传输方式：

```sql
-- 导出到文件
COPY table_name TO '/path/to/file.csv' WITH (FORMAT csv, HEADER);

-- 从文件导入
COPY table_name FROM '/path/to/file.csv' WITH (FORMAT csv, HEADER);

-- 客户端版本（通过 psql）
\copy table_name TO 'file.csv' WITH CSV HEADER
\copy table_name FROM 'file.csv' WITH CSV HEADER
```

COPY vs INSERT：
- COPY 直接以二进制或文本流加载数据，绕过 SQL 解析
- 大批量加载时，COPY 比逐行 INSERT 快 10-100 倍
- 支持 CSV、TEXT、BINARY 三种格式

## 代码示例

### 准备测试数据

```sql
-- 产品库存表
CREATE TABLE inventory (
    product_id   INTEGER PRIMARY KEY,
    product_name VARCHAR(100) NOT NULL,
    quantity     INTEGER NOT NULL DEFAULT 0,
    price        NUMERIC(10, 2),
    updated_at   TIMESTAMP DEFAULT now()
);

INSERT INTO inventory VALUES
    (1, '笔记本电脑', 100, 6999.00, now()),
    (2, '无线鼠标', 500, 129.90, now()),
    (3, '机械键盘', 200, 399.00, now()),
    (4, '显示器', 80, 2499.00, now());

-- 新到货数据（用作 MERGE/UPSERT 的源）
CREATE TABLE incoming_stock (
    product_id   INTEGER,
    product_name VARCHAR(100),
    quantity     INTEGER,
    price        NUMERIC(10, 2)
);

INSERT INTO incoming_stock VALUES
    (2, '无线鼠标', 200, 119.90),    -- 已存在：更新数量和价格
    (4, '显示器', 50, 2499.00),      -- 已存在：只加数量
    (5, 'USB Hub', 300, 89.00),      -- 新产品：插入
    (6, '摄像头', 150, 259.00);       -- 新产品：插入
```

### 示例一：MERGE 基本用法（PostgreSQL 15+）

```sql
-- 合并到货数据到库存表
MERGE INTO inventory AS inv
USING incoming_stock AS inc
ON inv.product_id = inc.product_id
WHEN MATCHED THEN
    UPDATE SET
        quantity = inv.quantity + inc.quantity,
        price = inc.price,
        updated_at = now()
WHEN NOT MATCHED THEN
    INSERT (product_id, product_name, quantity, price, updated_at)
    VALUES (inc.product_id, inc.product_name, inc.quantity, inc.price, now());
```

### 示例二：MERGE 带条件分支

```sql
-- 重置测试数据后执行
-- 匹配时根据不同条件执行不同操作
MERGE INTO inventory AS inv
USING incoming_stock AS inc
ON inv.product_id = inc.product_id
-- 匹配且价格有变化：更新
WHEN MATCHED AND inv.price <> inc.price THEN
    UPDATE SET
        quantity = inv.quantity + inc.quantity,
        price = inc.price,
        updated_at = now()
-- 匹配但价格不变：只更新数量
WHEN MATCHED THEN
    UPDATE SET
        quantity = inv.quantity + inc.quantity,
        updated_at = now()
-- 不匹配：插入
WHEN NOT MATCHED THEN
    INSERT (product_id, product_name, quantity, price)
    VALUES (inc.product_id, inc.product_name, inc.quantity, inc.price);
```

### 示例三：INSERT ON CONFLICT — DO NOTHING

```sql
-- 忽略冲突（重复插入时跳过）
INSERT INTO inventory (product_id, product_name, quantity, price)
VALUES
    (1, '笔记本电脑', 50, 6999.00),  -- 冲突：跳过
    (7, '耳机', 400, 199.00)          -- 新产品：插入
ON CONFLICT (product_id) DO NOTHING;
```

### 示例四：INSERT ON CONFLICT — DO UPDATE

```sql
-- UPSERT：存在则更新，不存在则插入
INSERT INTO inventory (product_id, product_name, quantity, price)
VALUES
    (2, '无线鼠标', 100, 109.90),
    (8, '鼠标垫', 1000, 29.90)
ON CONFLICT (product_id) DO UPDATE SET
    quantity = inventory.quantity + EXCLUDED.quantity,
    price = EXCLUDED.price,
    updated_at = now();

-- EXCLUDED 引用的是试图插入的那行数据
-- inventory 引用的是表中已存在的行
```

### 示例五：ON CONFLICT 带 WHERE 条件

```sql
-- 只在现有价格较高时才更新（保留最低价）
INSERT INTO inventory (product_id, product_name, quantity, price)
VALUES (2, '无线鼠标', 50, 99.90)
ON CONFLICT (product_id) DO UPDATE SET
    price = EXCLUDED.price,
    updated_at = now()
WHERE inventory.price > EXCLUDED.price;
-- 只有当库存中的价格高于新价格时才执行更新
```

### 示例六：ON CONFLICT 使用约束名

```sql
-- 使用约束名代替列名
INSERT INTO inventory (product_id, product_name, quantity, price)
VALUES (3, '机械键盘', 100, 379.00)
ON CONFLICT ON CONSTRAINT inventory_pkey DO UPDATE SET
    quantity = inventory.quantity + EXCLUDED.quantity,
    price = LEAST(inventory.price, EXCLUDED.price);
```

### 示例七：RETURNING 基本用法

```sql
-- INSERT 返回插入的行
INSERT INTO inventory (product_id, product_name, quantity, price)
VALUES (9, '转接头', 500, 39.90)
RETURNING *;

-- UPDATE 返回更新前后的关键信息
UPDATE inventory
SET price = price * 0.9, updated_at = now()
WHERE quantity > 300
RETURNING product_id, product_name, price AS new_price,
          price / 0.9 AS old_price;

-- DELETE 返回被删除的行
DELETE FROM inventory
WHERE quantity = 0
RETURNING *;
```

### 示例八：RETURNING 与 CTE 组合

```sql
-- 使用 RETURNING + CTE 实现"删除并归档"
CREATE TABLE inventory_archive (
    product_id   INTEGER,
    product_name VARCHAR(100),
    quantity     INTEGER,
    price        NUMERIC(10, 2),
    archived_at  TIMESTAMP DEFAULT now()
);

-- 一条语句完成：从主表删除 → 插入归档表
WITH deleted AS (
    DELETE FROM inventory
    WHERE product_id = 9
    RETURNING *
)
INSERT INTO inventory_archive (product_id, product_name, quantity, price)
SELECT product_id, product_name, quantity, price
FROM deleted;
```

### 示例九：RETURNING 与 UPSERT 配合

```sql
-- UPSERT 后返回结果，便于应用层判断是新增还是更新
INSERT INTO inventory (product_id, product_name, quantity, price)
VALUES (10, '网线', 200, 15.00)
ON CONFLICT (product_id) DO UPDATE SET
    quantity = inventory.quantity + EXCLUDED.quantity
RETURNING
    product_id,
    product_name,
    quantity,
    -- 判断是新增还是更新（如果 xmax=0 则为新插入）
    (xmax = 0) AS is_insert;
```

### 示例十二：批量 UPSERT 实践

```sql
-- 高效的批量 UPSERT 模式
-- 先将数据 COPY 到临时表，再用 INSERT ON CONFLICT 合并
CREATE TEMP TABLE staging_inventory (LIKE inventory INCLUDING ALL);

-- 批量加载到临时表（实际项目中用 COPY）
INSERT INTO staging_inventory VALUES
    (1, '笔记本电脑', 30, 6799.00, now()),
    (2, '无线鼠标', 100, 109.90, now()),
    (11, '充电器', 500, 49.90, now());

-- 从临时表合并到主表
INSERT INTO inventory
SELECT * FROM staging_inventory
ON CONFLICT (product_id) DO UPDATE SET
    quantity = inventory.quantity + EXCLUDED.quantity,
    price = EXCLUDED.price,
    updated_at = now();

-- 清理临时表
DROP TABLE staging_inventory;
```

## 常见问题

### Q1：MERGE 和 INSERT ON CONFLICT 应该选哪个？
**A**：如果使用 PostgreSQL 15+，MERGE 是更标准化的选择，尤其在需要同时处理 INSERT/UPDATE/DELETE 时。INSERT ON CONFLICT 在 PostgreSQL 9.5+ 就可以使用，且在简单 UPSERT 场景下语法更简洁。MERGE 支持多个 WHEN 分支和 DELETE 操作，灵活性更高。

### Q2：COPY 和 INSERT 在性能上差多少？
**A**：对于大批量数据（万行以上），COPY 通常比逐行 INSERT 快 10-100 倍。COPY 直接以流式协议传输数据，跳过了 SQL 解析和计划的开销。建议超过 1000 行的数据加载都使用 COPY。即使是应用程序中，很多驱动也支持 COPY 协议。

### Q3：RETURNING 能否返回更新前的旧值？
**A**：RETURNING 子句默认返回操作后的新值。要获取旧值，可以在 UPDATE 中通过子查询或者使用触发器。PostgreSQL 目前不直接支持 `RETURNING OLD.*` 语法，但可以通过 CTE 中的技巧间接实现（例如先查旧值，再更新）。
