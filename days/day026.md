# Day 026: 窗口函数（一）基础

## 学习目标

- 理解窗口函数的概念及其与聚合函数的核心区别
- 掌握 OVER() 子句的基本语法和作用
- 熟练使用 ROW_NUMBER()、RANK()、DENSE_RANK() 三种排名函数
- 理解 PARTITION BY 分区和 ORDER BY 排序的配合使用
- 能够在实际场景中选择合适的排名函数

## 理论讲解

### 什么是窗口函数

窗口函数（Window Function）对与当前行相关的一组行（称为"窗口"）执行计算，但**不会将多行折叠成一行**。这是它与聚合函数的核心区别：

| 特性 | 聚合函数 (GROUP BY) | 窗口函数 (OVER) |
|------|---------------------|-----------------|
| 是否折叠行 | 是，多行变一行 | 否，保留所有行 |
| 能否同时看到明细和汇总 | 不能 | 能 |
| 语法位置 | SELECT + GROUP BY | SELECT 中使用 OVER() |

### OVER() 子句

窗口函数的核心是 `OVER()` 子句，它定义了窗口的范围：

```sql
function_name(...) OVER (
    [PARTITION BY column1, column2, ...]
    [ORDER BY column3, column4, ...]
    [frame_clause]
)
```

- **PARTITION BY**：将结果集分成若干分区，窗口函数在每个分区内独立计算。类似于 GROUP BY，但不折叠行。
- **ORDER BY**：在每个分区内定义行的排列顺序，对排名函数尤其重要。
- **frame_clause**：定义窗口帧（详见 Day 027）。

### 三种排名函数

| 函数 | 说明 | 遇到并列时的行为 |
|------|------|------------------|
| `ROW_NUMBER()` | 为每行分配唯一的行号 | 并列值也分配不同的序号 |
| `RANK()` | 排名，并列值得到相同名次 | 并列后跳号（1,2,2,4） |
| `DENSE_RANK()` | 密集排名，并列值得到相同名次 | 并列后不跳号（1,2,2,3） |

### 排名函数对比示例

假设有分数：100, 95, 95, 90

| 分数 | ROW_NUMBER | RANK | DENSE_RANK |
|------|-----------|------|------------|
| 100 | 1 | 1 | 1 |
| 95 | 2 | 2 | 2 |
| 95 | 3 | 2 | 2 |
| 90 | 4 | 4 | 3 |

### 执行顺序

窗口函数在 `WHERE`、`GROUP BY`、`HAVING` 之后执行，但在 `ORDER BY` 和 `LIMIT` 之前。因此不能在 `WHERE` 中直接引用窗口函数的结果，需要使用子查询或 CTE。

### 示例一：最基本的 OVER() — 全局窗口

```sql
-- 每行都显示总销售额和总笔数
SELECT
    employee,
    amount,
    sale_date,
    SUM(amount) OVER () AS total_sales,   -- 不分区，全局汇总
    COUNT(*) OVER () AS total_rows
FROM sales
ORDER BY sale_date;
```

### 示例二：ROW_NUMBER() — 全局编号

```sql
-- 按销售额降序给所有记录编号
SELECT
    ROW_NUMBER() OVER (ORDER BY amount DESC) AS rn,
    employee,
    department,
    amount,
    sale_date
FROM sales;
```

### 示例三：PARTITION BY + ROW_NUMBER — 分区编号

```sql
-- 在每个部门内，按销售额降序编号
SELECT
    department,
    employee,
    amount,
    sale_date,
    ROW_NUMBER() OVER (
        PARTITION BY department
        ORDER BY amount DESC
    ) AS dept_rank
FROM sales;
```

### 示例四：三种排名函数对比

```sql
-- 对比 ROW_NUMBER、RANK、DENSE_RANK
SELECT
    employee,
    amount,
    ROW_NUMBER() OVER (ORDER BY amount DESC) AS row_num,
    RANK()       OVER (ORDER BY amount DESC) AS rank_val,
    DENSE_RANK() OVER (ORDER BY amount DESC) AS dense_rank_val
FROM sales;
-- 注意观察 amount=25000 的两条记录（李四1月和2月）
```

### 示例五：Top-N 查询 — 每个部门销售额前 2 名

```sql
-- 使用子查询过滤窗口函数结果
SELECT * FROM (
    SELECT
        department,
        employee,
        amount,
        sale_date,
        ROW_NUMBER() OVER (
            PARTITION BY department
            ORDER BY amount DESC
        ) AS rn
    FROM sales
) ranked
WHERE rn <= 2
ORDER BY department, rn;
```

### 示例六：每人最近一笔销售记录

```sql
-- 取每个员工的最新一条销售记录
SELECT * FROM (
    SELECT
        employee,
        department,
        amount,
        sale_date,
        ROW_NUMBER() OVER (
            PARTITION BY employee
            ORDER BY sale_date DESC
        ) AS rn
    FROM sales
) latest
WHERE rn = 1
ORDER BY employee;
```

### 示例七：排名与百分比

```sql
-- 显示每笔销售在部门内的排名和占比
SELECT
    department,
    employee,
    amount,
    RANK() OVER w AS dept_rank,
    ROUND(
        amount / SUM(amount) OVER (PARTITION BY department) * 100, 1
    ) AS dept_pct
FROM sales
WINDOW w AS (PARTITION BY department ORDER BY amount DESC);
-- WINDOW 子句定义可复用的窗口规范
```

### 示例八：WINDOW 子句复用

```sql
-- 使用 WINDOW 子句避免重复定义
SELECT
    employee,
    department,
    amount,
    ROW_NUMBER() OVER w AS rn,
    RANK()       OVER w AS rnk,
    DENSE_RANK() OVER w AS drnk
FROM sales
WINDOW w AS (PARTITION BY department ORDER BY amount DESC)
ORDER BY department, amount DESC;
```

### 示例九：去重 — 保留每组最新一条

```sql
-- 假设有重复数据，需要按某个条件去重保留一条
-- 常见场景：保留每个员工每月最高的一笔销售
SELECT * FROM (
    SELECT
        employee,
        extract(MONTH FROM sale_date) AS month,
        amount,
        sale_date,
        ROW_NUMBER() OVER (
            PARTITION BY employee, extract(MONTH FROM sale_date)
            ORDER BY amount DESC
        ) AS rn
    FROM sales
) deduped
WHERE rn = 1
ORDER BY employee, month;
```

## 常见问题

### Q1：能否在 WHERE 子句中使用窗口函数？
**A**：不能。窗口函数在 WHERE 之后执行，因此不能直接在 WHERE 中引用。解决方法是将窗口函数放在子查询或 CTE 中，然后在外层查询的 WHERE 中过滤。

### Q2：ROW_NUMBER 遇到并列值时顺序是确定的吗？
**A**：不确定。当 ORDER BY 的列存在重复值时，ROW_NUMBER 会为它们分配不同的序号，但具体谁先谁后是不确定的（非确定性）。如果需要确定性结果，应在 ORDER BY 中添加额外的列（如主键）作为 tiebreaker。

### Q3：PARTITION BY 可以使用多个列吗？
**A**：可以。`PARTITION BY col1, col2` 会按 col1 和 col2 的组合进行分区，类似于 `GROUP BY col1, col2`。
