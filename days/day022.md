## 理论讲解

### CASE 表达式概述

CASE 表达式是 SQL 中实现条件逻辑的核心工具，类似于编程语言中的 if-else。PostgreSQL 支持两种形式：

### 简单 CASE 表达式

```
CASE expression
    WHEN value1 THEN result1
    WHEN value2 THEN result2
    ...
    ELSE default_result
END
```

将 `expression` 的值逐一与每个 `WHEN` 后的值比较，匹配则返回对应的 `THEN` 结果。

### 搜索 CASE 表达式

```
CASE
    WHEN condition1 THEN result1
    WHEN condition2 THEN result2
    ...
    ELSE default_result
END
```

逐一评估每个 `WHEN` 后的布尔条件，第一个为真的条件对应的结果被返回。搜索 CASE 更加灵活，可以使用任意条件表达式。

### COALESCE 函数

`COALESCE(val1, val2, ..., valN)` 返回参数列表中第一个非 NULL 的值。它是处理 NULL 值最常用的函数，等价于：

```sql
CASE WHEN val1 IS NOT NULL THEN val1
     WHEN val2 IS NOT NULL THEN val2
     ...
     ELSE valN END
```

### NULLIF 函数

`NULLIF(a, b)` 在 `a = b` 时返回 NULL，否则返回 `a`。常用于避免除零错误：

```sql
-- 避免除零：当 divisor 为 0 时返回 NULL 而非报错
SELECT numerator / NULLIF(divisor, 0) FROM ...
```

### GREATEST 和 LEAST

- `GREATEST(val1, val2, ...)` 返回参数中的最大值
- `LEAST(val1, val2, ...)` 返回参数中的最小值
- 如果任一参数为 NULL，PostgreSQL 会忽略 NULL（与某些数据库不同）

### 示例一：简单 CASE — 等级描述

```sql
-- 将字母等级转换为中文描述
SELECT name, score, grade,
    CASE grade
        WHEN 'A' THEN '优秀'
        WHEN 'B' THEN '良好'
        WHEN 'C' THEN '中等'
        WHEN 'D' THEN '及格'
        ELSE '不及格'
    END AS grade_desc
FROM students
ORDER BY score DESC;
```

### 示例二：搜索 CASE — 分数区间判断

```sql
-- 根据分数范围进行评价（搜索 CASE 更灵活）
SELECT name, score,
    CASE
        WHEN score >= 90 THEN '优秀 (90+)'
        WHEN score >= 80 THEN '良好 (80-89)'
        WHEN score >= 70 THEN '中等 (70-79)'
        WHEN score >= 60 THEN '及格 (60-69)'
        ELSE '不及格 (<60)'
    END AS evaluation
FROM students;
```

### 示例三：CASE 用于聚合统计

```sql
-- 用 CASE + SUM 实现行转列的交叉统计
SELECT
    city,
    COUNT(*) AS total,
    SUM(CASE WHEN score >= 80 THEN 1 ELSE 0 END) AS high_score_count,
    SUM(CASE WHEN score < 60 THEN 1 ELSE 0 END) AS fail_count
FROM students
WHERE city IS NOT NULL
GROUP BY city
ORDER BY total DESC;
```

### 示例四：COALESCE 处理 NULL

```sql
-- 使用 COALESCE 为 NULL 值提供默认值
SELECT name,
    COALESCE(city, '未填写') AS city,
    COALESCE(bonus, 0) AS bonus,
    score + COALESCE(bonus, 0) AS total_score
FROM students;
```

### 示例五：多级 COALESCE

```sql
-- COALESCE 支持多个备选值，返回第一个非 NULL 的
SELECT COALESCE(NULL, NULL, 'third', 'fourth') AS result;
-- 结果：'third'

-- 实际场景：优先使用手机号，其次座机，最后显示"无联系方式"
-- SELECT COALESCE(mobile, phone, '无联系方式') AS contact FROM users;
```

### 示例六：NULLIF 避免除零错误

```sql
-- 模拟除法场景
SELECT
    name,
    score,
    COALESCE(bonus, 0) AS bonus,
    -- 如果 bonus 为 0 或 NULL，避免除零
    ROUND(score::NUMERIC / NULLIF(COALESCE(bonus, 0), 0), 2) AS score_per_bonus
FROM students;
```

### 示例七：GREATEST 和 LEAST

```sql
-- 取多个值中的最大和最小
SELECT
    name,
    score,
    COALESCE(bonus, 0) AS bonus,
    GREATEST(score, COALESCE(bonus, 0) * 10) AS max_val,
    LEAST(score, 100) AS capped_score  -- 封顶100分
FROM students;
```

### 示例八：CASE 在 ORDER BY 中使用

```sql
-- 自定义排序：优秀排前面，不及格排后面
SELECT name, score, grade FROM students
ORDER BY
    CASE grade
        WHEN 'A' THEN 1
        WHEN 'B' THEN 2
        WHEN 'C' THEN 3
        WHEN 'D' THEN 4
        ELSE 5
    END;
```

### 示例九：CASE 在 UPDATE 中使用

```sql
-- 批量更新：根据条件设置不同的 bonus
UPDATE students
SET bonus = CASE
    WHEN score >= 90 THEN 20
    WHEN score >= 80 THEN 15
    WHEN score >= 60 THEN 10
    ELSE 5
END
WHERE bonus IS NULL;
```

## 常见问题

### Q1：CASE 表达式中不写 ELSE 会怎样？
**A**：如果所有 WHEN 条件都不匹配且没有 ELSE 子句，CASE 表达式返回 NULL。建议总是写上 ELSE 以明确预期行为。

### Q2：COALESCE 和 CASE WHEN ... IS NOT NULL 有什么区别？
**A**：功能上等价，但 COALESCE 更简洁。COALESCE 是 SQL 标准函数，也更易读。对于简单的 NULL 替换场景推荐使用 COALESCE。

### Q3：CASE 表达式可以嵌套使用吗？
**A**：可以，CASE 表达式可以嵌套在另一个 CASE 的 THEN 或 ELSE 中。但深层嵌套会降低可读性，建议超过两层时考虑重构查询逻辑。
