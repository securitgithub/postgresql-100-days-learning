# Day 038: 数组与复合类型

- 掌握 PostgreSQL 数组类型的定义、插入与查询

- 能在实际场景中合理运用数组和复合类型

## 理论讲解

### 数组类型

PostgreSQL 是少数原生支持数组类型的关系数据库之一。任何内置或用户定义的基础类型都可以声明为数组。数组适用于：

- 存储一组同类型的值（标签、权限列表等）
- 避免为简单的一对多关系创建额外的关联表
- 函数返回多个值

**数组维度：** PostgreSQL 支持多维数组，但一维数组最常用。数组下标默认从 1 开始（不是 0）。

### 数组操作符

| 操作符 | 说明 | 示例 |
|--------|------|------|
| `@>` | 包含 | `ARRAY[1,2,3] @> ARRAY[2,3]` → true |
| `<@` | 被包含 | `ARRAY[2] <@ ARRAY[1,2,3]` → true |
| `&&` | 有交集（重叠） | `ARRAY[1,2] && ARRAY[2,3]` → true |
| `\|\|` | 连接 | `ARRAY[1,2] \|\| ARRAY[3]` → {1,2,3} |
| `=` | 相等 | `ARRAY[1,2] = ARRAY[1,2]` → true |
| `<>` | 不等 | `ARRAY[1,2] <> ARRAY[2,1]` → true |
| `[n]` | 下标访问 | `(ARRAY[10,20,30])[2]` → 20 |
| `[m:n]` | 切片 | `(ARRAY[10,20,30,40])[2:3]` → {20,30} |

### 数组函数

| 函数 | 说明 |
|------|------|
| `array_agg(expr)` | 聚合函数，将多行值合并为数组 |
| `unnest(array)` | 将数组展开为多行 |
| `array_length(array, dim)` | 返回指定维度的数组长度 |
| `cardinality(array)` | 返回数组中元素的总数 |
| `array_append(array, elem)` | 在数组末尾添加元素 |
| `array_prepend(elem, array)` | 在数组开头添加元素 |
| `array_remove(array, elem)` | 移除数组中所有匹配的元素 |
| `array_replace(array, old, new)` | 替换数组中的元素 |
| `array_cat(array1, array2)` | 连接两个数组 |
| `array_position(array, elem)` | 返回元素在数组中的位置 |
| `array_positions(array, elem)` | 返回元素所有出现位置 |
| `array_to_string(array, sep)` | 数组转字符串 |
| `string_to_array(str, sep)` | 字符串转数组 |

### 复合类型

复合类型是由多个字段组成的自定义类型，类似于 C 语言的结构体。每个表在创建时会自动创建一个同名的复合类型。

```sql
CREATE TYPE type_name AS (
    field1 type1,
    field2 type2,
    ...
);
```

## 代码示例

### 示例 1：数组的定义与基本操作

```sql
-- 创建包含数组列的表
CREATE TABLE articles (
    id         SERIAL PRIMARY KEY,
    title      VARCHAR(200) NOT NULL,
    tags       TEXT[] NOT NULL DEFAULT '{}',   -- 文本数组
    scores     INTEGER[],                       -- 整数数组
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 插入数组数据（多种语法）
-- 方式1：ARRAY 构造器
INSERT INTO articles (title, tags, scores)
VALUES ('PostgreSQL入门', ARRAY['数据库', 'PostgreSQL', '教程'], ARRAY[95, 88, 92]);

-- 方式2：大括号字面量
INSERT INTO articles (title, tags, scores)
VALUES ('Python编程', '{Python, 编程, 入门}', '{90, 85}');

-- 方式3：空数组
INSERT INTO articles (title, tags, scores)
VALUES ('草稿文章', '{}', NULL);

-- 查询数组
SELECT title, tags, scores FROM articles;

-- 访问数组元素（下标从1开始）
SELECT title, tags[1] AS first_tag, scores[1] AS first_score
FROM articles;

-- 数组切片
SELECT title, tags[1:2] AS first_two_tags
FROM articles;

-- 获取数组长度
SELECT title,
       array_length(tags, 1) AS tag_count,
       cardinality(scores) AS score_count
FROM articles;
```

### 示例 2：数组操作符查询

```sql
-- @> 包含：查找包含特定标签的文章
SELECT title, tags
FROM articles
WHERE tags @> ARRAY['PostgreSQL'];
-- 返回标签中包含 'PostgreSQL' 的文章

-- <@ 被包含：查找标签是指定集合子集的文章
SELECT title, tags
FROM articles
WHERE tags <@ ARRAY['数据库', 'PostgreSQL', '教程', 'SQL'];

-- && 重叠（有交集）：查找包含任意一个指定标签的文章
SELECT title, tags
FROM articles
WHERE tags && ARRAY['Python', 'PostgreSQL'];
-- 标签中包含 'Python' 或 'PostgreSQL' 的文章

-- || 连接：追加标签
UPDATE articles
SET tags = tags || ARRAY['热门']
WHERE id = 1;

-- 也可以追加单个元素
UPDATE articles
SET tags = tags || '推荐'
WHERE id = 1;

-- 使用 ANY 判断数组是否包含某值
SELECT title FROM articles
WHERE '数据库' = ANY(tags);

-- 使用 ALL 判断数组所有元素满足条件
SELECT title FROM articles
WHERE 80 < ALL(scores);  -- 所有分数都大于80
```

### 示例 3：数组函数实战

```sql
-- array_agg：将多行聚合为数组
SELECT department_id,
       array_agg(first_name ORDER BY salary DESC) AS employee_names,
       array_agg(salary ORDER BY salary DESC) AS salaries
FROM employees
GROUP BY department_id;

-- unnest：将数组展开为多行
SELECT id, title, unnest(tags) AS tag
FROM articles;
-- 每个标签展开为独立的一行

-- unnest 配合 DISTINCT 统计所有唯一标签
SELECT DISTINCT unnest(tags) AS tag
FROM articles
ORDER BY tag;

-- array_append / array_remove：修改数组
UPDATE articles
SET tags = array_append(tags, '精选')
WHERE id = 1;

UPDATE articles
SET tags = array_remove(tags, '入门')
WHERE id = 1;

-- array_replace：替换元素
UPDATE articles
SET tags = array_replace(tags, '教程', '高级教程')
WHERE id = 1;

-- array_position：查找元素位置
SELECT title, array_position(tags, 'PostgreSQL') AS pg_pos
FROM articles;

-- array_to_string：数组转逗号分隔字符串
SELECT title, array_to_string(tags, ', ') AS tags_str
FROM articles;

-- string_to_array：字符串转数组
SELECT string_to_array('apple,banana,cherry', ',');
-- 结果：{apple,banana,cherry}
```

### 示例 4：数组与 GIN 索引

```sql
-- 为数组列创建 GIN 索引（大幅提升包含查询性能）
CREATE INDEX idx_articles_tags ON articles USING GIN(tags);

-- 以下查询会使用 GIN 索引
SELECT * FROM articles WHERE tags @> ARRAY['PostgreSQL'];
SELECT * FROM articles WHERE tags && ARRAY['Python', 'SQL'];

-- 验证索引使用
EXPLAIN ANALYZE
SELECT * FROM articles WHERE tags @> ARRAY['PostgreSQL'];
```

### 示例 5：复合类型的定义与使用

```sql
-- 创建复合类型
CREATE TYPE address_type AS (
    street  TEXT,
    city    TEXT,
    state   TEXT,
    zipcode VARCHAR(10),
    country VARCHAR(50)
);

CREATE TYPE money_type AS (
    amount   NUMERIC(12,2),
    currency VARCHAR(3)
);

-- 在表中使用复合类型
CREATE TABLE companies (
    id              SERIAL PRIMARY KEY,
    name            VARCHAR(200) NOT NULL,
    headquarters    address_type,        -- 复合类型列
    annual_revenue  money_type
);

-- 插入复合类型数据
INSERT INTO companies (name, headquarters, annual_revenue)
VALUES (
    '示例科技有限公司',
    ROW('中关村大街1号', '北京', '北京市', '100080', '中国'),
    ROW(5000000.00, 'CNY')
);

-- 另一种插入语法
INSERT INTO companies (name, headquarters, annual_revenue)
VALUES (
    '另一家公司',
    ('南京路100号', '上海', '上海市', '200000', '中国')::address_type,
    (1000000.00, 'USD')::money_type
);

-- 查询复合类型的字段（使用括号和点号）
SELECT name,
       (headquarters).city AS city,
       (headquarters).street AS street,
       (annual_revenue).amount AS revenue,
       (annual_revenue).currency AS currency
FROM companies;

-- 在 WHERE 中使用复合类型字段
SELECT name
FROM companies
WHERE (headquarters).city = '北京';

-- 更新复合类型的单个字段
UPDATE companies
SET headquarters.city = '深圳'
WHERE id = 1;
```

### 示例 6：复合类型数组

```sql
-- 复合类型的数组：存储多个地址
CREATE TABLE customers (
    id        SERIAL PRIMARY KEY,
    name      VARCHAR(100) NOT NULL,
    addresses address_type[]   -- 复合类型的数组
);

INSERT INTO customers (name, addresses)
VALUES (
    '张三',
    ARRAY[
        ROW('工作地址路1号', '北京', '北京市', '100000', '中国'),
        ROW('家庭地址路2号', '广州', '广东省', '510000', '中国')
    ]::address_type[]
);

-- 查询数组中的复合类型
SELECT name,
       (addresses[1]).city AS first_city,
       (addresses[2]).city AS second_city,
       array_length(addresses, 1) AS address_count
FROM customers;
```

### 示例 7：实用场景 — 标签系统

```sql
-- 方案一：使用数组存储标签（简单场景）
CREATE TABLE posts (
    id         SERIAL PRIMARY KEY,
    title      VARCHAR(200) NOT NULL,
    content    TEXT,
    tags       TEXT[] DEFAULT '{}',
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_posts_tags ON posts USING GIN(tags);

-- 添加标签（避免重复）
CREATE OR REPLACE FUNCTION add_tag(p_post_id INTEGER, p_tag TEXT)
RETURNS TEXT[]
LANGUAGE plpgsql
AS $$
DECLARE
    v_tags TEXT[];
BEGIN
    SELECT tags INTO v_tags FROM posts WHERE id = p_post_id;

    -- 检查标签是否已存在
    IF p_tag = ANY(v_tags) THEN
        RETURN v_tags;  -- 已存在，不重复添加
    END IF;

    UPDATE posts
    SET tags = array_append(tags, p_tag)
    WHERE id = p_post_id
    RETURNING tags INTO v_tags;

    RETURN v_tags;
END;
$$;

-- 移除标签
CREATE OR REPLACE FUNCTION remove_tag(p_post_id INTEGER, p_tag TEXT)
RETURNS TEXT[]
LANGUAGE plpgsql
AS $$
DECLARE
    v_tags TEXT[];
BEGIN
    UPDATE posts
    SET tags = array_remove(tags, p_tag)
    WHERE id = p_post_id
    RETURNING tags INTO v_tags;

    RETURN v_tags;
END;
$$;

-- 按标签统计文章数量
SELECT tag, COUNT(*) AS post_count
FROM posts, unnest(tags) AS tag
GROUP BY tag
ORDER BY post_count DESC;

-- 查找同时包含多个标签的文章
SELECT * FROM posts
WHERE tags @> ARRAY['PostgreSQL', '数据库'];

-- 查找包含任意一个标签的文章
SELECT * FROM posts
WHERE tags && ARRAY['Python', 'PostgreSQL'];
```

## 常见问题

### Q1：什么时候用数组，什么时候用关联表？

**用数组的场景：**
- 数据量小（通常 < 100 个元素）
- 总是作为整体读取和写入
- 不需要对数组元素建立复杂的关联关系
- 典型例子：标签、权限列表、简单配置

**用关联表的场景：**
- 数据量大或不确定
- 需要对元素进行独立的查询、更新、删除
- 元素本身有复杂的属性
- 需要在元素间建立关系
- 需要外键约束保证引用完整性

### Q2：数组查询性能如何？需要注意什么？

未建索引时，数组查询需要全表扫描。建议：
- 对频繁查询的数组列创建 GIN 索引
- GIN 索引支持 `@>`、`<@`、`&&`、`=` 操作符
- `ANY()` 和 `ALL()` 不能直接使用 GIN 索引
- 避免数组过大（上百个元素），考虑改用关联表

### Q3：复合类型和 JSONB 该怎么选？

| 特性 | 复合类型 | JSONB |
|------|----------|-------|
| 结构 | 固定字段，强类型 | 灵活字段，弱类型 |
| 性能 | 访问字段更快 | 需要解析，稍慢 |
| 索引 | 有限支持 | GIN 索引功能强大 |
| 修改结构 | 需要 ALTER TYPE | 直接添加新字段 |
| 适用场景 | 结构固定的嵌套数据 | 结构灵活或经常变化的数据 |

实际项目中，JSONB 使用更广泛，因为它更灵活且工具支持更好。
