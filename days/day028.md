# Day 028: 窗口函数（三）聚合窗口

## 学习目标

- 掌握聚合函数（SUM、AVG、COUNT、MIN、MAX）作为窗口函数使用的方法
- 熟练实现累计求和（Running Total）
- 理解移动平均（Moving Average）的计算方式与窗口帧配置
- 能够使用聚合窗口函数计算占比、百分比等业务指标
- 综合运用聚合窗口函数解决复杂的数据分析需求

## 理论讲解

### 聚合函数 + OVER()

几乎所有的聚合函数都可以通过添加 `OVER()` 子句变成窗口函数：

```sql
SUM(amount)   OVER (...)
AVG(amount)   OVER (...)
COUNT(*)      OVER (...)
MIN(amount)   OVER (...)
MAX(amount)   OVER (...)
```

不同于普通聚合函数会折叠行，聚合窗口函数保留所有原始行，同时在每一行上显示聚合结果。

### 累计求和（Running Total）

累计求和是聚合窗口最经典的用法。通过 ORDER BY 指定排序后，默认帧是从分区起点到当前行：

```sql
SUM(amount) OVER (ORDER BY sale_date)
-- 等价于
SUM(amount) OVER (
    ORDER BY sale_date
    ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
)
```

### 移动平均（Moving Average）

移动平均需要自定义窗口帧，指定计算范围：

```sql
-- 3期移动平均（当前行 + 前2行）
AVG(amount) OVER (
    ORDER BY sale_date
    ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
)

-- 5期居中移动平均（前2行 + 当前行 + 后2行）
AVG(amount) OVER (
    ORDER BY sale_date
    ROWS BETWEEN 2 PRECEDING AND 2 FOLLOWING
)
```

### 百分比计算

通过将当前行值除以分区总和来计算占比：

```sql
-- 每行占分区总和的百分比
amount / SUM(amount) OVER (PARTITION BY department) * 100
```

### 帧范围对聚合窗口的影响

| 帧定义 | 效果 |
|--------|------|
| 无 ORDER BY，空 OVER() | 全局聚合 |
| 有 ORDER BY，默认帧 | 累计聚合（从头到当前行） |
| `ROWS BETWEEN n PRECEDING AND CURRENT ROW` | n+1 行滑动窗口 |
| `ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING` | 整个分区 |

## 代码示例

### 准备测试数据

```sql
-- 沿用 Day 026 的 sales 表，同时创建月度汇总表
CREATE TABLE monthly_revenue (
    month       DATE,
    department  VARCHAR(50),
    revenue     NUMERIC(12, 2)
);

INSERT INTO monthly_revenue VALUES
    ('2025-01-01', '销售一部', 40000),
    ('2025-02-01', '销售一部', 47000),
    ('2025-03-01', '销售一部', 48000),
    ('2025-04-01', '销售一部', 52000),
    ('2025-05-01', '销售一部', 55000),
    ('2025-06-01', '销售一部', 60000),
    ('2025-01-01', '销售二部', 32000),
    ('2025-02-01', '销售二部', 47000),
    ('2025-03-01', '销售二部', 57000),
    ('2025-04-01', '销售二部', 45000),
    ('2025-05-01', '销售二部', 50000),
    ('2025-06-01', '销售二部', 62000);
```

### 示例一：全局聚合窗口

```sql
-- 每行显示全局汇总数据
SELECT
    month,
    department,
    revenue,
    SUM(revenue) OVER () AS total_revenue,           -- 全局总和
    ROUND(AVG(revenue) OVER (), 2) AS avg_revenue,   -- 全局平均
    COUNT(*) OVER () AS total_rows,                   -- 总行数
    MIN(revenue) OVER () AS global_min,
    MAX(revenue) OVER () AS global_max
FROM monthly_revenue
ORDER BY department, month;
```

### 示例二：分区聚合

```sql
-- 每行显示所属部门的汇总数据
SELECT
    department,
    month,
    revenue,
    SUM(revenue) OVER (PARTITION BY department) AS dept_total,
    ROUND(AVG(revenue) OVER (PARTITION BY department), 2) AS dept_avg,
    COUNT(*) OVER (PARTITION BY department) AS dept_count
FROM monthly_revenue
ORDER BY department, month;
```

### 示例三：累计求和

```sql
-- 每个部门按月累计营收
SELECT
    department,
    to_char(month, 'YYYY-MM') AS month_str,
    revenue,
    SUM(revenue) OVER (
        PARTITION BY department
        ORDER BY month
    ) AS cumulative_revenue
FROM monthly_revenue
ORDER BY department, month;
```

### 示例四：累计计数与累计平均

```sql
-- 累计笔数和累计平均
SELECT
    department,
    to_char(month, 'YYYY-MM') AS month_str,
    revenue,
    COUNT(*) OVER (
        PARTITION BY department ORDER BY month
    ) AS cum_count,
    ROUND(AVG(revenue) OVER (
        PARTITION BY department ORDER BY month
    ), 2) AS cum_avg,
    SUM(revenue) OVER (
        PARTITION BY department ORDER BY month
    ) AS cum_sum
FROM monthly_revenue
ORDER BY department, month;
```

### 示例五：移动平均

```sql
-- 3个月移动平均（当前月 + 前两个月）
SELECT
    department,
    to_char(month, 'YYYY-MM') AS month_str,
    revenue,
    ROUND(AVG(revenue) OVER (
        PARTITION BY department
        ORDER BY month
        ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
    ), 2) AS moving_avg_3m,
    -- 移动平均涵盖的实际行数
    COUNT(*) OVER (
        PARTITION BY department
        ORDER BY month
        ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
    ) AS window_size
FROM monthly_revenue
ORDER BY department, month;
```

### 示例六：百分比计算

```sql
-- 每笔收入占部门总收入的百分比
SELECT
    department,
    to_char(month, 'YYYY-MM') AS month_str,
    revenue,
    SUM(revenue) OVER (PARTITION BY department) AS dept_total,
    ROUND(
        revenue * 100.0 / SUM(revenue) OVER (PARTITION BY department),
        2
    ) AS pct_of_dept,
    -- 占全公司总收入的百分比
    ROUND(
        revenue * 100.0 / SUM(revenue) OVER (),
        2
    ) AS pct_of_total
FROM monthly_revenue
ORDER BY department, month;
```

### 示例七：累计百分比（帕累托分析）

```sql
-- 按收入降序排列，计算累计百分比（用于80/20分析）
SELECT
    department,
    to_char(month, 'YYYY-MM') AS month_str,
    revenue,
    SUM(revenue) OVER (ORDER BY revenue DESC) AS cum_revenue,
    ROUND(
        SUM(revenue) OVER (ORDER BY revenue DESC) * 100.0
        / SUM(revenue) OVER (),
        2
    ) AS cum_pct
FROM monthly_revenue
ORDER BY revenue DESC;
```

### 示例八：与前值差值 + 累计

```sql
-- 综合应用：环比变化 + 累计 + 占比
SELECT
    department,
    to_char(month, 'YYYY-MM') AS month_str,
    revenue,
    -- 环比变化
    revenue - LAG(revenue) OVER (
        PARTITION BY department ORDER BY month
    ) AS mom_change,
    -- 累计营收
    SUM(revenue) OVER (
        PARTITION BY department ORDER BY month
    ) AS ytd_revenue,
    -- 月度占年度总额的比例
    ROUND(
        revenue * 100.0 / SUM(revenue) OVER (PARTITION BY department),
        1
    ) AS month_weight_pct
FROM monthly_revenue
ORDER BY department, month;
```

### 示例九：移动最大值与最小值

```sql
-- 3个月滑动窗口内的最高和最低营收
SELECT
    department,
    to_char(month, 'YYYY-MM') AS month_str,
    revenue,
    MAX(revenue) OVER (
        PARTITION BY department
        ORDER BY month
        ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
    ) AS rolling_max_3m,
    MIN(revenue) OVER (
        PARTITION BY department
        ORDER BY month
        ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
    ) AS rolling_min_3m,
    -- 波动范围
    MAX(revenue) OVER (
        PARTITION BY department ORDER BY month
        ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
    ) - MIN(revenue) OVER (
        PARTITION BY department ORDER BY month
        ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
    ) AS rolling_range_3m
FROM monthly_revenue
ORDER BY department, month;
```

### 示例十：年度目标完成度追踪

```sql
-- 假设每个部门年度目标为 300000，追踪完成进度
WITH targets AS (
    SELECT '销售一部' AS department, 300000 AS annual_target
    UNION ALL
    SELECT '销售二部', 280000
)
SELECT
    mr.department,
    to_char(mr.month, 'YYYY-MM') AS month_str,
    mr.revenue,
    SUM(mr.revenue) OVER (
        PARTITION BY mr.department ORDER BY mr.month
    ) AS ytd_revenue,
    t.annual_target,
    ROUND(
        SUM(mr.revenue) OVER (
            PARTITION BY mr.department ORDER BY mr.month
        ) * 100.0 / t.annual_target,
        1
    ) AS completion_pct
FROM monthly_revenue mr
JOIN targets t ON mr.department = t.department
ORDER BY mr.department, mr.month;
```

## 常见问题

### Q1：空的 OVER() 和带 ORDER BY 的 OVER 有什么区别？
**A**：`SUM(x) OVER ()` 计算整个分区的总和（每行结果相同）；`SUM(x) OVER (ORDER BY col)` 计算从分区起点到当前行的累计总和（每行结果可能不同）。这是因为 ORDER BY 会触发默认帧 `RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW`。

### Q2：移动平均在数据不足时如何处理？
**A**：当窗口帧的前几行不足指定行数时（如第 1 行没有"前 2 行"），PostgreSQL 会使用可用的行来计算。例如，第 1 行的"3 期移动平均"实际上只是它自己的值。可以使用 `COUNT(*) OVER (...)` 来检查实际参与计算的行数。

### Q3：聚合窗口函数会影响查询性能吗？
**A**：会。每个窗口函数都需要对数据进行排序和帧计算。多个使用相同 PARTITION BY 和 ORDER BY 的窗口函数可以共享排序操作，因此建议使用 `WINDOW` 子句定义可复用的窗口。不同分区/排序的窗口函数会导致多次排序。
