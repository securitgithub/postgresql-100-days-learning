## 理论讲解

### 1. PostgreSQL 的逻辑存储层次

PostgreSQL 的逻辑结构是三级层次：

```
PostgreSQL 实例 (Cluster)
  ├── Database (数据库) ─── 最顶层的逻辑隔离单位
  │     ├── Schema (模式) ─── 数据库内的命名空间
  │     │     ├── Table (表)
  │     │     ├── View (视图)
  │     │     ├── Function (函数)
  │     │     ├── Sequence (序列)
  │     │     ├── Type (类型)
  │     │     └── ...
  │     ├── Schema: public (默认)
  │     ├── Schema: pg_catalog (系统目录)
  │     └── Schema: information_schema (SQL标准信息)
  ├── Database: postgres (默认)
  ├── Database: template0 (干净模板)
  └── Database: template1 (用户模板)
```

**关键点：**
- 不同数据库之间是完全隔离的，不能直接跨库查询
- Schema 是数据库内部的命名空间，不同 Schema 中可以有同名对象
- 每个数据库默认都有 `public` Schema

### 2. 模板数据库

PostgreSQL 有两个特殊的模板数据库：

| 数据库 | 用途 | 可修改 | 可连接 |
|--------|------|--------|--------|
| template0 | 原始干净模板 | 否 | 通常否 |
| template1 | 用户自定义模板 | 是 | 是 |

当执行 `CREATE DATABASE` 时，默认从 `template1` 复制。如果你在 `template1` 中安装了扩展或创建了表，新数据库都会继承这些对象。

### 3. Schema 的作用

Schema 在实际开发中的应用场景：
- **多租户隔离**：每个租户一个 Schema，共用同一个数据库
- **功能模块分组**：如 `sales`、`inventory`、`hr` 分别存放不同模块的表
- **权限控制**：对 Schema 级别设置访问权限
- **避免命名冲突**：不同 Schema 下可以有同名的表

### 4. search_path 机制

当你访问一个表而不加 Schema 前缀时，PostgreSQL 按照 `search_path` 的顺序搜索：

```
默认 search_path = "$user", public
```

搜索顺序：
1. 与当前用户同名的 Schema（如用户名为 `alice`，先找 `alice` Schema）
2. `public` Schema
3. 如果都没找到，报错

## 代码示例

### 创建数据库

```sql
-- 最简单的创建方式
CREATE DATABASE myapp;

-- 完整语法：指定所有选项
CREATE DATABASE myapp
    OWNER = myuser             -- 所有者
    TEMPLATE = template0       -- 使用干净模板
    ENCODING = 'UTF8'          -- 字符编码
    LC_COLLATE = 'zh_CN.UTF-8' -- 排序规则
    LC_CTYPE = 'zh_CN.UTF-8'   -- 字符分类
    TABLESPACE = pg_default     -- 表空间
    CONNECTION LIMIT = 100      -- 最大连接数（-1 为无限制）
    IS_TEMPLATE = false;        -- 是否为模板数据库

-- 从 template0 创建（推荐用于指定特殊编码）
CREATE DATABASE myapp_utf8
    TEMPLATE = template0
    ENCODING = 'UTF8'
    LC_COLLATE = 'en_US.UTF-8'
    LC_CTYPE = 'en_US.UTF-8';

-- 使用 IF NOT EXISTS 避免重复创建错误
-- 注意：PostgreSQL 原生不支持 IF NOT EXISTS，需要用其他方法
-- 方法1：在 psql 中使用条件
SELECT 'CREATE DATABASE myapp'
WHERE NOT EXISTS (SELECT FROM pg_database WHERE datname = 'myapp')\gexec
```

### 管理数据库

```sql
-- 查看所有数据库
SELECT datname, datowner, encoding, datcollate, datctype,
       pg_size_pretty(pg_database_size(datname)) AS size
FROM pg_database
ORDER BY pg_database_size(datname) DESC;

-- 修改数据库属性
ALTER DATABASE myapp SET timezone = 'Asia/Shanghai';
ALTER DATABASE myapp RENAME TO myapp_v2;
ALTER DATABASE myapp_v2 OWNER TO newowner;
ALTER DATABASE myapp_v2 SET search_path TO myschema, public;
ALTER DATABASE myapp_v2 CONNECTION LIMIT 50;

-- 删除数据库
DROP DATABASE myapp_v2;

-- 安全删除（存在才删除）
DROP DATABASE IF EXISTS myapp_v2;

-- 注意：不能删除当前正在连接的数据库
-- 需要先断开所有连接
SELECT pg_terminate_backend(pid)
FROM pg_stat_activity
WHERE datname = 'myapp_v2' AND pid <> pg_backend_pid();
```

### Schema 管理

```sql
-- 创建 Schema
CREATE SCHEMA sales;
CREATE SCHEMA inventory;
CREATE SCHEMA hr;

-- 创建 Schema 并指定所有者
CREATE SCHEMA sales AUTHORIZATION sales_admin;

-- 创建 Schema 并同时创建对象
CREATE SCHEMA sales
    CREATE TABLE orders (
        id SERIAL PRIMARY KEY,
        customer_id INT,
        total NUMERIC(10,2)
    )
    CREATE VIEW recent_orders AS
        SELECT * FROM sales.orders
        WHERE id > 1000;

-- 查看所有 Schema
SELECT schema_name, schema_owner
FROM information_schema.schemata
ORDER BY schema_name;

-- 或者使用系统目录
SELECT nspname, nspowner, pg_catalog.pg_get_userbyid(nspowner) AS owner
FROM pg_catalog.pg_namespace
WHERE nspname NOT LIKE 'pg_%'
ORDER BY nspname;

-- 删除 Schema
DROP SCHEMA sales;                  -- Schema 必须为空
DROP SCHEMA sales CASCADE;          -- 级联删除 Schema 及其所有对象
DROP SCHEMA IF EXISTS sales CASCADE; -- 安全删除

-- 修改 Schema
ALTER SCHEMA sales RENAME TO sales_v2;
ALTER SCHEMA sales_v2 OWNER TO new_owner;
```

### search_path 配置

```sql
-- 查看当前 search_path
SHOW search_path;

-- 临时修改当前会话的 search_path
SET search_path TO sales, public;

-- 永久修改用户的默认 search_path
ALTER ROLE myuser SET search_path TO sales, public;

-- 永久修改数据库的默认 search_path
ALTER DATABASE myapp SET search_path TO myschema, public;

-- 验证 search_path 的效果
CREATE SCHEMA test_schema;
CREATE TABLE test_schema.users (id INT, name TEXT);
CREATE TABLE public.users (id INT, email TEXT);

-- 当 search_path 包含 test_schema 在前
SET search_path TO test_schema, public;
\d users  -- 会显示 test_schema.users 的结构

-- 明确指定 Schema
SELECT * FROM test_schema.users;
SELECT * FROM public.users;

-- 查看一个表实际属于哪个 Schema
SELECT schemaname, tablename
FROM pg_tables
WHERE tablename = 'users';
```

### 使用 information_schema 查询元信息

```sql
-- information_schema 是 SQL 标准定义的信息查询接口

-- 查看当前数据库的所有表
SELECT table_schema, table_name, table_type
FROM information_schema.tables
WHERE table_schema NOT IN ('pg_catalog', 'information_schema')
ORDER BY table_schema, table_name;

-- 查看某张表的所有列信息
SELECT column_name, data_type, character_maximum_length,
       is_nullable, column_default
FROM information_schema.columns
WHERE table_schema = 'public'
  AND table_name = 'users'
ORDER BY ordinal_position;

-- 查看所有约束
SELECT constraint_name, table_schema, table_name, constraint_type
FROM information_schema.table_constraints
WHERE table_schema = 'public';

-- 查看所有索引（使用系统目录，information_schema 不提供索引信息）
SELECT schemaname, tablename, indexname, indexdef
FROM pg_indexes
WHERE schemaname = 'public';

-- 查看数据库中各 Schema 的对象数量统计
SELECT schemaname, count(*) AS table_count
FROM pg_tables
WHERE schemaname NOT IN ('pg_catalog', 'information_schema')
GROUP BY schemaname
ORDER BY table_count DESC;

-- 查看各 Schema 的磁盘占用
SELECT schemaname,
       pg_size_pretty(SUM(pg_total_relation_size(schemaname || '.' || tablename))) AS total_size
FROM pg_tables
WHERE schemaname NOT IN ('pg_catalog', 'information_schema')
GROUP BY schemaname
ORDER BY SUM(pg_total_relation_size(schemaname || '.' || tablename)) DESC;
```

### 多租户场景示例

```sql
-- 创建租户 Schema
CREATE SCHEMA tenant_001;
CREATE SCHEMA tenant_002;

-- 为每个租户创建相同结构的表
CREATE TABLE tenant_001.products (
    id SERIAL PRIMARY KEY,
    name TEXT NOT NULL,
    price NUMERIC(10,2)
);

CREATE TABLE tenant_002.products (
    id SERIAL PRIMARY KEY,
    name TEXT NOT NULL,
    price NUMERIC(10,2)
);

-- 插入不同租户的数据
INSERT INTO tenant_001.products (name, price) VALUES ('产品A', 99.99);
INSERT INTO tenant_002.products (name, price) VALUES ('产品B', 199.99);

-- 通过切换 search_path 实现租户切换
SET search_path TO tenant_001, public;
SELECT * FROM products;  -- 返回租户1的数据

SET search_path TO tenant_002, public;
SELECT * FROM products;  -- 返回租户2的数据

-- 清理
DROP SCHEMA tenant_001 CASCADE;
DROP SCHEMA tenant_002 CASCADE;
```

## 常见问题

### Q1: 什么时候应该创建新的数据库，什么时候创建新的 Schema？

**A:**
- **创建新数据库**：当应用之间完全独立、不需要共享数据时。例如完全不同的项目。
- **创建新 Schema**：当数据在逻辑上属于同一应用但需要组织分类时。例如同一应用的不同模块，或多租户场景。
- 核心区别：跨数据库不能直接 JOIN，跨 Schema 可以。

### Q2: public Schema 可以删除吗？

**A:** 技术上可以删除，但不推荐。如果删除了 `public` Schema：
- 新建表时必须明确指定 Schema，否则报错
- 许多第三方工具默认使用 public Schema
- 可以通过 `CREATE SCHEMA public;` 重新创建
- 如果不想让用户使用 public，可以撤销其创建权限：`REVOKE CREATE ON SCHEMA public FROM PUBLIC;`

### Q3: 如何查看当前连接使用的是哪个 Schema？

**A:** PostgreSQL 没有"当前 Schema"的概念，而是使用 `search_path` 列表。创建新对象时使用 search_path 中第一个存在的 Schema：
```sql
-- 查看 search_path
SHOW search_path;

-- 查看创建对象时实际使用的 Schema
SELECT current_schema();

-- 查看完整搜索路径（展开后）
SELECT current_schemas(true);  -- true 包含隐式 Schema
SELECT current_schemas(false); -- false 仅显示显式 Schema
```
