## 理论讲解

### 1. 实现阶段概览

```
Day 096 设计 → Day 097 实现 → Day 098 优化 → Day 099 运维
                   │
                   ├── 建表建索引
                   ├── 存储过程/触发器
                   ├── 测试数据导入
                   └── 核心查询实现
```

### 2. 存储过程的设计原则

- **原子操作封装**：将需要事务保证的多步操作封装为存储过程
- **输入验证**：在存储过程内部做参数校验
- **错误处理**：使用 EXCEPTION 块捕获和处理异常
- **返回值设计**：使用 OUT 参数或 RETURNS TABLE 返回结构化结果

### 3. 触发器的使用场景

- **审计日志**：自动记录数据变更
- **数据同步**：更新冗余字段（如计数、汇总值）
- **业务规则**：强制执行复杂的业务逻辑
- **时间戳更新**：自动更新 `updated_at`

### 4. 测试数据生成策略

百万级数据生成方法：
- `generate_series` + 随机函数：最简单，纯 SQL
- `COPY` 导入 CSV：速度最快
- PL/pgSQL 批量插入：灵活性好
- 外部脚本（Python/Go）：可生成更真实的数据

```sql
-- 生成订单编号
CREATE OR REPLACE FUNCTION generate_order_no()
RETURNS TEXT AS $$
DECLARE
    seq_val BIGINT;
BEGIN
    seq_val := nextval('order_no_seq');
    RETURN 'ORD' || to_char(NOW(), 'YYYYMMDD') || lpad(seq_val::text, 8, '0');
END;
$$ LANGUAGE plpgsql;
```

### 示例2：核心业务存储过程 —— 下单

```sql
-- ============================================================
-- 下单存储过程：原子操作保证数据一致性
-- ============================================================
CREATE OR REPLACE FUNCTION create_order(
    p_user_id       BIGINT,
    p_address_id    BIGINT,
    p_items         JSONB,      -- [{"sku_id": 1, "quantity": 2}, ...]
    p_remark        TEXT DEFAULT NULL
)
RETURNS TABLE (
    order_id    BIGINT,
    order_no    TEXT,
    pay_amount  NUMERIC
)
LANGUAGE plpgsql
AS $$
DECLARE
    v_order_id      BIGINT;
    v_order_no      TEXT;
    v_total_amount  NUMERIC(12, 2) := 0;
    v_address       JSONB;
    v_item          JSONB;
    v_sku           RECORD;
    v_subtotal      NUMERIC(12, 2);
BEGIN
    -- 1. 验证用户
    IF NOT EXISTS (SELECT 1 FROM users WHERE id = p_user_id AND status = 1) THEN
        RAISE EXCEPTION '用户不存在或已被禁用' USING ERRCODE = 'P0001';
    END IF;

    -- 2. 获取地址快照
    SELECT jsonb_build_object(
        'receiver', ua.receiver,
        'phone', ua.phone,
        'province', ua.province,
        'city', ua.city,
        'district', ua.district,
        'address', ua.address
    ) INTO v_address
    FROM user_addresses ua
    WHERE ua.id = p_address_id AND ua.user_id = p_user_id;

    IF v_address IS NULL THEN
        RAISE EXCEPTION '收货地址不存在' USING ERRCODE = 'P0002';
    END IF;

    -- 3. 生成订单号
    v_order_no := generate_order_no();

    -- 4. 创建订单（先插入，后面更新金额）
    INSERT INTO orders (order_no, user_id, address_snapshot, total_amount, pay_amount, status, remark, created_at)
    VALUES (v_order_no, p_user_id, v_address, 0, 0, 0, p_remark, NOW())
    RETURNING id INTO v_order_id;

    -- 5. 处理每个订单项
    FOR v_item IN SELECT * FROM jsonb_array_elements(p_items)
    LOOP
        -- 锁定 SKU 行并检查库存
        SELECT ps.id, ps.product_id, ps.price, ps.stock, ps.reserved,
               ps.attributes, p.name AS product_name
        INTO v_sku
        FROM product_skus ps
        JOIN products p ON p.id = ps.product_id
        WHERE ps.id = (v_item->>'sku_id')::bigint
          AND ps.is_active = TRUE
        FOR UPDATE;  -- 行级锁，防止超卖

        IF v_sku IS NULL THEN
            RAISE EXCEPTION 'SKU % 不存在或已下架', v_item->>'sku_id'
            USING ERRCODE = 'P0003';
        END IF;

        -- 检查可用库存
        IF v_sku.stock - v_sku.reserved < (v_item->>'quantity')::int THEN
            RAISE EXCEPTION '商品 % 库存不足', v_sku.product_name
            USING ERRCODE = 'P0004';
        END IF;

        v_subtotal := v_sku.price * (v_item->>'quantity')::int;
        v_total_amount := v_total_amount + v_subtotal;

        -- 插入订单项
        INSERT INTO order_items (order_id, product_id, sku_id, sku_snapshot, quantity, unit_price, subtotal)
        VALUES (
            v_order_id,
            v_sku.product_id,
            v_sku.id,
            jsonb_build_object(
                'product_name', v_sku.product_name,
                'sku_attributes', v_sku.attributes,
                'price', v_sku.price
            ),
            (v_item->>'quantity')::int,
            v_sku.price,
            v_subtotal
        );

        -- 预留库存
        UPDATE product_skus
        SET reserved = reserved + (v_item->>'quantity')::int
        WHERE id = v_sku.id;
    END LOOP;

    -- 6. 更新订单金额
    UPDATE orders
    SET total_amount = v_total_amount,
        pay_amount = v_total_amount  -- 暂不考虑折扣
    WHERE id = v_order_id;

    -- 7. 记录状态日志
    INSERT INTO order_status_logs (order_id, from_status, to_status, operator, remark)
    VALUES (v_order_id, NULL, 0, '系统', '订单创建');

    -- 8. 返回结果
    RETURN QUERY SELECT v_order_id, v_order_no, v_total_amount;
END;
$$;

-- 调用示例
SELECT * FROM create_order(
    1,      -- user_id
    1,      -- address_id
    '[{"sku_id": 1, "quantity": 2}, {"sku_id": 5, "quantity": 1}]'::jsonb,
    '请尽快发货'
);
```

### 示例3：支付回调与订单状态流转

```sql
-- ============================================================
-- 支付成功回调处理
-- ============================================================
CREATE OR REPLACE FUNCTION process_payment_callback(
    p_payment_no    TEXT,
    p_third_party_no TEXT,
    p_success       BOOLEAN
)
RETURNS VOID
LANGUAGE plpgsql
AS $$
DECLARE
    v_payment   RECORD;
    v_item      RECORD;
BEGIN
    -- 1. 查找并锁定支付记录
    SELECT * INTO v_payment
    FROM payments
    WHERE payment_no = p_payment_no
    FOR UPDATE;

    IF v_payment IS NULL THEN
        RAISE EXCEPTION '支付记录不存在: %', p_payment_no;
    END IF;

    IF v_payment.status != 0 THEN
        RAISE NOTICE '支付记录已处理，跳过: %', p_payment_no;
        RETURN;  -- 幂等处理
    END IF;

    IF p_success THEN
        -- 2a. 支付成功
        UPDATE payments
        SET status = 1, third_party_no = p_third_party_no, paid_at = NOW()
        WHERE id = v_payment.id;

        -- 更新订单状态
        UPDATE orders
        SET status = 1, payment_method = v_payment.method, paid_at = NOW()
        WHERE id = v_payment.order_id AND status = 0;

        -- 扣减实际库存，释放预留
        FOR v_item IN
            SELECT sku_id, quantity FROM order_items WHERE order_id = v_payment.order_id
        LOOP
            UPDATE product_skus
            SET stock = stock - v_item.quantity,
                reserved = reserved - v_item.quantity
            WHERE id = v_item.sku_id;
        END LOOP;

        -- 更新商品销量
        UPDATE products p
        SET total_sales = total_sales + oi.total_qty
        FROM (
            SELECT product_id, SUM(quantity) AS total_qty
            FROM order_items
            WHERE order_id = v_payment.order_id
            GROUP BY product_id
        ) oi
        WHERE p.id = oi.product_id;

        -- 记录状态日志
        INSERT INTO order_status_logs (order_id, from_status, to_status, operator, remark)
        VALUES (v_payment.order_id, 0, 1, '系统', '支付成功: ' || p_third_party_no);
    ELSE
        -- 2b. 支付失败
        UPDATE payments SET status = 2 WHERE id = v_payment.id;
    END IF;
END;
$$;
```

### 示例4：订单超时自动取消触发器

```sql
-- ============================================================
-- 取消订单（释放库存）
-- ============================================================
CREATE OR REPLACE FUNCTION cancel_order(
    p_order_id  BIGINT,
    p_reason    TEXT DEFAULT '用户取消'
)
RETURNS VOID
LANGUAGE plpgsql
AS $$
DECLARE
    v_order RECORD;
    v_item  RECORD;
BEGIN
    SELECT * INTO v_order
    FROM orders WHERE id = p_order_id FOR UPDATE;

    IF v_order IS NULL THEN
        RAISE EXCEPTION '订单不存在';
    END IF;

    IF v_order.status != 0 THEN
        RAISE EXCEPTION '只有待支付订单才能取消，当前状态: %', v_order.status;
    END IF;

    -- 释放预留库存
    FOR v_item IN
        SELECT sku_id, quantity FROM order_items WHERE order_id = p_order_id
    LOOP
        UPDATE product_skus
        SET reserved = reserved - v_item.quantity
        WHERE id = v_item.sku_id;
    END LOOP;

    -- 更新订单状态
    UPDATE orders SET status = 5 WHERE id = p_order_id;

    -- 记录日志
    INSERT INTO order_status_logs (order_id, from_status, to_status, operator, remark)
    VALUES (p_order_id, 0, 5, '系统', p_reason);
END;
$$;

-- 评价触发器：自动更新商品的评价统计
CREATE OR REPLACE FUNCTION update_product_review_stats()
RETURNS TRIGGER AS $$
BEGIN
    UPDATE products
    SET review_count = (
            SELECT COUNT(*) FROM reviews WHERE product_id = NEW.product_id
        ),
        avg_rating = (
            SELECT ROUND(AVG(rating), 2) FROM reviews WHERE product_id = NEW.product_id
        )
    WHERE id = NEW.product_id;

    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trg_reviews_stats
    AFTER INSERT ON reviews
    FOR EACH ROW EXECUTE FUNCTION update_product_review_stats();
```

### 示例6：核心业务查询

```sql
-- ============================================================
-- 核心查询1：商品搜索（全文搜索 + 过滤 + 排序 + 分页）
-- ============================================================
CREATE OR REPLACE FUNCTION search_products(
    p_keyword   TEXT DEFAULT NULL,
    p_category  INTEGER DEFAULT NULL,
    p_min_price NUMERIC DEFAULT NULL,
    p_max_price NUMERIC DEFAULT NULL,
    p_sort      TEXT DEFAULT 'relevance',  -- relevance/price_asc/price_desc/sales
    p_page      INTEGER DEFAULT 1,
    p_size      INTEGER DEFAULT 20
)
RETURNS TABLE (
    id BIGINT, name TEXT, brand TEXT, base_price NUMERIC,
    total_sales INT, avg_rating NUMERIC, review_count INT
)
LANGUAGE plpgsql
AS $$
BEGIN
    RETURN QUERY
    SELECT
        p.id, p.name::text, p.brand::text, p.base_price,
        p.total_sales, p.avg_rating, p.review_count
    FROM products p
    WHERE p.status = 1
      AND (p_keyword IS NULL OR p.search_vector @@ plainto_tsquery('simple', p_keyword))
      AND (p_category IS NULL OR p.category_id = p_category)
      AND (p_min_price IS NULL OR p.base_price >= p_min_price)
      AND (p_max_price IS NULL OR p.base_price <= p_max_price)
    ORDER BY
        CASE p_sort
            WHEN 'price_asc' THEN p.base_price
        END ASC NULLS LAST,
        CASE p_sort
            WHEN 'price_desc' THEN p.base_price
        END DESC NULLS LAST,
        CASE p_sort
            WHEN 'sales' THEN p.total_sales
        END DESC NULLS LAST,
        CASE
            WHEN p_keyword IS NOT NULL
            THEN ts_rank(p.search_vector, plainto_tsquery('simple', p_keyword))
        END DESC NULLS LAST,
        p.id DESC
    LIMIT p_size OFFSET (p_page - 1) * p_size;
END;
$$;

-- 使用示例
SELECT * FROM search_products('手机', NULL, 1000, 5000, 'sales', 1, 20);

-- ============================================================
-- 核心查询2：用户订单列表
-- ============================================================
SELECT
    o.id,
    o.order_no,
    o.status,
    o.pay_amount,
    o.created_at,
    jsonb_agg(
        jsonb_build_object(
            'product_name', oi.sku_snapshot->>'product_name',
            'quantity', oi.quantity,
            'unit_price', oi.unit_price,
            'subtotal', oi.subtotal
        )
    ) AS items
FROM orders o
LEFT JOIN order_items oi ON oi.order_id = o.id
WHERE o.user_id = 12345
GROUP BY o.id, o.order_no, o.status, o.pay_amount, o.created_at
ORDER BY o.created_at DESC
LIMIT 20 OFFSET 0;

-- ============================================================
-- 核心查询3：商品详情（含SKU列表和评价摘要）
-- ============================================================
SELECT
    p.*,
    (
        SELECT jsonb_agg(
            jsonb_build_object(
                'id', s.id,
                'sku_code', s.sku_code,
                'attributes', s.attributes,
                'price', s.price,
                'stock', s.stock - s.reserved
            )
        )
        FROM product_skus s
        WHERE s.product_id = p.id AND s.is_active = TRUE
    ) AS skus,
    (
        SELECT jsonb_build_object(
            'total', COUNT(*),
            'avg_rating', ROUND(AVG(rating), 1),
            'distribution', jsonb_build_object(
                '5star', COUNT(*) FILTER (WHERE rating = 5),
                '4star', COUNT(*) FILTER (WHERE rating = 4),
                '3star', COUNT(*) FILTER (WHERE rating = 3),
                '2star', COUNT(*) FILTER (WHERE rating = 2),
                '1star', COUNT(*) FILTER (WHERE rating = 1)
            )
        )
        FROM reviews r
        WHERE r.product_id = p.id
    ) AS review_summary
FROM products p
WHERE p.id = 100;
```

## 常见问题

### Q1: 百万级数据插入很慢怎么办？

**A:** 几个关键优化：(1) 插入前临时禁用索引和触发器，插入后重建；(2) 使用 `COPY` 命令而非 INSERT；(3) 增大 `maintenance_work_mem` 和临时关闭 `synchronous_commit`；(4) 分批提交（每1万条 COMMIT 一次）而非一个大事务；(5) 使用 `UNLOGGED TABLE` 临时表加载再转入正式表。

### Q2: 存储过程中如何处理并发下单的库存超卖问题？

**A:** 示例中使用了 `SELECT ... FOR UPDATE` 对 SKU 行加排他锁，确保同一时刻只有一个事务能操作同一个 SKU 的库存。这是悲观锁方案。另一种是乐观锁：`UPDATE product_skus SET stock = stock - $1 WHERE id = $2 AND stock >= $1`，通过 WHERE 条件保证不超卖，如果 `UPDATE` 影响 0 行则说明库存不足。乐观锁在低竞争场景下性能更好。

### Q3: 订单表分区后，order_items 的外键怎么处理？

**A:** PostgreSQL 的分区表从 PG 12 开始支持被引用为外键。但在高并发场景下，外键约束会增加开销（需要检查父表）。实践中，很多电商系统选择在应用层保证一致性而不使用数据库外键，以获得更好的写入性能。如果使用外键，确保分区表的主键包含分区键。
