## 理论讲解

### 1. 窗口函数高级应用

窗口函数是 SQL 数据分析的核心利器。与聚合函数不同，窗口函数不会折叠行，而是在每一行上计算一个基于"窗口"的结果值。

窗口函数的基本语法：

```
函数名(...) OVER (
    [PARTITION BY 分区列]
    [ORDER BY 排序列]
    [ROWS/RANGE BETWEEN 窗口范围]
)
```

PostgreSQL 支持的窗口函数分类：

| 类别 | 函数 | 用途 |
|------|------|------|
| 排名函数 | ROW_NUMBER, RANK, DENSE_RANK, NTILE | 排名、分组 |
| 偏移函数 | LAG, LEAD, FIRST_VALUE, LAST_VALUE, NTH_VALUE | 访问前后行 |
| 聚合窗口 | SUM, AVG, COUNT, MIN, MAX | 滑动聚合 |
| 统计函数 | PERCENT_RANK, CUME_DIST | 百分位、累积分布 |

窗口帧（Frame）是窗口函数的关键概念：

- `ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW`：从分区开头到当前行
- `ROWS BETWEEN 3 PRECEDING AND 3 FOLLOWING`：当前行前后各3行
- `RANGE BETWEEN INTERVAL '7 days' PRECEDING AND CURRENT ROW`：时间范围窗口

### 2. GROUPING SETS / ROLLUP / CUBE

标准 GROUP BY 只能产生一种分组方式。GROUPING SETS 系列功能允许在一次查询中生成多种分组组合的结果，相当于多个 GROUP BY 查询的 UNION ALL，但效率更高。

- **GROUPING SETS**：明确指定要计算的分组组合
- **ROLLUP**：层级汇总（从细到粗），适合"省-市-区"等层级结构
- **CUBE**：所有可能的分组组合，适合多维分析

```
GROUPING SETS ((a, b), (a), ())
等价于：
  GROUP BY a, b
  UNION ALL
  GROUP BY a
  UNION ALL
  全局汇总（无 GROUP BY）
```

### 3. TABLESAMPLE 数据采样

当数据量非常大而只需要估算值时，可以使用 TABLESAMPLE 对表进行采样：

- **BERNOULLI 采样**：按行随机采样，每行有 N% 概率被选中，结果更随机
- **SYSTEM 采样**：按页（块）采样，速度更快但随机性较差

采样适用场景：数据探索、估算统计值、测试查询逻辑。

### 4. 分析查询优化

分析查询通常涉及大量数据扫描和排序，优化策略包括：

- **合理使用索引**：BRIN 索引适合时序/有序数据
- **物化视图预计算**：将常用聚合结果缓存
- **并行查询**：调大 `max_parallel_workers_per_gather`
- **分区裁剪**：确保 WHERE 条件能命中分区键
- **增量计算**：避免每次全量重算

## 代码示例

### 示例1：窗口函数实战 —— 销售数据分析

```sql
-- 创建销售数据表
CREATE TABLE sales (
    sale_id     SERIAL PRIMARY KEY,
    sale_date   DATE NOT NULL,
    region      TEXT NOT NULL,
    product     TEXT NOT NULL,
    salesperson TEXT NOT NULL,
    amount      NUMERIC(12, 2) NOT NULL,
    quantity    INTEGER NOT NULL
);

-- 插入模拟数据
INSERT INTO sales (sale_date, region, product, salesperson, amount, quantity)
SELECT
    '2025-01-01'::date + (random() * 365)::int AS sale_date,
    (ARRAY['华东', '华南', '华北', '西南', '西北'])[floor(random()*5+1)::int] AS region,
    (ARRAY['产品A', '产品B', '产品C', '产品D'])[floor(random()*4+1)::int] AS product,
    (ARRAY['张三', '李四', '王五', '赵六', '钱七', '孙八'])[floor(random()*6+1)::int],
    round((random() * 10000 + 100)::numeric, 2) AS amount,
    floor(random() * 100 + 1)::int AS quantity
FROM generate_series(1, 10000);

-- (1) 排名：每个区域销售额 TOP 3 销售人员
SELECT * FROM (
    SELECT
        region,
        salesperson,
        SUM(amount) AS total_sales,
        RANK() OVER (
            PARTITION BY region
            ORDER BY SUM(amount) DESC
        ) AS rank_in_region
    FROM sales
    GROUP BY region, salesperson
) ranked
WHERE rank_in_region <= 3
ORDER BY region, rank_in_region;

-- (2) 环比分析：每月销售额及环比增长率
WITH monthly AS (
    SELECT
        date_trunc('month', sale_date) AS month,
        SUM(amount) AS total
    FROM sales
    GROUP BY month
)
SELECT
    month,
    total,
    LAG(total) OVER (ORDER BY month) AS prev_month,
    CASE
        WHEN LAG(total) OVER (ORDER BY month) IS NOT NULL
        THEN round(
            (total - LAG(total) OVER (ORDER BY month))
            / LAG(total) OVER (ORDER BY month) * 100, 2
        )
    END AS growth_pct
FROM monthly
ORDER BY month;

-- (3) 累计求和与占比
SELECT
    sale_date,
    region,
    amount,
    SUM(amount) OVER (
        PARTITION BY region
        ORDER BY sale_date
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS cumulative_amount,
    round(
        amount / SUM(amount) OVER (PARTITION BY region) * 100, 2
    ) AS pct_of_region
FROM sales
WHERE region = '华东'
ORDER BY sale_date
LIMIT 20;
```

### 示例2：移动平均与趋势分析

```sql
-- 7日移动平均
WITH daily_sales AS (
    SELECT
        sale_date,
        SUM(amount) AS daily_total
    FROM sales
    GROUP BY sale_date
)
SELECT
    sale_date,
    daily_total,
    round(AVG(daily_total) OVER (
        ORDER BY sale_date
        ROWS BETWEEN 6 PRECEDING AND CURRENT ROW
    ), 2) AS moving_avg_7d,
    round(AVG(daily_total) OVER (
        ORDER BY sale_date
        ROWS BETWEEN 29 PRECEDING AND CURRENT ROW
    ), 2) AS moving_avg_30d
FROM daily_sales
ORDER BY sale_date;

-- 使用 NTILE 将销售人员分为4个梯队
SELECT
    salesperson,
    total_sales,
    NTILE(4) OVER (ORDER BY total_sales DESC) AS tier
FROM (
    SELECT salesperson, SUM(amount) AS total_sales
    FROM sales
    GROUP BY salesperson
) t;

-- 百分位排名
SELECT
    salesperson,
    total_sales,
    round(PERCENT_RANK() OVER (ORDER BY total_sales) * 100, 1) AS percentile,
    round(CUME_DIST() OVER (ORDER BY total_sales) * 100, 1) AS cumulative_dist
FROM (
    SELECT salesperson, SUM(amount) AS total_sales
    FROM sales
    GROUP BY salesperson
) t;
```

### 示例3：GROUPING SETS / ROLLUP / CUBE

```sql
-- GROUPING SETS：同时获得多种维度的汇总
SELECT
    COALESCE(region, '【全部区域】') AS region,
    COALESCE(product, '【全部产品】') AS product,
    SUM(amount) AS total_sales,
    COUNT(*) AS order_count,
    GROUPING(region) AS is_region_total,
    GROUPING(product) AS is_product_total
FROM sales
GROUP BY GROUPING SETS (
    (region, product),   -- 区域+产品 明细
    (region),            -- 区域小计
    (product),           -- 产品小计
    ()                   -- 总计
)
ORDER BY
    GROUPING(region), region,
    GROUPING(product), product;

-- ROLLUP：层级汇总（区域 → 产品 → 明细）
SELECT
    COALESCE(region, '合计') AS region,
    COALESCE(product, '小计') AS product,
    SUM(amount) AS total_sales,
    round(AVG(amount), 2) AS avg_sale
FROM sales
GROUP BY ROLLUP (region, product)
ORDER BY region NULLS LAST, product NULLS LAST;

-- CUBE：所有维度组合
SELECT
    region,
    product,
    date_trunc('quarter', sale_date)::date AS quarter,
    SUM(amount) AS total_sales
FROM sales
GROUP BY CUBE (region, product, date_trunc('quarter', sale_date))
HAVING SUM(amount) > 100000
ORDER BY region NULLS LAST, product NULLS LAST, quarter NULLS LAST;

-- 使用 FILTER 实现条件聚合（比 CASE WHEN 更清晰）
SELECT
    region,
    COUNT(*) AS total_orders,
    SUM(amount) AS total_sales,
    COUNT(*) FILTER (WHERE amount > 5000) AS high_value_orders,
    SUM(amount) FILTER (WHERE product = '产品A') AS product_a_sales,
    round(AVG(amount) FILTER (WHERE quantity > 50), 2) AS avg_large_order
FROM sales
GROUP BY region
ORDER BY total_sales DESC;
```

### 示例4：TABLESAMPLE 数据采样

```sql
-- BERNOULLI 采样：约10%的行
SELECT COUNT(*), AVG(amount), SUM(amount)
FROM sales TABLESAMPLE BERNOULLI(10);

-- 可重复采样（指定种子）
SELECT COUNT(*), AVG(amount)
FROM sales TABLESAMPLE BERNOULLI(10) REPEATABLE(42);

-- SYSTEM 采样：按页面采样，速度更快
SELECT COUNT(*), AVG(amount)
FROM sales TABLESAMPLE SYSTEM(5);

-- 采样精度对比测试
DO $$
DECLARE
    full_avg NUMERIC;
    sample_avg NUMERIC;
BEGIN
    SELECT AVG(amount) INTO full_avg FROM sales;
    SELECT AVG(amount) INTO sample_avg
    FROM sales TABLESAMPLE BERNOULLI(10) REPEATABLE(1);

    RAISE NOTICE '全量平均: %, 采样平均: %, 误差: %',
        round(full_avg, 2),
        round(sample_avg, 2),
        round(abs(full_avg - sample_avg) / full_avg * 100, 2) || '%';
END $$;

-- 结合 TABLESAMPLE 做数据探索
SELECT
    product,
    COUNT(*) AS sample_count,
    round(AVG(amount), 2) AS avg_amount,
    round(STDDEV(amount), 2) AS stddev_amount
FROM sales TABLESAMPLE BERNOULLI(20) REPEATABLE(123)
GROUP BY product
ORDER BY avg_amount DESC;
```

### 示例5：分析查询优化

```sql
-- 为分析查询创建合适的索引
CREATE INDEX idx_sales_date ON sales (sale_date);
CREATE INDEX idx_sales_region_date ON sales (region, sale_date);

-- 使用 BRIN 索引（适合有序数据，占用空间极小）
CREATE INDEX idx_sales_date_brin ON sales USING BRIN (sale_date);

-- 创建物化视图缓存常用聚合
CREATE MATERIALIZED VIEW mv_monthly_sales AS
SELECT
    date_trunc('month', sale_date) AS month,
    region,
    product,
    SUM(amount) AS total_sales,
    COUNT(*) AS order_count,
    AVG(amount) AS avg_amount
FROM sales
GROUP BY month, region, product;

CREATE UNIQUE INDEX ON mv_monthly_sales (month, region, product);

-- 刷新物化视图
REFRESH MATERIALIZED VIEW CONCURRENTLY mv_monthly_sales;

-- 分析查询优化前后对比
EXPLAIN (ANALYZE, BUFFERS, FORMAT TEXT)
SELECT region, SUM(amount)
FROM sales
WHERE sale_date BETWEEN '2025-01-01' AND '2025-06-30'
GROUP BY region;

-- 调整并行查询参数
SET max_parallel_workers_per_gather = 4;
SET parallel_tuple_cost = 0.001;
SET parallel_setup_cost = 100;

EXPLAIN (ANALYZE)
SELECT product, region, SUM(amount), COUNT(*)
FROM sales
GROUP BY product, region;
```

## 常见问题

### Q1: 窗口函数中 ROWS 和 RANGE 有什么区别？

**A:** `ROWS` 按物理行数定义窗口帧，`RANGE` 按值的逻辑范围定义。例如 `ROWS BETWEEN 3 PRECEDING AND CURRENT ROW` 固定包含当前行和前3行（共4行）；而 `RANGE BETWEEN 3 PRECEDING AND CURRENT ROW` 则包含与当前行的 ORDER BY 列值差不超过3的所有行（行数不固定）。当 ORDER BY 列有重复值时，RANGE 会将相同值的行视为同一位置。默认情况下，如果指定了 ORDER BY 但没有指定帧，PostgreSQL 使用 `RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW`。

### Q2: GROUPING SETS 与多次 GROUP BY 加 UNION ALL 相比有什么优势？

**A:** 性能上，GROUPING SETS 只需扫描一次表，而多个 UNION ALL 需要扫描多次。功能上，GROUPING() 函数能标识每行属于哪个分组层级，方便在应用层处理。此外 GROUPING SETS 的查询也更简洁易维护。在数据量大的情况下，性能差距可能达到数倍。

### Q3: TABLESAMPLE 的结果每次执行都不同吗？

**A:** 默认情况下是的，每次执行的采样结果不同。如果需要可重复的结果，可以使用 `REPEATABLE(seed)` 子句指定随机种子，相同种子在数据不变的情况下会返回相同的结果。注意 SYSTEM 采样因为按页面采样，在数据量较小时可能采样偏差较大。
