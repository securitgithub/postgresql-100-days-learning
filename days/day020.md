### 电商数据库设计要点

一个基础的电商系统至少需要以下核心表：

1. **用户表（users）**：存储用户基本信息
2. **商品表（products）**：存储商品信息和库存
3. **订单表（orders）**：存储订单主信息（谁买的、什么时候、总金额）
4. **订单详情表（order_items）**：存储每个订单包含的商品明细

表之间的关系：
- 用户 1:N 订单（一个用户可以有多个订单）
- 订单 1:N 订单详情（一个订单可以包含多个商品）
- 商品 1:N 订单详情（一个商品可以出现在多个订单中）

## 代码示例

### 一、数据库设计与建表

```sql
-- ============================================
-- 电商数据库 DDL
-- ============================================

-- 1. 用户表
CREATE TABLE users (
    id          SERIAL PRIMARY KEY,
    username    VARCHAR(50) NOT NULL UNIQUE,
    email       VARCHAR(100) NOT NULL UNIQUE,
    phone       VARCHAR(20),
    password    VARCHAR(128) NOT NULL,       -- 实际应存储哈希值
    real_name   VARCHAR(50),
    city        VARCHAR(50),
    status      VARCHAR(20) DEFAULT 'active' CHECK (status IN ('active', 'inactive', 'banned')),
    created_at  TIMESTAMP DEFAULT now(),
    updated_at  TIMESTAMP DEFAULT now()
);

-- 2. 商品分类表
CREATE TABLE categories (
    id    SERIAL PRIMARY KEY,
    name  VARCHAR(50) NOT NULL UNIQUE
);

-- 3. 商品表
CREATE TABLE products (
    id           SERIAL PRIMARY KEY,
    name         VARCHAR(200) NOT NULL,
    category_id  INT REFERENCES categories(id),
    price        NUMERIC(10,2) NOT NULL CHECK (price >= 0),
    stock        INT NOT NULL DEFAULT 0 CHECK (stock >= 0),
    description  TEXT,
    status       VARCHAR(20) DEFAULT 'on_sale' CHECK (status IN ('on_sale', 'off_sale', 'deleted')),
    created_at   TIMESTAMP DEFAULT now(),
    updated_at   TIMESTAMP DEFAULT now()
);

-- 4. 订单表
CREATE TABLE orders (
    id            SERIAL PRIMARY KEY,
    user_id       INT NOT NULL REFERENCES users(id),
    order_no      VARCHAR(32) NOT NULL UNIQUE,           -- 订单编号
    total_amount  NUMERIC(12,2) NOT NULL DEFAULT 0,
    status        VARCHAR(20) DEFAULT 'pending'
                  CHECK (status IN ('pending', 'paid', 'shipped', 'completed', 'cancelled')),
    address       TEXT,
    created_at    TIMESTAMP DEFAULT now(),
    updated_at    TIMESTAMP DEFAULT now()
);

-- 5. 订单详情表
CREATE TABLE order_items (
    id          SERIAL PRIMARY KEY,
    order_id    INT NOT NULL REFERENCES orders(id),
    product_id  INT NOT NULL REFERENCES products(id),
    quantity    INT NOT NULL CHECK (quantity > 0),
    unit_price  NUMERIC(10,2) NOT NULL,                  -- 下单时的价格快照
    UNIQUE(order_id, product_id)                          -- 同一订单中同一商品只出现一次
);

-- 创建常用索引
CREATE INDEX idx_orders_user_id ON orders (user_id);
CREATE INDEX idx_orders_status ON orders (status);
CREATE INDEX idx_orders_created ON orders (created_at DESC);
CREATE INDEX idx_order_items_order_id ON order_items (order_id);
CREATE INDEX idx_order_items_product_id ON order_items (product_id);
CREATE INDEX idx_products_category_id ON products (category_id);
CREATE INDEX idx_products_status ON products (status);
```

### 三、综合查询练习

#### 查询 1：基本查询——查看所有在售商品及其分类

```sql
SELECT
    p.id,
    p.name AS product_name,
    c.name AS category,
    p.price,
    p.stock
FROM products p
JOIN categories c ON p.category_id = c.id
WHERE p.status = 'on_sale'
ORDER BY c.name, p.price;
```

#### 查询 2：聚合查询——每个分类的商品数量和平均价格

```sql
SELECT
    c.name AS category,
    COUNT(*) AS product_count,
    ROUND(AVG(p.price), 2) AS avg_price,
    MIN(p.price) AS min_price,
    MAX(p.price) AS max_price
FROM products p
JOIN categories c ON p.category_id = c.id
WHERE p.status = 'on_sale'
GROUP BY c.name
ORDER BY product_count DESC;
```

#### 查询 3：JOIN 查询——订单详情报表

```sql
SELECT
    o.order_no,
    u.real_name AS customer,
    u.city,
    o.status AS order_status,
    p.name AS product,
    oi.quantity,
    oi.unit_price,
    oi.quantity * oi.unit_price AS subtotal,
    o.created_at::DATE AS order_date
FROM orders o
JOIN users u ON o.user_id = u.id
JOIN order_items oi ON o.id = oi.order_id
JOIN products p ON oi.product_id = p.id
ORDER BY o.created_at DESC, o.id;
```

#### 查询 4：子查询——消费金额最高的用户

```sql
SELECT u.real_name, u.city, user_stats.total_spent, user_stats.order_count
FROM users u
JOIN (
    SELECT
        user_id,
        SUM(total_amount) AS total_spent,
        COUNT(*) AS order_count
    FROM orders
    WHERE status != 'cancelled'
    GROUP BY user_id
) AS user_stats ON u.id = user_stats.user_id
ORDER BY user_stats.total_spent DESC;
```

#### 查询 5：LEFT JOIN——查找从未下单的用户

```sql
SELECT u.real_name, u.email, u.city, u.created_at
FROM users u
LEFT JOIN orders o ON u.id = o.user_id
WHERE o.id IS NULL;
```

#### 查询 6：EXISTS——查找购买过书籍的用户

```sql
SELECT u.real_name, u.email
FROM users u
WHERE EXISTS (
    SELECT 1
    FROM orders o
    JOIN order_items oi ON o.id = oi.order_id
    JOIN products p ON oi.product_id = p.id
    JOIN categories c ON p.category_id = c.id
    WHERE o.user_id = u.id
      AND o.status != 'cancelled'
      AND c.name = '书籍'
);
```

#### 查询 7：聚合 + HAVING——月销售额统计

```sql
SELECT
    TO_CHAR(o.created_at, 'YYYY-MM') AS month,
    COUNT(DISTINCT o.id) AS order_count,
    SUM(oi.quantity) AS items_sold,
    SUM(oi.quantity * oi.unit_price) AS revenue
FROM orders o
JOIN order_items oi ON o.id = oi.order_id
WHERE o.status != 'cancelled'
GROUP BY TO_CHAR(o.created_at, 'YYYY-MM')
HAVING SUM(oi.quantity * oi.unit_price) > 0
ORDER BY month;
```

#### 查询 8：排序与分页——商品列表分页

```sql
-- 第 1 页，每页 5 条
SELECT p.name, c.name AS category, p.price, p.stock
FROM products p
JOIN categories c ON p.category_id = c.id
WHERE p.status = 'on_sale'
ORDER BY p.created_at DESC
LIMIT 5 OFFSET 0;

-- 第 2 页
SELECT p.name, c.name AS category, p.price, p.stock
FROM products p
JOIN categories c ON p.category_id = c.id
WHERE p.status = 'on_sale'
ORDER BY p.created_at DESC
LIMIT 5 OFFSET 5;
```

#### 查询 9：UPDATE 综合——批量更新订单状态

```sql
-- 将超过 7 天未支付的 pending 订单改为 cancelled
BEGIN;

UPDATE orders
SET status = 'cancelled',
    updated_at = now()
WHERE status = 'pending'
  AND created_at < now() - INTERVAL '7 days'
RETURNING id, order_no, status;

COMMIT;
```

#### 查询 10：综合业务查询——商品销售排行榜

```sql
SELECT
    p.name AS product_name,
    c.name AS category,
    p.price,
    COALESCE(sales.total_qty, 0) AS total_sold,
    COALESCE(sales.total_revenue, 0) AS total_revenue,
    p.stock AS remaining_stock
FROM products p
JOIN categories c ON p.category_id = c.id
LEFT JOIN (
    SELECT
        oi.product_id,
        SUM(oi.quantity) AS total_qty,
        SUM(oi.quantity * oi.unit_price) AS total_revenue
    FROM order_items oi
    JOIN orders o ON oi.order_id = o.id
    WHERE o.status != 'cancelled'
    GROUP BY oi.product_id
) AS sales ON p.id = sales.product_id
WHERE p.status = 'on_sale'
ORDER BY total_revenue DESC;
```

## 常见问题

### Q1: 为什么订单详情表要单独存储 unit_price？

**A**：因为商品价格可能随时调整（促销、涨价），但已成交的订单应该保留**下单时的价格**。如果直接关联商品表的当前价格，历史订单的金额就会发生变化，导致财务数据不准确。这就是"价格快照"的设计思路。

### Q2: 订单表的 total_amount 是冗余数据吗？

**A**：从规范化角度看，`total_amount` 可以通过 `order_items` 计算得出，是冗余的。但在实际业务中，订单总金额涉及折扣、优惠券、运费等复杂逻辑，由应用层计算后存入。且查询订单列表时不需要每次 JOIN 明细表求和，冗余存储可以提升查询性能。这是一种常见的**反规范化**设计。
