## 理论讲解

### 1. 扩展机制概述

PostgreSQL 扩展是一种打包和分发数据库功能的标准化方式。一个扩展通常包含：
- SQL 对象定义（函数、类型、运算符、表等）
- C 共享库（.so 文件）
- 控制文件（.control）
- SQL 脚本文件（安装和升级脚本）

扩展的优势：
- **原子安装/卸载**：所有对象作为一个整体管理
- **版本管理**：支持升级路径
- **依赖管理**：可声明对其他扩展的依赖
- **命名空间隔离**：扩展对象归属于扩展

扩展分类：
- **contrib 扩展**：随 PostgreSQL 源码一起发布（如 pg_stat_statements、pgcrypto）
- **第三方扩展**：社区开发（如 PostGIS、TimescaleDB、pg_repack）
- **内置扩展**：不需要安装即可使用（如 plpgsql）

### 2. 扩展的安装要求

- `shared_preload_libraries`：某些扩展需要预加载共享库（需重启）
- `CREATE` 权限：需要在目标数据库中有创建权限
- 系统级安装：需要先在操作系统级别安装扩展包

### 3. 常用扩展概览

| 扩展 | 用途 | 预加载要求 |
|------|------|-----------|
| pg_stat_statements | SQL 执行统计 | 需要 |
| pgcrypto | 加密函数 | 不需要 |
| uuid-ossp | UUID 生成 | 不需要 |
| pg_trgm | 三元组匹配/模糊搜索 | 不需要 |
| hstore | 键值对存储 | 不需要 |
| pg_repack | 在线表重组 | 不需要 |
| postgis | 地理空间数据 | 不需要 |
| btree_gist | GiST 索引支持更多类型 | 不需要 |
| btree_gin | GIN 索引支持更多类型 | 不需要 |
| tablefunc | 交叉表等表函数 | 不需要 |
| citext | 大小写不敏感的文本类型 | 不需要 |

### 4. pg_trgm 三元组模糊搜索

pg_trgm 模块提供基于三元组（trigram）的文本相似度匹配功能。

三元组是将字符串拆分为三个连续字符的组合。例如 "hello" 的三元组：
```
"  h", " he", "hel", "ell", "llo", "lo "
```

通过比较两个字符串的三元组集合的重叠程度来计算相似度。

pg_trgm 的核心能力：
- `similarity()` 函数：计算两个字符串的相似度（0-1）
- `%` 运算符：相似度匹配
- `<->` 运算符：距离运算（1 - similarity）
- GIN/GiST 索引支持 `LIKE`、`ILIKE`、`~`（正则）查询加速

## 代码示例

### 示例1：扩展管理基本操作

```sql
-- 查看系统中可用的扩展
SELECT name, default_version, comment
FROM pg_available_extensions
ORDER BY name;

-- 查看当前数据库已安装的扩展
SELECT extname, extversion, extnamespace::regnamespace
FROM pg_extension;

-- 查看扩展的可用版本
SELECT * FROM pg_available_extension_versions
WHERE name = 'pg_trgm';

-- 安装扩展
CREATE EXTENSION IF NOT EXISTS pg_trgm;

-- 安装到指定 schema
CREATE EXTENSION pgcrypto SCHEMA public;

-- 安装指定版本
CREATE EXTENSION pg_trgm VERSION '1.6';

-- 升级扩展
ALTER EXTENSION pg_trgm UPDATE TO '1.7';

-- 查看扩展包含的对象
SELECT pg_describe_object(classid, objid, objsubid) AS object_description
FROM pg_depend
WHERE refclassid = 'pg_extension'::regclass
  AND refobjid = (SELECT oid FROM pg_extension WHERE extname = 'pg_trgm')
  AND deptype = 'e'
ORDER BY object_description;

-- 卸载扩展
DROP EXTENSION pg_trgm;

-- 卸载扩展及其依赖对象
DROP EXTENSION pg_trgm CASCADE;
```

### 示例2：pgcrypto 加密扩展

```sql
-- 安装 pgcrypto
CREATE EXTENSION IF NOT EXISTS pgcrypto;

-- 生成随机字节
SELECT gen_random_bytes(16);

-- 生成 UUID（PG 13+ 内置 gen_random_uuid()）
SELECT gen_random_uuid();

-- 密码哈希（使用 bcrypt）
SELECT crypt('my_password', gen_salt('bf'));
-- 结果: $2a$06$abc...

-- 验证密码
SELECT (crypt('my_password', stored_hash) = stored_hash) AS password_valid;

-- 实际应用：用户注册
CREATE TABLE users (
    id serial PRIMARY KEY,
    username varchar(50) UNIQUE NOT NULL,
    password_hash text NOT NULL,
    created_at timestamptz DEFAULT now()
);

INSERT INTO users (username, password_hash)
VALUES ('alice', crypt('secure_password_123', gen_salt('bf', 10)));

-- 用户登录验证
SELECT id, username
FROM users
WHERE username = 'alice'
  AND password_hash = crypt('secure_password_123', password_hash);

-- 对称加密（AES）
SELECT encode(
    encrypt('sensitive data'::bytea, 'encryption_key'::bytea, 'aes'),
    'hex'
) AS encrypted;

-- 解密
SELECT convert_from(
    decrypt(
        decode('加密后的hex', 'hex'),
        'encryption_key'::bytea,
        'aes'
    ),
    'UTF8'
) AS decrypted;

-- 数据摘要（SHA-256）
SELECT encode(digest('hello world', 'sha256'), 'hex') AS sha256_hash;

-- HMAC
SELECT encode(hmac('message', 'secret_key', 'sha256'), 'hex') AS hmac_value;
```

### 示例3：uuid-ossp 扩展

```sql
-- 安装 uuid-ossp
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- Version 1: 基于时间和 MAC 地址
SELECT uuid_generate_v1();

-- Version 1mc: 随机 MAC 地址
SELECT uuid_generate_v1mc();

-- Version 4: 完全随机（最常用）
SELECT uuid_generate_v4();

-- Version 5: 基于命名空间和名称的确定性 UUID
SELECT uuid_generate_v5(uuid_ns_url(), 'https://example.com');

-- 在表中使用 UUID 作为主键
CREATE TABLE articles (
    id uuid DEFAULT uuid_generate_v4() PRIMARY KEY,
    title text NOT NULL,
    content text,
    created_at timestamptz DEFAULT now()
);

INSERT INTO articles (title, content) VALUES ('Test', 'Content');
SELECT * FROM articles;

-- PostgreSQL 13+ 内置函数，无需扩展
-- SELECT gen_random_uuid();
```

### 示例4：pg_trgm 基础使用

```sql
-- 安装 pg_trgm
CREATE EXTENSION IF NOT EXISTS pg_trgm;

-- 查看字符串的三元组
SELECT show_trgm('PostgreSQL');

-- 计算两个字符串的相似度
SELECT similarity('PostgreSQL', 'PostgrSQL');   -- 拼写错误
SELECT similarity('PostgreSQL', 'MySQL');        -- 完全不同

-- 单词相似度
SELECT word_similarity('SQL', 'PostgreSQL');

-- 严格单词相似度（PG 13+）
SELECT strict_word_similarity('SQL', 'PostgreSQL');

-- 设置相似度阈值
SHOW pg_trgm.similarity_threshold;   -- 默认 0.3
SET pg_trgm.similarity_threshold = 0.2;  -- 降低阈值以匹配更多结果
```

### 示例5：pg_trgm 模糊搜索实战

```sql
-- 创建测试数据
CREATE TABLE products (
    id serial PRIMARY KEY,
    name text NOT NULL,
    category text,
    description text
);

INSERT INTO products (name, category, description) VALUES
('iPhone 15 Pro', '手机', 'Apple 最新旗舰手机'),
('iPhone 15', '手机', 'Apple 标准版手机'),
('Samsung Galaxy S24', '手机', '三星旗舰手机'),
('MacBook Pro 16', '笔记本', 'Apple 专业笔记本电脑'),
('ThinkPad X1 Carbon', '笔记本', '联想商务笔记本'),
('iPad Pro 12.9', '平板', 'Apple 专业平板电脑'),
('AirPods Pro 2', '耳机', 'Apple 降噪耳机'),
('Sony WH-1000XM5', '耳机', '索尼降噪头戴耳机'),
('Dell XPS 15', '笔记本', '戴尔轻薄笔记本'),
('Pixel 8 Pro', '手机', 'Google 旗舰手机');

-- 创建 GIN 三元组索引
CREATE INDEX idx_products_name_trgm ON products USING gin (name gin_trgm_ops);
CREATE INDEX idx_products_desc_trgm ON products USING gin (description gin_trgm_ops);

-- 模糊搜索：LIKE 查询（可使用索引！）
EXPLAIN ANALYZE
SELECT * FROM products WHERE name LIKE '%iPhone%';

-- 模糊搜索：ILIKE（大小写不敏感）
SELECT * FROM products WHERE name ILIKE '%iphone%';

-- 相似度匹配
SELECT name, similarity(name, 'iPone') AS sim
FROM products
WHERE name % 'iPone'   -- 使用 % 运算符
ORDER BY sim DESC;

-- 带距离的排序（越小越相似）
SELECT name, name <-> 'MacBook' AS distance
FROM products
ORDER BY name <-> 'MacBook'
LIMIT 5;

-- 正则匹配也可使用索引
EXPLAIN ANALYZE
SELECT * FROM products WHERE name ~ 'i[Pp]hone';
```

### 示例6：pg_trgm 用于搜索建议

```sql
-- 搜索建议（自动补全）
CREATE OR REPLACE FUNCTION search_suggest(search_term text, max_results int DEFAULT 5)
RETURNS TABLE(product_name text, relevance real) AS $$
BEGIN
    RETURN QUERY
    SELECT p.name, similarity(p.name, search_term) AS sim
    FROM products p
    WHERE p.name % search_term
       OR p.name ILIKE '%' || search_term || '%'
    ORDER BY similarity(p.name, search_term) DESC
    LIMIT max_results;
END;
$$ LANGUAGE plpgsql;

-- 使用搜索建议
SELECT * FROM search_suggest('iPhon');
SELECT * FROM search_suggest('笔记本');

-- 多字段联合搜索
SELECT name, category,
       greatest(
           similarity(name, 'Apple 手机'),
           similarity(description, 'Apple 手机')
       ) AS best_match
FROM products
WHERE name % 'Apple 手机'
   OR description % 'Apple 手机'
ORDER BY best_match DESC;
```

### 示例7：GiST vs GIN 索引选择

```sql
-- GIN 索引：查询快，更新慢，占用空间大
CREATE INDEX idx_gin ON products USING gin (name gin_trgm_ops);

-- GiST 索引：查询较慢，更新快，占用空间小
CREATE INDEX idx_gist ON products USING gist (name gist_trgm_ops);

-- GiST 支持 KNN 距离排序（<-> 运算符）
-- GIN 不支持 <-> 排序

-- 推荐：
-- 读多写少 -> GIN
-- 写多且需要 KNN 排序 -> GiST

-- 对比查询计划
EXPLAIN ANALYZE SELECT * FROM products WHERE name LIKE '%Pro%';
EXPLAIN ANALYZE SELECT * FROM products ORDER BY name <-> 'Pro' LIMIT 5;
```

## 常见问题

### Q1：为什么 CREATE EXTENSION 报错 "could not open extension control file"？

这说明扩展的系统级包还没有安装。对于 contrib 扩展，需要安装 `postgresql-contrib` 包（Debian/Ubuntu: `apt install postgresql-16-contrib`，RHEL: `yum install postgresql16-contrib`）。第三方扩展需要从源码编译或使用对应的系统包。

### Q2：pg_trgm 对中文的支持如何？

pg_trgm 对中文支持有限，因为三元组是基于字符级别的。中文单字符信息量大于拉丁字符，相似度计算可能不够准确。对于中文全文搜索，推荐使用 `zhparser`（基于 SCWS 分词）或 `pg_jieba`（基于结巴分词）扩展配合 PostgreSQL 的全文搜索功能。

### Q3：shared_preload_libraries 添加新扩展需要重启吗？

是的，`shared_preload_libraries` 的变更必须重启 PostgreSQL 才能生效。这是因为这些库需要在服务启动时加载到共享内存中。需要预加载的常见扩展包括 `pg_stat_statements`、`auto_explain`、`pg_cron` 等。
