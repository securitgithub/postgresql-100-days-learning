# Day 071: 全文搜索（二）— 排名、高亮与中文搜索

## 学习目标

- 掌握 `ts_rank` 和 `ts_rank_cd` 函数对搜索结果进行相关性排名
- 学会使用 `ts_headline` 对匹配文本进行高亮显示
- 理解并创建自定义文本搜索配置（Text Search Configuration）
- 了解中文全文搜索扩展 zhparser 和 pg_jieba 的安装与使用
- 能够构建一个完整的中文全文搜索方案

## 理论讲解

### 1. 搜索结果排名

全文搜索不仅需要找到匹配的文档，还需要按相关性排序。PostgreSQL 提供了两个排名函数：

**ts_rank(vector, query [, normalization])**

`ts_rank` 基于词频（TF）来计算相关性分数。它统计查询词在文档中出现的次数，出现越多分数越高。

normalization 参数用于控制归一化方式：
- `0`（默认）：不归一化
- `1`：除以文档长度的对数
- `2`：除以文档长度
- `4`：除以两个范围之间的平均调和距离
- `8`：除以文档中唯一词的数量
- `16`：除以唯一词数量的对数
- `32`：除以排名本身加1

**ts_rank_cd(vector, query [, normalization])**

`ts_rank_cd` 基于覆盖密度（Cover Density）排名。它不仅考虑词频，还考虑匹配词之间的距离——匹配词越靠近，分数越高。这通常能提供更好的排名效果。

### 2. 搜索结果高亮

`ts_headline` 函数返回带有高亮标记的文本片段，让用户快速看到匹配的上下文。

函数签名：
```
ts_headline([config,] document, query [, options])
```

常用选项：
- `StartSel`/`StopSel`：高亮标记，默认 `<b>` 和 `</b>`
- `MaxWords`/`MinWords`：片段的最大/最小词数
- `ShortWord`：忽略短于此长度的词
- `MaxFragments`：最大片段数，0 表示使用整个文档
- `FragmentDelimiter`：片段分隔符

### 3. 自定义文本搜索配置

PostgreSQL 的文本搜索配置（Text Search Configuration）定义了文本处理的完整流程：
1. **Parser（解析器）**：将文本拆分为 token
2. **Dictionary（字典）**：将 token 转换为 lexeme（词素）
3. **Template（模板）**：字典的实现模板

通过自定义配置，可以针对不同场景（如技术文档、新闻、产品描述）优化搜索行为。

### 4. 中文全文搜索

PostgreSQL 内置的解析器是面向西文的，使用空格分词。对于中文，需要专门的分词扩展：

**zhparser**：基于 SCWS（Simple Chinese Word Segmentation）的中文分词解析器，成熟稳定。

**pg_jieba**：基于结巴分词（jieba）的中文分词扩展，支持用户自定义词典，分词效果更好。

两者的工作流程类似：
1. 安装扩展
2. 创建基于中文解析器的文本搜索配置
3. 使用该配置进行 `to_tsvector` 和 `to_tsquery`

## 代码示例

### 示例1：ts_rank 基本排名

```sql
-- 创建测试表
CREATE TABLE articles (
    id SERIAL PRIMARY KEY,
    title TEXT NOT NULL,
    content TEXT NOT NULL,
    content_tsv tsvector
);

-- 插入测试数据
INSERT INTO articles (title, content) VALUES
('PostgreSQL Full Text Search', 'PostgreSQL provides powerful full text search capabilities. Full text search examines natural language documents to find matches.'),
('Database Indexing', 'Database indexing is crucial for performance. PostgreSQL supports multiple index types including B-tree, Hash, GiST, and GIN.'),
('PostgreSQL Performance', 'PostgreSQL performance tuning involves query optimization, indexing strategies, and configuration. PostgreSQL is a powerful database system.'),
('Search Engine Architecture', 'Search engines use inverted indexes for full text search. Full text search is the core of any search engine system.');

-- 更新 tsvector 列
UPDATE articles SET content_tsv = to_tsvector('english', content);

-- 使用 ts_rank 排名
SELECT
    id,
    title,
    ts_rank(content_tsv, to_tsquery('english', 'full & text & search')) AS rank
FROM articles
WHERE content_tsv @@ to_tsquery('english', 'full & text & search')
ORDER BY rank DESC;

-- 使用归一化参数（除以文档长度）
SELECT
    id,
    title,
    ts_rank(content_tsv, to_tsquery('english', 'postgresql'), 2) AS rank_normalized
FROM articles
WHERE content_tsv @@ to_tsquery('english', 'postgresql')
ORDER BY rank_normalized DESC;
```

### 示例2：ts_rank_cd 覆盖密度排名

```sql
-- 对比 ts_rank 和 ts_rank_cd
SELECT
    id,
    title,
    ts_rank(content_tsv, q) AS rank_freq,
    ts_rank_cd(content_tsv, q) AS rank_cd
FROM
    articles,
    to_tsquery('english', 'postgresql & performance') AS q
WHERE content_tsv @@ q
ORDER BY rank_cd DESC;

-- ts_rank_cd 在词距离近时给出更高分数
-- 例如 "PostgreSQL performance" 紧挨在一起的文档会排名更高
```

### 示例3：ts_headline 高亮显示

```sql
-- 基本高亮
SELECT
    id,
    title,
    ts_headline('english', content,
                to_tsquery('english', 'full & text & search')) AS highlighted
FROM articles
WHERE content_tsv @@ to_tsquery('english', 'full & text & search');

-- 自定义高亮标记（用于 HTML 页面）
SELECT
    id,
    title,
    ts_headline('english', content,
                to_tsquery('english', 'postgresql'),
                'StartSel=<mark>, StopSel=</mark>, MaxWords=30, MinWords=15')
        AS highlighted
FROM articles
WHERE content_tsv @@ to_tsquery('english', 'postgresql');

-- 多片段高亮
SELECT
    id,
    title,
    ts_headline('english', content,
                to_tsquery('english', 'search | indexing'),
                'MaxFragments=3, FragmentDelimiter= ... , StartSel=**, StopSel=**')
        AS highlighted
FROM articles
WHERE content_tsv @@ to_tsquery('english', 'search | indexing');
```

### 示例4：自定义文本搜索配置

```sql
-- 查看当前可用的配置
SELECT cfgname FROM pg_ts_config;

-- 查看某个配置的详细映射
SELECT
    m.mapdict::regdictionary,
    t.alias AS token_type
FROM pg_ts_config_map m
JOIN pg_ts_parser p ON true
JOIN (SELECT * FROM ts_token_type('default')) t ON t.tokid = m.maptokentype
WHERE m.mapcfg = 'english'::regconfig
ORDER BY t.alias;

-- 创建自定义配置（基于 english 配置的副本）
CREATE TEXT SEARCH CONFIGURATION my_english (COPY = english);

-- 为自定义配置添加同义词字典
-- 首先创建同义词字典文件 /usr/share/postgresql/tsearch_data/my_synonyms.syn
-- 内容示例：pg postgresql
--          db database

-- 创建同义词字典
CREATE TEXT SEARCH DICTIONARY my_synonyms (
    TEMPLATE = synonym,
    SYNONYMS = my_synonyms
);

-- 将同义词字典添加到配置中（在默认字典之前）
ALTER TEXT SEARCH CONFIGURATION my_english
    ALTER MAPPING FOR asciiword
    WITH my_synonyms, english_stem;

-- 测试自定义配置
SELECT to_tsvector('my_english', 'pg is a great db system');
-- 应该将 pg 映射为 postgresql，db 映射为 database
```

### 示例5：中文搜索配置（zhparser）

```sql
-- 安装 zhparser 扩展（需要先编译安装）
CREATE EXTENSION zhparser;

-- 创建中文文本搜索配置
CREATE TEXT SEARCH CONFIGURATION chinese_zh (PARSER = zhparser);

-- 添加 token 映射
-- zhparser 的 token 类型：n(名词) v(动词) a(形容词) 等
ALTER TEXT SEARCH CONFIGURATION chinese_zh
    ADD MAPPING FOR n, v, a, i, e, l
    WITH simple;

-- 使用中文搜索
SELECT to_tsvector('chinese_zh', '数据库全文搜索是一项强大的功能');
SELECT to_tsquery('chinese_zh', '数据库搜索');

-- 创建中文全文搜索索引
CREATE TABLE zh_articles (
    id SERIAL PRIMARY KEY,
    title TEXT,
    content TEXT
);

INSERT INTO zh_articles (title, content) VALUES
('PostgreSQL全文搜索', '本文介绍PostgreSQL数据库的全文搜索功能，包括中文分词技术。'),
('数据库性能优化', '数据库性能优化包括查询优化、索引设计和配置调优。');

-- 创建 GIN 索引
CREATE INDEX idx_zh_articles_content
    ON zh_articles USING gin(to_tsvector('chinese_zh', content));

-- 搜索
SELECT id, title,
    ts_headline('chinese_zh', content,
                to_tsquery('chinese_zh', '数据库'),
                'StartSel=【, StopSel=】') AS highlighted
FROM zh_articles
WHERE to_tsvector('chinese_zh', content) @@ to_tsquery('chinese_zh', '数据库');
```

### 示例7：综合排名与高亮应用

```sql
-- 综合示例：构建一个搜索 API 的查询
WITH search_query AS (
    SELECT to_tsquery('english', 'postgresql & search') AS q
)
SELECT
    a.id,
    a.title,
    -- 排名分数
    ts_rank_cd(a.content_tsv, sq.q) AS relevance,
    -- 高亮片段
    ts_headline('english', a.content, sq.q,
                'StartSel=<b>, StopSel=</b>, MaxWords=35, MinWords=20') AS snippet,
    -- 文档长度（可选）
    length(a.content) AS doc_length
FROM articles a, search_query sq
WHERE a.content_tsv @@ sq.q
ORDER BY relevance DESC
LIMIT 10;

-- 带权重的排名
-- 假设 title 权重为 A，content 权重为 D
ALTER TABLE articles ADD COLUMN full_tsv tsvector;
UPDATE articles
SET full_tsv = setweight(to_tsvector('english', title), 'A') ||
               setweight(to_tsvector('english', content), 'D');

CREATE INDEX idx_articles_full_tsv ON articles USING gin(full_tsv);

SELECT
    id, title,
    ts_rank_cd(full_tsv, to_tsquery('english', 'search')) AS rank
FROM articles
WHERE full_tsv @@ to_tsquery('english', 'search')
ORDER BY rank DESC;
```

## 常见问题

### Q1: ts_rank 和 ts_rank_cd 应该选哪个？

**A:** 对于大多数应用场景，推荐使用 `ts_rank_cd`。它基于覆盖密度算法，不仅考虑词频，还考虑匹配词之间的距离，通常能产生更符合用户预期的排名。`ts_rank` 基于纯词频，在短文本或匹配词分散的情况下效果可能不理想。如果对性能要求极高且文本较短，`ts_rank` 会稍快一些。

### Q2: ts_headline 是否会影响查询性能？

**A:** 是的，`ts_headline` 需要对原始文档文本重新处理，不能利用 tsvector 索引。它在每一行结果上都执行文本分析和片段提取，因此当结果集较大时会显著影响性能。最佳实践是先通过 `WHERE ... @@ ...` 和 `ORDER BY rank` 加 `LIMIT` 获取少量结果，再对这些结果调用 `ts_headline`。可以使用子查询或 CTE 先筛选 TOP N 再高亮。

### Q3: 如何在不安装扩展的情况下实现基本的中文搜索？

**A:** 如果无法安装 zhparser 或 pg_jieba，可以考虑以下替代方案：
1. **应用层分词**：在应用代码中使用 jieba（Python）等库分词后，将空格分隔的词存入数据库，使用 `simple` 配置建立 tsvector。
2. **pg_trgm**：使用 trigram 相似度搜索，对中文也有一定效果，但不如专业分词精确。
3. **外部搜索引擎**：使用 Elasticsearch 等专业搜索引擎处理中文搜索，PostgreSQL 负责数据存储。
