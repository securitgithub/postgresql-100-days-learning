## 理论讲解

### 1. Python 连接 PostgreSQL 的方式

Python 生态有多种连接 PostgreSQL 的方式：

| 方式 | 说明 | 适用场景 |
|------|------|---------|
| psycopg2 | 最经典的 Python PostgreSQL 驱动 | 通用场景、生产环境 |
| psycopg3 (psycopg) | 新一代驱动，支持异步 | 新项目、异步应用 |
| SQLAlchemy | ORM 框架，支持多种数据库 | Web 应用、复杂业务逻辑 |
| pandas + sqlalchemy | 数据分析管道 | 数据科学、ETL |
| PL/Python | 数据库内执行 Python | 复杂计算、UDF |
| asyncpg | 纯异步驱动 | 高并发异步应用 |

### 2. psycopg2 vs psycopg3

psycopg2 是 Python 连接 PostgreSQL 的事实标准，已稳定运行十余年。psycopg3（包名 `psycopg`）是其继任者。

主要差异：

| 特性 | psycopg2 | psycopg3 |
|------|----------|----------|
| 异步支持 | 有限 | 原生 async/await |
| 类型适配 | 自动 | 更灵活可配置 |
| COPY 协议 | 基础支持 | 流式 COPY |
| Pipeline 模式 | 不支持 | 支持（批量操作） |
| 连接池 | 外部 pool | 内置 ConnectionPool |
| 二进制传输 | 文本协议 | 支持二进制协议 |

### 3. SQLAlchemy 架构

SQLAlchemy 分为两层：

- **Core 层**：SQL 表达式语言，类型安全的 SQL 构建器
- **ORM 层**：对象关系映射，将表映射为 Python 类

SQLAlchemy 2.0 引入了新风格的 API，推荐在新项目中使用。

### 4. pandas 与 PostgreSQL

pandas 的 `read_sql` 和 `to_sql` 方法可以直接与数据库交互，非常适合数据分析场景。配合 SQLAlchemy engine 使用效果最佳。

### 5. PL/Python

PL/Python 允许在 PostgreSQL 内部编写 Python 函数。它可以访问 Python 的全部生态（numpy、scipy 等），适合在数据库端完成复杂计算，避免大量数据传输。

### 示例1：psycopg2 基础操作

```python
import psycopg2
from psycopg2 import sql, extras
from contextlib import contextmanager

# 基本连接
conn = psycopg2.connect(
    host="localhost",
    port=5432,
    dbname="mydb",
    user="postgres",
    password="secret"
)

# 使用连接字符串
conn = psycopg2.connect("postgresql://postgres:secret@localhost:5432/mydb")

# 推荐：使用上下文管理器自动管理连接和事务
@contextmanager
def get_db_connection():
    conn = psycopg2.connect("postgresql://postgres:secret@localhost:5432/mydb")
    try:
        yield conn
        conn.commit()
    except Exception:
        conn.rollback()
        raise
    finally:
        conn.close()

# 基本 CRUD 操作
with get_db_connection() as conn:
    with conn.cursor() as cur:
        # 创建表
        cur.execute("""
            CREATE TABLE IF NOT EXISTS users (
                id SERIAL PRIMARY KEY,
                username TEXT UNIQUE NOT NULL,
                email TEXT NOT NULL,
                created_at TIMESTAMPTZ DEFAULT NOW()
            )
        """)

        # 插入（使用参数化查询防止 SQL 注入）
        cur.execute(
            "INSERT INTO users (username, email) VALUES (%s, %s) RETURNING id",
            ("张三", "zhangsan@example.com")
        )
        user_id = cur.fetchone()[0]
        print(f"新用户ID: {user_id}")

        # 批量插入（executemany 效率不高）
        users = [
            ("李四", "lisi@example.com"),
            ("王五", "wangwu@example.com"),
            ("赵六", "zhaoliu@example.com"),
        ]
        cur.executemany(
            "INSERT INTO users (username, email) VALUES (%s, %s)",
            users
        )

        # 推荐：使用 execute_batch 或 execute_values 批量插入
        extras.execute_values(
            cur,
            "INSERT INTO users (username, email) VALUES %s",
            [("钱七", "qianqi@example.com"), ("孙八", "sunba@example.com")],
            template="(%s, %s)"
        )

        # 查询
        cur.execute("SELECT * FROM users WHERE id = %s", (user_id,))
        row = cur.fetchone()
        print(f"用户: {row}")

        # 使用字典游标
with get_db_connection() as conn:
    with conn.cursor(cursor_factory=extras.RealDictCursor) as cur:
        cur.execute("SELECT * FROM users LIMIT 5")
        for row in cur:
            print(f"用户名: {row['username']}, 邮箱: {row['email']}")
```

### 示例2：psycopg2 连接池

```python
from psycopg2 import pool
import threading

# 创建连接池
connection_pool = pool.ThreadedConnectionPool(
    minconn=2,
    maxconn=10,
    host="localhost",
    dbname="mydb",
    user="postgres",
    password="secret"
)

def query_with_pool(user_id):
    """从连接池获取连接执行查询"""
    conn = connection_pool.getconn()
    try:
        with conn.cursor(cursor_factory=extras.RealDictCursor) as cur:
            cur.execute("SELECT * FROM users WHERE id = %s", (user_id,))
            return cur.fetchone()
    finally:
        # 归还连接到池中（不是关闭）
        connection_pool.putconn(conn)

# 多线程使用连接池
threads = []
for uid in range(1, 6):
    t = threading.Thread(target=lambda u: print(query_with_pool(u)), args=(uid,))
    threads.append(t)
    t.start()

for t in threads:
    t.join()

# 关闭连接池
connection_pool.closeall()
```

### 示例3：psycopg3 (psycopg) 新特性

```python
import psycopg
from psycopg.rows import dict_row, class_row
from dataclasses import dataclass

# psycopg3 基本连接（自动提交模式）
with psycopg.connect("postgresql://postgres:secret@localhost/mydb") as conn:
    # 默认使用事务
    result = conn.execute("SELECT version()").fetchone()
    print(result[0])

# 使用字典行工厂
with psycopg.connect(
    "postgresql://postgres:secret@localhost/mydb",
    row_factory=dict_row
) as conn:
    rows = conn.execute("SELECT * FROM users LIMIT 5").fetchall()
    for row in rows:
        print(row["username"])

# 使用 dataclass 行工厂
@dataclass
class User:
    id: int
    username: str
    email: str
    created_at: str

with psycopg.connect("postgresql://postgres:secret@localhost/mydb") as conn:
    cur = conn.execute("SELECT id, username, email, created_at FROM users")
    cur.row_factory = class_row(User)
    users = cur.fetchall()
    for u in users:
        print(f"{u.username} ({u.email})")

# psycopg3 内置连接池
from psycopg_pool import ConnectionPool

pool = ConnectionPool(
    "postgresql://postgres:secret@localhost/mydb",
    min_size=4,
    max_size=10
)

with pool.connection() as conn:
    result = conn.execute("SELECT COUNT(*) FROM users").fetchone()
    print(f"用户数: {result[0]}")

pool.close()

# 异步使用
import asyncio

async def async_query():
    async with await psycopg.AsyncConnection.connect(
        "postgresql://postgres:secret@localhost/mydb"
    ) as conn:
        async with conn.cursor(row_factory=dict_row) as cur:
            await cur.execute("SELECT * FROM users LIMIT 5")
            rows = await cur.fetchall()
            for row in rows:
                print(row)

asyncio.run(async_query())

# Pipeline 模式（批量操作优化）
with psycopg.connect("postgresql://postgres:secret@localhost/mydb") as conn:
    with conn.pipeline():
        # 多条语句在一次网络往返中发送
        conn.execute("INSERT INTO users (username, email) VALUES (%s, %s)",
                      ("test1", "t1@example.com"))
        conn.execute("INSERT INTO users (username, email) VALUES (%s, %s)",
                      ("test2", "t2@example.com"))
        conn.execute("UPDATE users SET email = %s WHERE username = %s",
                      ("newemail@example.com", "test1"))
```

### 示例4：SQLAlchemy ORM

```python
from sqlalchemy import create_engine, Column, Integer, String, DateTime, ForeignKey, func
from sqlalchemy.orm import DeclarativeBase, Session, relationship, sessionmaker
from datetime import datetime

# 创建引擎
engine = create_engine(
    "postgresql+psycopg2://postgres:secret@localhost/mydb",
    pool_size=5,
    max_overflow=10,
    pool_pre_ping=True,  # 自动检测断开的连接
    echo=False           # 设为 True 可打印 SQL 日志
)

# SQLAlchemy 2.0 风格的模型定义
class Base(DeclarativeBase):
    pass

class Department(Base):
    __tablename__ = "departments"

    id = Column(Integer, primary_key=True)
    name = Column(String(100), unique=True, nullable=False)
    employees = relationship("Employee", back_populates="department")

class Employee(Base):
    __tablename__ = "employees"

    id = Column(Integer, primary_key=True)
    name = Column(String(100), nullable=False)
    email = Column(String(200), unique=True)
    salary = Column(Integer)
    department_id = Column(Integer, ForeignKey("departments.id"))
    created_at = Column(DateTime, default=datetime.utcnow)

    department = relationship("Department", back_populates="employees")

    def __repr__(self):
        return f"<Employee(name={self.name}, salary={self.salary})>"

# 创建表
Base.metadata.create_all(engine)

# 使用 Session 进行 CRUD
SessionLocal = sessionmaker(bind=engine)

with SessionLocal() as session:
    # 创建
    dept = Department(name="工程部")
    session.add(dept)
    session.flush()  # 获取生成的 ID

    emp1 = Employee(name="张三", email="zs@co.com", salary=15000, department_id=dept.id)
    emp2 = Employee(name="李四", email="ls@co.com", salary=18000, department_id=dept.id)
    session.add_all([emp1, emp2])
    session.commit()

    # 查询
    emps = session.query(Employee).filter(
        Employee.salary >= 15000
    ).order_by(Employee.salary.desc()).all()
    for e in emps:
        print(e)

    # SQLAlchemy 2.0 风格查询
    from sqlalchemy import select
    stmt = select(Employee).where(Employee.salary > 10000).join(Department)
    results = session.execute(stmt).scalars().all()

    # 聚合查询
    stats = session.query(
        Department.name,
        func.count(Employee.id).label("emp_count"),
        func.avg(Employee.salary).label("avg_salary")
    ).join(Employee).group_by(Department.name).all()

    for name, count, avg_sal in stats:
        print(f"{name}: {count}人, 平均薪资: {avg_sal:.0f}")

    # 更新
    session.query(Employee).filter(Employee.name == "张三").update(
        {"salary": Employee.salary + 2000}
    )
    session.commit()

    # 删除
    session.query(Employee).filter(Employee.name == "测试").delete()
    session.commit()
```

### 示例5：pandas + PostgreSQL

```python
import pandas as pd
from sqlalchemy import create_engine

engine = create_engine("postgresql+psycopg2://postgres:secret@localhost/mydb")

# 从 PostgreSQL 读取数据到 DataFrame
df = pd.read_sql(
    """
    SELECT sale_date, region, product, amount, quantity
    FROM sales
    WHERE sale_date >= '2025-01-01'
    """,
    engine,
    parse_dates=["sale_date"]
)

print(f"数据量: {len(df)} 行")
print(df.describe())

# 使用 pandas 进行数据分析
monthly = df.groupby([pd.Grouper(key="sale_date", freq="M"), "region"]).agg(
    total_sales=("amount", "sum"),
    avg_amount=("amount", "mean"),
    order_count=("amount", "count")
).reset_index()

print(monthly.head(20))

# 透视表分析
pivot = df.pivot_table(
    values="amount",
    index="region",
    columns="product",
    aggfunc=["sum", "mean", "count"]
)
print(pivot)

# 将分析结果写回 PostgreSQL
monthly.to_sql(
    "monthly_sales_report",
    engine,
    if_exists="replace",  # 'append' 追加, 'replace' 替换, 'fail' 报错
    index=False,
    method="multi",        # 批量插入
    chunksize=1000         # 每次插入1000行
)

# 大数据量分块读取（避免内存溢出）
chunks = pd.read_sql(
    "SELECT * FROM sales",
    engine,
    chunksize=10000  # 每次读取10000行
)

total = 0
for chunk in chunks:
    total += chunk["amount"].sum()
print(f"销售总额: {total}")

# 使用 COPY 协议高速导入 CSV
from io import StringIO

csv_data = df.to_csv(index=False, header=False)
with engine.raw_connection() as conn:
    with conn.cursor() as cur:
        cur.copy_expert(
            "COPY sales (sale_date, region, product, amount, quantity) FROM STDIN WITH CSV",
            StringIO(csv_data)
        )
    conn.commit()
```

### 示例6：PL/Python 存储过程

```sql
-- 安装 PL/Python 扩展（需要超级用户权限）
CREATE EXTENSION IF NOT EXISTS plpython3u;

-- 简单的 PL/Python 函数
CREATE OR REPLACE FUNCTION py_reverse(text)
RETURNS text
LANGUAGE plpython3u
AS $$
    return args[0][::-1]
$$;

SELECT py_reverse('Hello PostgreSQL');

-- 使用 Python 库进行数据处理
CREATE OR REPLACE FUNCTION calculate_statistics(table_name text, column_name text)
RETURNS jsonb
LANGUAGE plpython3u
AS $$
    import json

    # 在 PL/Python 中使用 plpy 模块执行 SQL
    query = f"SELECT {column_name} FROM {table_name} WHERE {column_name} IS NOT NULL"
    result = plpy.execute(query)

    values = [row[column_name] for row in result]

    if not values:
        return json.dumps({"error": "no data"})

    n = len(values)
    mean = sum(values) / n
    variance = sum((x - mean) ** 2 for x in values) / n
    stddev = variance ** 0.5

    sorted_vals = sorted(values)
    median = sorted_vals[n // 2] if n % 2 == 1 else (sorted_vals[n//2 - 1] + sorted_vals[n//2]) / 2

    stats = {
        "count": n,
        "mean": round(mean, 4),
        "median": round(median, 4),
        "stddev": round(stddev, 4),
        "min": min(values),
        "max": max(values),
        "p25": sorted_vals[int(n * 0.25)],
        "p75": sorted_vals[int(n * 0.75)]
    }
    return json.dumps(stats)
$$;

SELECT calculate_statistics('sales', 'amount');

-- PL/Python 触发器函数
CREATE OR REPLACE FUNCTION py_audit_trigger()
RETURNS trigger
LANGUAGE plpython3u
AS $$
    import json
    from datetime import datetime

    # TD 是触发器数据字典
    event = TD["event"]       # INSERT, UPDATE, DELETE
    table = TD["table_name"]

    if event == "INSERT":
        new_data = json.dumps(TD["new"], default=str)
        plpy.execute(
            "INSERT INTO audit_log (table_name, action, new_data, changed_at) "
            f"VALUES ('{table}', 'INSERT', '{new_data}', NOW())"
        )
    elif event == "UPDATE":
        old_data = json.dumps(TD["old"], default=str)
        new_data = json.dumps(TD["new"], default=str)
        plpy.execute(
            "INSERT INTO audit_log (table_name, action, old_data, new_data, changed_at) "
            f"VALUES ('{table}', 'UPDATE', '{old_data}', '{new_data}', NOW())"
        )

    return "OK"
$$;
```

## 常见问题

### Q1: psycopg2 和 psycopg3 该选哪个？

**A:** 如果是新项目且使用 Python 3.7+，推荐 psycopg3（`psycopg` 包），它有更好的异步支持、内置连接池和性能优化。如果是维护现有项目或者依赖的第三方库（如某些版本的 Django）还没有适配 psycopg3，继续使用 psycopg2 也完全没问题，它非常稳定成熟。

### Q2: pandas 的 to_sql 写入速度很慢怎么办？

**A:** `to_sql` 默认是逐行插入，可以通过以下方式加速：(1) 设置 `method="multi"` 启用批量插入；(2) 设置 `chunksize=1000` 分批提交；(3) 对于大数据量（百万行+），使用 COPY 协议，通过 `psycopg2` 的 `copy_expert` 方法配合 `StringIO` 可以获得最佳写入性能，通常比 `to_sql` 快 10-50 倍。

### Q3: PL/Python 的 plpython3u 中的 "u" 是什么意思？

**A:** "u" 代表 "untrusted"（不受信任），意味着该语言可以访问操作系统的文件系统、网络等资源，因此只有超级用户才能创建 PL/Python 函数。PostgreSQL 没有提供 trusted 版本的 PL/Python（即 `plpython3`），因为 Python 语言本身很难做到安全沙箱化。在生产环境中要谨慎使用，确保函数代码经过安全审计。
