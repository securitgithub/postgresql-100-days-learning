### 4. 关键 SQL 模式速查

以下是日常工作中最常用的 SQL 模式：

```sql
-- ============================================================
-- 模式1：分页查询（高效写法）
-- ============================================================
-- 避免大 OFFSET 的性能问题
SELECT * FROM orders
WHERE id > :last_id  -- 游标分页
ORDER BY id
LIMIT 20;

-- ============================================================
-- 模式2：UPSERT（存在则更新，不存在则插入）
-- ============================================================
INSERT INTO product_stats (product_id, view_count, last_viewed)
VALUES (100, 1, NOW())
ON CONFLICT (product_id) DO UPDATE SET
    view_count = product_stats.view_count + 1,
    last_viewed = NOW();

-- ============================================================
-- 模式3：批量更新（从 VALUES 列表）
-- ============================================================
UPDATE products AS p SET
    price = v.new_price,
    stock = v.new_stock
FROM (VALUES
    (1, 99.00, 100),
    (2, 199.00, 50),
    (3, 299.00, 200)
) AS v(id, new_price, new_stock)
WHERE p.id = v.id;

-- ============================================================
-- 模式4：递归查询（树结构）
-- ============================================================
WITH RECURSIVE category_tree AS (
    SELECT id, name, parent_id, 1 AS level,
           name::text AS path
    FROM categories WHERE parent_id IS NULL
    UNION ALL
    SELECT c.id, c.name, c.parent_id, ct.level + 1,
           ct.path || ' > ' || c.name
    FROM categories c
    JOIN category_tree ct ON c.parent_id = ct.id
)
SELECT * FROM category_tree ORDER BY path;

-- ============================================================
-- 模式5：条件聚合（一次扫描多种统计）
-- ============================================================
SELECT
    COUNT(*) AS total,
    COUNT(*) FILTER (WHERE status = 1) AS paid,
    COUNT(*) FILTER (WHERE status = 4) AS completed,
    SUM(pay_amount) FILTER (WHERE status >= 1) AS revenue,
    AVG(pay_amount) FILTER (WHERE status >= 1) AS avg_order
FROM orders
WHERE created_at >= date_trunc('month', CURRENT_DATE);

-- ============================================================
-- 模式6：横转纵 / 纵转横
-- ============================================================
-- 纵转横（使用 FILTER）
SELECT
    product_id,
    AVG(rating) FILTER (WHERE date_trunc('month', created_at) = '2025-01-01') AS jan_rating,
    AVG(rating) FILTER (WHERE date_trunc('month', created_at) = '2025-02-01') AS feb_rating,
    AVG(rating) FILTER (WHERE date_trunc('month', created_at) = '2025-03-01') AS mar_rating
FROM reviews
GROUP BY product_id;

-- ============================================================
-- 模式7：间隔分析（查找数据中的间隔）
-- ============================================================
WITH ordered AS (
    SELECT *, LAG(created_at) OVER (ORDER BY created_at) AS prev_time
    FROM orders WHERE user_id = 100
)
SELECT *,
    created_at - prev_time AS gap
FROM ordered
WHERE created_at - prev_time > INTERVAL '30 days';
```

### 5. 后续学习方向

#### 方向A：内核源码研究

对于想要深入理解 PostgreSQL 原理的开发者：

- **源码阅读入门**：从 `src/backend/executor/` 开始理解查询执行
- **存储引擎**：`src/backend/storage/` 理解 MVCC 实现
- **WAL 机制**：`src/backend/access/transam/` 理解事务日志
- **查询优化器**：`src/backend/optimizer/` 理解执行计划生成
- **推荐书籍**：《The Internals of PostgreSQL》（免费在线）

- **报告 Bug**：在 pgsql-bugs 邮件列表报告发现的问题

- **参与代码评审**：在 commitfest 上评审他人的补丁
- **翻译文档**：参与 PostgreSQL 中文文档翻译
- **开发扩展**：在 PGXN 发布自己的扩展

#### 方向C：认证考试

- **PostgreSQL CE（Certified Engineer）**：由 EnterpriseDB 提供
- **考试内容**：安装配置、SQL、管理、调优、高可用、安全
- **准备建议**：在真实环境中练习所有操作，不仅仅是理论

### 示例1：个人知识库查询助手

```sql
-- ============================================================
-- 创建个人 PostgreSQL 知识库
-- ============================================================
CREATE TABLE pg_knowledge_base (
    id          SERIAL PRIMARY KEY,
    category    TEXT NOT NULL,          -- 分类
    topic       TEXT NOT NULL,          -- 主题
    content     TEXT NOT NULL,          -- 内容
    tags        TEXT[],                 -- 标签
    difficulty  SMALLINT DEFAULT 3,     -- 难度 1-5
    importance  SMALLINT DEFAULT 3,     -- 重要性 1-5
    day_learned INTEGER,               -- 第几天学到的
    notes       TEXT,                   -- 个人笔记
    search_vec  TSVECTOR,
    created_at  TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_kb_search ON pg_knowledge_base USING GIN (search_vec);
CREATE INDEX idx_kb_tags ON pg_knowledge_base USING GIN (tags);
CREATE INDEX idx_kb_category ON pg_knowledge_base (category);

-- 自动更新搜索向量
CREATE TRIGGER trg_kb_search_vec
    BEFORE INSERT OR UPDATE ON pg_knowledge_base
    FOR EACH ROW EXECUTE FUNCTION
    tsvector_update_trigger(search_vec, 'pg_catalog.simple', topic, content, notes);

-- 插入知识条目
INSERT INTO pg_knowledge_base (category, topic, content, tags, difficulty, importance, day_learned)
VALUES
    ('索引', 'B-tree 索引原理',
     '默认索引类型，适用于等值和范围查询。支持排序、唯一约束。叶子节点按顺序存储。',
     ARRAY['索引', 'B-tree', '性能'], 3, 5, 21),
    ('索引', 'GIN 索引',
     '广义倒排索引，适用于数组、JSONB、全文搜索等多值类型。构建慢但查询快。',
     ARRAY['索引', 'GIN', 'JSONB', '全文搜索'], 4, 4, 25),
    ('事务', 'MVCC 原理',
     '多版本并发控制，每个事务看到数据的一致性快照。通过 xmin/xmax 判断行可见性。',
     ARRAY['事务', 'MVCC', '并发'], 5, 5, 45);

-- 搜索知识库
SELECT topic, content, tags, difficulty, importance
FROM pg_knowledge_base
WHERE search_vec @@ plainto_tsquery('simple', '索引 查询')
ORDER BY importance DESC, difficulty;

-- 按分类浏览
SELECT
    category,
    COUNT(*) AS entries,
    array_agg(topic ORDER BY importance DESC) AS topics
FROM pg_knowledge_base
GROUP BY category
ORDER BY category;
```

### 示例2：学习进度追踪

```sql
-- ============================================================
-- 学习进度追踪系统
-- ============================================================
CREATE TABLE learning_progress (
    day         INTEGER PRIMARY KEY,
    topic       TEXT NOT NULL,
    status      TEXT DEFAULT 'completed',  -- completed/reviewed/needs_review
    confidence  SMALLINT DEFAULT 3,        -- 掌握程度 1-5
    time_spent  INTEGER,                   -- 学习时长（分钟）
    key_points  TEXT[],                    -- 关键知识点
    review_date DATE,                      -- 下次复习日期
    notes       TEXT
);

-- 学习曲线分析
SELECT
    CASE
        WHEN day BETWEEN 1 AND 20 THEN '阶段1: 基础入门'
        WHEN day BETWEEN 21 AND 40 THEN '阶段2: 核心技能'
        WHEN day BETWEEN 41 AND 60 THEN '阶段3: 高级特性'
        WHEN day BETWEEN 61 AND 80 THEN '阶段4: 架构运维'
        WHEN day BETWEEN 81 AND 100 THEN '阶段5: 综合实践'
    END AS phase,
    COUNT(*) AS days_completed,
    ROUND(AVG(confidence), 1) AS avg_confidence,
    SUM(time_spent) AS total_minutes,
    array_agg(DISTINCT topic ORDER BY topic) AS topics
FROM learning_progress
GROUP BY
    CASE
        WHEN day BETWEEN 1 AND 20 THEN '阶段1: 基础入门'
        WHEN day BETWEEN 21 AND 40 THEN '阶段2: 核心技能'
        WHEN day BETWEEN 41 AND 60 THEN '阶段3: 高级特性'
        WHEN day BETWEEN 61 AND 80 THEN '阶段4: 架构运维'
        WHEN day BETWEEN 81 AND 100 THEN '阶段5: 综合实践'
    END
ORDER BY phase;

-- 需要复习的知识点（艾宾浩斯遗忘曲线）
SELECT day, topic, confidence, review_date
FROM learning_progress
WHERE confidence < 4
   OR review_date <= CURRENT_DATE
ORDER BY confidence, review_date;
```

### 示例4：常用运维命令速查

```bash
#!/bin/bash
# ============================================================
# PostgreSQL 运维常用命令速查
# ============================================================

# === 服务管理 ===
# 启动/停止/重启/重载配置
sudo systemctl start postgresql
sudo systemctl stop postgresql
sudo systemctl restart postgresql
sudo systemctl reload postgresql    # 重载配置不断开连接

# === 连接 ===
psql -h localhost -p 5432 -U postgres -d mydb
psql "postgresql://user:pass@host:5432/db?sslmode=require"

# === 备份 ===
pg_dump -Fc -f backup.dump mydb                  # 自定义格式备份
pg_dump -Fp -f backup.sql mydb                   # SQL 文本备份
pg_dumpall -f all_databases.sql                   # 全部数据库
pg_basebackup -D /backup -Ft -z --wal-method=stream  # 物理备份

# === 恢复 ===
pg_restore -d mydb -j 4 backup.dump              # 并行恢复
psql -d mydb -f backup.sql                       # SQL文本恢复

# === 常用查询快捷方式 ===
psql -c "SELECT version();"
psql -c "SELECT pg_size_pretty(pg_database_size('mydb'));"
psql -c "SELECT count(*) FROM pg_stat_activity;"
psql -c "SELECT * FROM pg_stat_replication;"

# === 性能 ===
pg_isready -h localhost                           # 检查服务是否运行
pgbench -i -s 100 mydb                           # 初始化测试数据

# === 日志 ===
tail -f /var/log/postgresql/postgresql-16-main.log

# === 配置文件位置 ===
```

## 常见问题

### Q1: 100 天学完之后，如何持续提升 PostgreSQL 技能？

**A:** 几个建议：(1) **在工作中实践**是最好的老师，主动承担数据库相关任务；(2) **关注社区动态**，订阅 Planet PostgreSQL 和邮件列表，了解新版本特性；(3) **参加技术会议**，如 PGConf、PostgreSQL 中文社区活动；(4) **阅读源码**，从自己感兴趣的模块开始（如索引实现、查询优化器）；(5) **定期复习**，使用间隔重复法巩固知识；(6) **分享输出**，写博客、做技术分享，教别人的过程中加深理解。

### Q2: PostgreSQL DBA 和 PostgreSQL 开发者的技能侧重有何不同？

**A:** **DBA 侧重运维方向**：备份恢复、高可用搭建、性能监控、容量规划、升级迁移、安全合规、故障诊断。需要深入理解 WAL、MVCC、autovacuum、复制等内部机制。**开发者侧重应用方向**：高效 SQL 编写、索引策略、ORM 使用、数据模型设计、存储过程开发、扩展使用。当然两者有很大重叠，理想的状态是两方面都有扎实基础。

### Q3: PostgreSQL 相比 MySQL 在哪些场景有明显优势？

**A:** (1) **复杂查询**：PostgreSQL 的查询优化器更先进，支持窗口函数、CTE、LATERAL JOIN 等高级特性；(2) **数据类型**：原生支持 JSON、数组、范围、几何等丰富类型；(3) **扩展性**：强大的扩展机制，PostGIS、TimescaleDB、pgvector 等生态丰富；(4) **标准合规**：对 SQL 标准的支持最完整；(5) **并发控制**：真正的 MVCC 实现，读写不相互阻塞；(6) **可靠性**：数据一致性保证强，适合金融等对数据准确性要求高的场景。
