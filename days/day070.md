# Day 070: 全文搜索（一）

## 学习目标

- 理解 PostgreSQL 全文搜索的核心概念和数据类型
- 掌握 tsvector 和 tsquery 类型的创建和使用
- 学会使用 to_tsvector() 和 to_tsquery() 函数
- 理解 @@ 匹配操作符的工作原理
- 掌握 GIN 全文索引的创建和使用

## 理论讲解

### 一、为什么需要全文搜索？

传统的 `LIKE` 和正则表达式搜索存在明显局限：

```sql
-- LIKE 搜索的问题
SELECT * FROM articles WHERE body LIKE '%数据库%';
```

问题包括：
- **无法使用索引**：`LIKE '%keyword%'` 必须全表扫描
- **无法处理词形变化**：searching/searched/searches 被视为不同词
- **无法排序相关度**：无法判断哪篇文章与关键词最相关
- **无法处理同义词、停用词**
- **不支持布尔运算**：难以表达"包含 A 且包含 B 但不包含 C"

PostgreSQL 内置的全文搜索（Full Text Search，FTS）解决了所有这些问题。

### 二、核心数据类型

#### 2.1 tsvector（文本搜索向量）

`tsvector` 是对文档进行预处理后的结构化表示。它将文本分解为**词位（lexeme）**——经过标准化处理的词。

```sql
SELECT 'The quick brown fox jumps over the lazy dog'::tsvector;
-- 结果: 'The' 'brown' 'dog' 'fox' 'jumps' 'lazy' 'over' 'quick' 'the'
```

注意：直接转换 `::tsvector` 不会做语言处理，只是简单分词。

使用 `to_tsvector()` 函数会进行完整的语言处理：

```sql
SELECT to_tsvector('english', 'The quick brown fox jumps over the lazy dog');
-- 结果: 'brown':3 'dog':9 'fox':4 'jump':5 'lazi':8 'quick':2
```

处理过程包括：
1. **分词（Parser）**：将文本拆分为 token
2. **标准化（Normalization）**：转小写、词干提取（jumping → jump）
3. **去停用词（Stop Words）**：移除 the、a、is 等无意义词
4. **记录位置**：每个词位保留在原文中的位置信息

#### 2.2 tsquery（文本搜索查询）

`tsquery` 表示搜索条件，支持逻辑操作符：

| 操作符 | 含义 | 示例 |
|--------|------|------|
| `&` | AND | `'cat' & 'dog'` |
| `\|` | OR | `'cat' \| 'dog'` |
| `!` | NOT | `!'cat'` |
| `<->` | 紧跟（FOLLOWED BY） | `'big' <-> 'data'`（big 紧跟 data） |
| `<N>` | 距离 N 个词 | `'big' <2> 'data'`（间隔 2 个词） |

```sql
SELECT to_tsquery('english', 'quick & brown & !cat');
-- 结果: 'quick' & 'brown' & !'cat'

SELECT to_tsquery('english', 'jumping');
-- 结果: 'jump'  （自动词干提取）
```

### 三、匹配操作符 @@

`@@` 操作符用于判断 `tsvector` 是否匹配 `tsquery`：

```sql
SELECT to_tsvector('english', 'The quick brown fox')
    @@ to_tsquery('english', 'quick & fox');
-- 结果: true

SELECT to_tsvector('english', 'The quick brown fox')
    @@ to_tsquery('english', 'quick & cat');
-- 结果: false
```

`@@` 操作符的重载形式：
- `tsvector @@ tsquery`：标准用法
- `tsquery @@ tsvector`：参数顺序反过来也可以
- `text @@ tsquery`：自动将 text 转为 tsvector（使用默认配置）
- `text @@ text`：左侧转 tsvector，右侧转 plainto_tsquery

### 四、常用函数

#### 4.1 to_tsvector()

```sql
-- 指定语言配置
to_tsvector('english', 'text content')
to_tsvector('simple', 'text content')    -- simple 不做词干提取
to_tsvector('chinese', '中文内容')       -- 需要中文分词支持

-- 使用默认配置
to_tsvector('text content')  -- 使用 default_text_search_config
```

#### 4.2 to_tsquery()

要求输入格式化的查询字符串（词之间用 `&`、`|` 等连接）：

```sql
to_tsquery('english', 'cat & dog')
to_tsquery('english', 'cat | dog')
to_tsquery('english', 'super <-> hero')
```

#### 4.3 plainto_tsquery()

接受普通文本，所有词之间用 AND 连接：

```sql
plainto_tsquery('english', 'quick brown fox')
-- 等同于 to_tsquery('english', 'quick & brown & fox')
```

#### 4.4 phraseto_tsquery()

词之间保持原有顺序（相邻）：

```sql
phraseto_tsquery('english', 'quick brown fox')
-- 等同于 to_tsquery('english', 'quick <-> brown <-> fox')
```

#### 4.5 websearch_to_tsquery()（PostgreSQL 11+）

支持类似搜索引擎的语法：

```sql
websearch_to_tsquery('english', 'quick brown -cat "exact phrase"')
-- 'quick' & 'brown' & !'cat' & 'exact' <-> 'phrase'
```

### 五、GIN 全文索引

对 `tsvector` 列创建 GIN 索引可以极大加速全文搜索：

```sql
CREATE INDEX idx_fts ON articles USING gin (to_tsvector('english', body));
```

两种创建方式：
1. **表达式索引**：直接在文本列上用函数创建
2. **存储列 + 索引**：添加一个 tsvector 类型的列，用触发器维护

### 六、中文全文搜索

PostgreSQL 默认不支持中文分词。需要安装扩展：
- **pg_jieba**：基于结巴分词
- **zhparser**：基于 SCWS 分词
- **pg_bigm**：基于 2-gram 的模糊匹配

安装后配置为文本搜索配置即可使用。

## 代码示例

### 示例1：tsvector 和 tsquery 基础

```sql
-- tsvector 的创建
SELECT 'hello world'::tsvector;
-- 'hello' 'world'（无位置信息）

SELECT to_tsvector('english', 'Hello World! PostgreSQL is amazing.');
-- 'amaz':5 'hello':1 'postgresql':3 'world':2

-- 查看分词细节
SELECT alias, description, token
FROM ts_debug('english', 'Hello World! PostgreSQL is amazing.');
-- 可以看到每个 token 的处理过程

-- tsquery 的创建
SELECT to_tsquery('english', 'hello & world');
SELECT to_tsquery('english', 'running');  -- 会变成 'run'

-- plainto_tsquery：普通文本输入
SELECT plainto_tsquery('english', 'hello beautiful world');
-- 'hello' & 'beauti' & 'world'

-- phraseto_tsquery：短语搜索
SELECT phraseto_tsquery('english', 'hello world');
-- 'hello' <-> 'world'

-- websearch_to_tsquery：搜索引擎风格
SELECT websearch_to_tsquery('english', '"hello world" postgresql -mysql');
-- 'hello' <-> 'world' & 'postgresql' & !'mysql'
```

### 示例2：@@ 匹配操作

```sql
-- 基本匹配
SELECT to_tsvector('english', 'PostgreSQL is a powerful database system')
    @@ to_tsquery('english', 'powerful & database');
-- true

SELECT to_tsvector('english', 'PostgreSQL is a powerful database system')
    @@ to_tsquery('english', 'powerful & nosql');
-- false

-- 短语匹配（词的顺序和位置）
SELECT to_tsvector('english', 'big data processing')
    @@ to_tsquery('english', 'big <-> data');
-- true（big 紧跟 data）

SELECT to_tsvector('english', 'data is big')
    @@ to_tsquery('english', 'big <-> data');
-- false（顺序不对）

-- OR 匹配
SELECT to_tsvector('english', 'I love PostgreSQL')
    @@ to_tsquery('english', 'mysql | postgresql');
-- true

-- NOT 匹配
SELECT to_tsvector('english', 'I love PostgreSQL')
    @@ to_tsquery('english', 'postgresql & !mysql');
-- true
```

### 示例4：存储 tsvector 列（推荐方式）

```sql
-- 添加 tsvector 列
ALTER TABLE articles ADD COLUMN search_vector tsvector;

-- 填充 tsvector 列（合并标题和正文，标题权重更高）
UPDATE articles SET search_vector =
    setweight(to_tsvector('english', title), 'A') ||
    setweight(to_tsvector('english', body), 'B');

-- 创建 GIN 索引
CREATE INDEX idx_articles_search ON articles USING gin (search_vector);

-- 创建触发器自动维护 search_vector
CREATE OR REPLACE FUNCTION articles_search_trigger() RETURNS trigger AS $$
BEGIN
    NEW.search_vector :=
        setweight(to_tsvector('english', COALESCE(NEW.title, '')), 'A') ||
        setweight(to_tsvector('english', COALESCE(NEW.body, '')), 'B');
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER trig_articles_search
    BEFORE INSERT OR UPDATE ON articles
    FOR EACH ROW EXECUTE FUNCTION articles_search_trigger();

-- 测试触发器
INSERT INTO articles (title, body, author) VALUES
('Getting Started with PostgreSQL Indexes',
 'This guide covers B-tree, GIN, GiST, and BRIN indexes in PostgreSQL. Learn when to use each type for optimal performance.',
 'Alice');

-- 验证 search_vector 已自动填充
SELECT id, title, search_vector FROM articles WHERE id = 9;

-- 搜索
SELECT id, title
FROM articles
WHERE search_vector @@ to_tsquery('english', 'index & performance');
```

### 示例5：搜索结果排序（相关度排名）

```sql
-- ts_rank：基于词频的排名
SELECT
    id,
    title,
    ts_rank(search_vector, query) AS rank
FROM articles, to_tsquery('english', 'postgresql') AS query
WHERE search_vector @@ query
ORDER BY rank DESC;

-- ts_rank_cd：基于覆盖密度的排名（考虑词的距离）
SELECT
    id,
    title,
    ts_rank_cd(search_vector, query) AS rank
FROM articles, to_tsquery('english', 'postgresql & feature') AS query
WHERE search_vector @@ query
ORDER BY rank DESC;

-- 权重会影响排名（标题中的匹配权重更高）
-- setweight 设置的 A 权重 > B 权重
SELECT
    id,
    title,
    ts_rank(search_vector, query, 0) AS rank
FROM articles, to_tsquery('english', 'postgresql') AS query
WHERE search_vector @@ query
ORDER BY rank DESC;
```

### 示例6：搜索结果高亮

```sql
-- ts_headline：在原文中高亮匹配的词
```

## 常见问题

### Q1：PostgreSQL 全文搜索和 Elasticsearch 相比如何？

**A**：两者各有优势：
- **PostgreSQL FTS** 的优势：无需额外组件、与关系数据事务一致、维护简单、适合中等规模（百万到千万文档）
- **Elasticsearch** 的优势：更丰富的分析器和 tokenizer、更好的分布式支持、更强的聚合分析能力、适合超大规模（数十亿文档）

如果搜索是核心功能且数据量巨大，选择 Elasticsearch。如果搜索只是辅助功能且不想增加架构复杂度，PostgreSQL FTS 足够。

### Q2：如何在 PostgreSQL 中实现中文全文搜索？

**A**：需要安装中文分词扩展：
1. 安装 `zhparser` 或 `pg_jieba` 扩展
2. 创建对应的文本搜索配置
3. 使用该配置创建 tsvector 和 tsquery

```sql
-- 以 zhparser 为例（需要先安装）
CREATE EXTENSION zhparser;
CREATE TEXT SEARCH CONFIGURATION chinese (PARSER = zhparser);
ALTER TEXT SEARCH CONFIGURATION chinese
    ADD MAPPING FOR n,v,a,i,e,l WITH simple;

SELECT to_tsvector('chinese', 'PostgreSQL是一个强大的数据库');
```

如果不想安装扩展，可以使用 `pg_bigm`（基于 bi-gram）作为替代方案，不需要分词。

### Q3：tsvector 列应该存储还是每次计算？

**A**：推荐**存储 tsvector 列并通过触发器维护**：
- 存储列 + GIN 索引：搜索时直接读取索引，最快
- 表达式索引：每次 INSERT/UPDATE 时计算，查询时也可能需要重新计算
- 实时计算（不存储）：每次搜索都要全表计算，最慢

存储的代价是额外的磁盘空间和写入时的触发器开销，但对于搜索密集型应用完全值得。
