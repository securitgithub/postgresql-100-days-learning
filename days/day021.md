# Day 021: 集合操作 — UNION/UNION ALL、INTERSECT、EXCEPT

## 学习目标

- 理解 SQL 集合操作的基本概念及其与数学集合论的对应关系

- 熟练使用 INTERSECT 取交集和 EXCEPT 取差集
- 了解集合操作的列匹配规则、类型转换规则与排序方法
- 能够在实际业务场景中灵活组合多种集合操作

## 理论讲解

### 什么是集合操作

SQL 中的集合操作允许我们将两个或多个 `SELECT` 查询的结果合并为一个结果集。PostgreSQL 支持以下四种集合操作：

| 操作 | 说明 | 是否去重 |
|------|------|----------|
| `UNION` | 合并两个结果集，去除重复行 | 是 |
| `UNION ALL` | 合并两个结果集，保留所有行 | 否 |
| `INTERSECT` | 返回两个结果集的交集 | 是 |
| `EXCEPT` | 返回在第一个结果集中但不在第二个中的行 | 是 |

### 集合操作的规则

1. **列数必须相同**：参与集合操作的每个 SELECT 语句必须返回相同数量的列。
2. **类型兼容**：对应位置的列必须具有兼容的数据类型，PostgreSQL 会尝试隐式类型转换。
3. **列名取自第一个查询**：最终结果集的列名由第一个 SELECT 决定。
4. **ORDER BY 只能出现在最后**：排序子句只能放在整个集合操作的末尾。

### UNION 与 UNION ALL 的性能差异

`UNION` 需要对合并后的结果进行去重操作（类似于 `DISTINCT`），这涉及排序或哈希操作，因此在大数据量下性能开销较大。如果你确定不会有重复行，或者不关心重复，应优先使用 `UNION ALL`。

### INTERSECT 与 EXCEPT 的顺序敏感性

- `INTERSECT` 是对称的：`A INTERSECT B` 等价于 `B INTERSECT A`。
- `EXCEPT` 是不对称的：`A EXCEPT B` 与 `B EXCEPT A` 结果通常不同。

### 操作优先级

在同一查询中使用多种集合操作时，`INTERSECT` 的优先级高于 `UNION` 和 `EXCEPT`。可以使用括号来明确优先级。

## 代码示例

### 准备测试数据

```sql
-- 创建两个部门的员工表
CREATE TABLE dept_a_employees (
    emp_id   INTEGER PRIMARY KEY,
    name     VARCHAR(50),
    skill    VARCHAR(50)
);

CREATE TABLE dept_b_employees (
    emp_id   INTEGER PRIMARY KEY,
    name     VARCHAR(50),
    skill    VARCHAR(50)
);

INSERT INTO dept_a_employees VALUES
    (1, '张三', 'Python'),
    (2, '李四', 'Java'),
    (3, '王五', 'PostgreSQL'),
    (4, '赵六', 'Go');

INSERT INTO dept_b_employees VALUES
    (3, '王五', 'PostgreSQL'),
    (4, '赵六', 'Go'),
    (5, '孙七', 'Rust'),
    (6, '周八', 'Python');
```

### 示例一：UNION — 合并并去重

```sql
-- 查询两个部门的所有员工（去除重复）
SELECT emp_id, name, skill FROM dept_a_employees
UNION
SELECT emp_id, name, skill FROM dept_b_employees
ORDER BY emp_id;

-- 结果：返回 6 行（王五和赵六只出现一次）
```

### 示例二：UNION ALL — 合并保留全部

```sql
-- 查询两个部门的所有记录（保留重复）
SELECT emp_id, name, skill FROM dept_a_employees
UNION ALL
SELECT emp_id, name, skill FROM dept_b_employees
ORDER BY emp_id;

-- 结果：返回 8 行（王五和赵六各出现两次）
```

### 示例三：INTERSECT — 取交集

```sql
-- 查询同时在两个部门的员工
SELECT emp_id, name, skill FROM dept_a_employees
INTERSECT
SELECT emp_id, name, skill FROM dept_b_employees;

-- 结果：返回王五和赵六
```

### 示例四：EXCEPT — 取差集

```sql
-- 查询只在部门 A 但不在部门 B 的员工
SELECT emp_id, name, skill FROM dept_a_employees
EXCEPT
SELECT emp_id, name, skill FROM dept_b_employees;

-- 结果：返回张三和李四

-- 反向差集：只在部门 B 但不在部门 A
SELECT emp_id, name, skill FROM dept_b_employees
EXCEPT
SELECT emp_id, name, skill FROM dept_a_employees;

-- 结果：返回孙七和周八
```

### 示例五：组合使用集合操作

```sql
-- 使用括号控制优先级
-- 需求：(A 并 B) 减去 (只会 Python 的人)
(
    SELECT emp_id, name, skill FROM dept_a_employees
    UNION
    SELECT emp_id, name, skill FROM dept_b_employees
)
EXCEPT
SELECT emp_id, name, skill FROM dept_a_employees WHERE skill = 'Python';
```

### 示例六：集合操作中的类型转换

```sql
-- 不同类型的列可以通过隐式转换合并
SELECT 1 AS id, 'hello'::VARCHAR AS info
UNION
SELECT 2, 'world'::TEXT;

-- integer 和 integer 兼容，varchar 和 text 兼容
```

### 示例七：使用 UNION ALL 实现报表汇总

```sql
-- 模拟月度销售汇总报表
SELECT '部门A' AS department, COUNT(*) AS headcount FROM dept_a_employees
UNION ALL
SELECT '部门B', COUNT(*) FROM dept_b_employees
UNION ALL
SELECT '总计', (SELECT COUNT(DISTINCT emp_id)
                FROM (
                    SELECT emp_id FROM dept_a_employees
                    UNION
                    SELECT emp_id FROM dept_b_employees
                ) t);
```

## 常见问题

### Q1：UNION 和 JOIN 有什么区别？
**A**：`UNION` 是纵向合并（增加行数），将多个查询的结果上下拼接；`JOIN` 是横向合并（增加列数），将多个表的列按条件匹配后左右拼接。它们解决的是完全不同的问题。

### Q2：为什么 UNION 比 UNION ALL 慢？
**A**：`UNION` 在合并后需要执行去重操作，PostgreSQL 通常使用 Hash 或 Sort 算法来实现，这会消耗额外的 CPU 和内存。如果数据量大且不需要去重，务必使用 `UNION ALL`。

### Q3：集合操作中可以使用 WHERE 子句吗？
**A**：可以。每个参与集合操作的 `SELECT` 语句都可以有自己的 `WHERE`、`GROUP BY`、`HAVING` 等子句，但 `ORDER BY` 只能出现在最外层。
