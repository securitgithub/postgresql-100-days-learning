# Day 073: 范围类型与排他约束 — 区间数据的优雅处理

## 学习目标

- 理解 PostgreSQL 范围类型（Range Types）的概念和内置类型
- 掌握范围操作符 `@>`、`<@`、`&&`、`-|-` 等的使用
- 学会创建和使用排他约束（Exclusion Constraints）解决区间重叠问题
- 理解 GiST 索引在范围类型上的应用
- 能够在实际业务中运用范围类型处理时间区间、价格区间等场景

## 理论讲解

### 1. 范围类型概述

范围类型（Range Types）用于表示某个元素类型的值的区间。PostgreSQL 内置了以下范围类型：

| 类型 | 基础类型 | 说明 |
|------|---------|------|
| `int4range` | integer | 整数范围 |
| `int8range` | bigint | 长整数范围 |
| `numrange` | numeric | 数字范围 |
| `tsrange` | timestamp | 时间戳范围（无时区） |
| `tstzrange` | timestamptz | 时间戳范围（有时区） |
| `daterange` | date | 日期范围 |

范围的边界可以是包含（inclusive）或排除（exclusive）：
- `[a, b]`：包含 a 和 b
- `[a, b)`：包含 a，排除 b（最常用）
- `(a, b]`：排除 a，包含 b
- `(a, b)`：排除 a 和 b
- `empty`：空范围

### 2. 范围操作符

PostgreSQL 提供了丰富的范围操作符：

| 操作符 | 含义 | 示例 |
|--------|------|------|
| `@>` | 包含 | `[1,10) @> 5` → true |
| `<@` | 被包含 | `5 <@ [1,10)` → true |
| `&&` | 重叠 | `[1,5) && [3,8)` → true |
| `<<` | 完全左边 | `[1,3) << [5,8)` → true |
| `>>` | 完全右边 | `[5,8) >> [1,3)` → true |
| `-\|-` | 相邻 | `[1,5) -\|- [5,8)` → true |
| `+` | 并集 | `[1,5) + [3,8)` → [1,8) |
| `*` | 交集 | `[1,5) * [3,8)` → [3,5) |
| `-` | 差集 | `[1,8) - [3,5)` → 取决于结果 |

### 3. 排他约束（Exclusion Constraints）

排他约束是 PostgreSQL 特有的约束类型，它保证对于指定的列和操作符，任意两行之间的比较不能全部为真。

最典型的应用是防止时间区间重叠：确保同一资源在同一时间段不会被重复预订。

排他约束需要 GiST 索引支持，对于范围类型需要安装 `btree_gist` 扩展来支持等值比较。

### 4. GiST 索引与范围类型

GiST（Generalized Search Tree）索引天然支持范围类型的操作符查询。它可以高效地回答"哪些范围包含这个点"、"哪些范围与给定范围重叠"等查询。

## 代码示例

### 示例1：范围类型基本操作

```sql
-- 创建不同类型的范围值
SELECT
    int4range(1, 10) AS int_range,          -- [1,10)
    int4range(1, 10, '[]') AS int_closed,   -- [1,11) （整数自动规范化）
    numrange(1.5, 9.5) AS num_range,        -- [1.5,9.5)
    daterange('2024-01-01', '2024-12-31') AS date_range,
    tsrange('2024-01-01 08:00', '2024-01-01 17:00') AS ts_range;

-- 范围边界提取
SELECT
    lower(int4range(1, 10)) AS lower_bound,  -- 1
    upper(int4range(1, 10)) AS upper_bound,  -- 10
    lower_inc(int4range(1, 10)) AS lower_inclusive,  -- true
    upper_inc(int4range(1, 10)) AS upper_inclusive,  -- false
    isempty(int4range(1, 1)) AS is_empty;    -- true ([1,1) 是空的)

-- 特殊范围值
SELECT
    'empty'::int4range AS empty_range,
    int4range(NULL, 10) AS no_lower,   -- (,10) 无下界
    int4range(1, NULL) AS no_upper,    -- [1,)  无上界
    '[,]'::int4range AS infinite;      -- (,)   全范围
```

### 示例2：范围操作符实战

```sql
-- 包含检查
SELECT
    int4range(1, 10) @> 5 AS contains_elem,      -- true
    int4range(1, 10) @> int4range(3, 7) AS contains_range,  -- true
    5 <@ int4range(1, 10) AS elem_in_range;       -- true

-- 重叠检查
SELECT
    int4range(1, 5) && int4range(3, 8) AS overlaps,       -- true
    int4range(1, 3) && int4range(5, 8) AS no_overlap,     -- false
    int4range(1, 5) && int4range(5, 8) AS boundary_check; -- false ([1,5)和[5,8)不重叠)

-- 相邻检查
SELECT
    int4range(1, 5) -|- int4range(5, 8) AS adjacent;  -- true

-- 位置比较
SELECT
    int4range(1, 3) << int4range(5, 8) AS strictly_left,   -- true
    int4range(5, 8) >> int4range(1, 3) AS strictly_right;  -- true

-- 集合运算
SELECT
    int4range(1, 5) + int4range(3, 8) AS union_range,      -- [1,8)
    int4range(1, 8) * int4range(3, 10) AS intersect_range, -- [3,8)
    int4range(1, 10) - int4range(3, 5) AS diff_range;      -- 报错！差集可能不连续

-- 日期范围示例
SELECT
    daterange('2024-01-01', '2024-06-30') &&
    daterange('2024-03-01', '2024-09-30') AS date_overlap,  -- true
    daterange('2024-01-01', '2024-06-30') *
    daterange('2024-03-01', '2024-09-30') AS overlap_period; -- [2024-03-01,2024-06-30)
```

### 示例3：会议室预订系统（排他约束）

```sql
-- 安装 btree_gist 扩展（排他约束中混合使用范围和等值比较需要）
CREATE EXTENSION IF NOT EXISTS btree_gist;

-- 创建会议室预订表
CREATE TABLE room_bookings (
    id SERIAL PRIMARY KEY,
    room_id INTEGER NOT NULL,
    booked_by TEXT NOT NULL,
    booking_period tsrange NOT NULL,
    purpose TEXT,

    -- 排他约束：同一会议室的预订时间不能重叠
    CONSTRAINT no_overlapping_bookings
        EXCLUDE USING gist (
            room_id WITH =,
            booking_period WITH &&
        )
);

-- 正常预订
INSERT INTO room_bookings (room_id, booked_by, booking_period, purpose) VALUES
(1, '张三', tsrange('2024-03-15 09:00', '2024-03-15 10:00'), '项目会议'),
(1, '李四', tsrange('2024-03-15 10:00', '2024-03-15 11:30'), '需求评审'),
(2, '王五', tsrange('2024-03-15 09:00', '2024-03-15 11:00'), '培训');

-- 尝试重叠预订 — 将被约束拒绝！
INSERT INTO room_bookings (room_id, booked_by, booking_period, purpose) VALUES
(1, '赵六', tsrange('2024-03-15 09:30', '2024-03-15 10:30'), '临时会议');
-- ERROR: conflicting key value violates exclusion constraint "no_overlapping_bookings"

-- 不同会议室同一时间 — 允许
INSERT INTO room_bookings (room_id, booked_by, booking_period, purpose) VALUES
(2, '赵六', tsrange('2024-03-15 11:00', '2024-03-15 12:00'), '临时会议');

-- 查询某个时间段可用的会议室
SELECT DISTINCT room_id
FROM (VALUES (1), (2), (3), (4), (5)) AS rooms(room_id)
WHERE room_id NOT IN (
    SELECT room_id FROM room_bookings
    WHERE booking_period && tsrange('2024-03-15 10:00', '2024-03-15 11:00')
);
```

### 示例4：价格区间与范围索引

```sql
-- 商品价格区间分类
CREATE TABLE price_tiers (
    id SERIAL PRIMARY KEY,
    tier_name TEXT NOT NULL,
    price_range numrange NOT NULL,
    discount_rate NUMERIC(3,2) NOT NULL
);

INSERT INTO price_tiers (tier_name, price_range, discount_rate) VALUES
('经济型', numrange(0, 100), 0.00),
('标准型', numrange(100, 500), 0.05),
('高端型', numrange(500, 2000), 0.10),
('奢侈型', numrange(2000, NULL), 0.15);

-- 创建 GiST 索引
CREATE INDEX idx_price_tiers_range ON price_tiers USING gist(price_range);

-- 查询某个价格属于哪个等级
SELECT tier_name, discount_rate
FROM price_tiers
WHERE price_range @> 350::numeric;
-- 结果：标准型, 0.05

-- 确保价格区间不重叠
ALTER TABLE price_tiers
ADD CONSTRAINT no_overlapping_tiers
    EXCLUDE USING gist (price_range WITH &&);

-- 查看执行计划，确认使用了 GiST 索引
EXPLAIN ANALYZE
SELECT * FROM price_tiers
WHERE price_range @> 1500::numeric;
```

### 示例5：员工排班系统

```sql
-- 排班表
CREATE TABLE employee_schedules (
    id SERIAL PRIMARY KEY,
    employee_id INTEGER NOT NULL,
    work_date DATE NOT NULL,
    shift_time tsrange NOT NULL,
    shift_type TEXT CHECK (shift_type IN ('早班', '中班', '晚班')),

    -- 同一员工同一天不能有重叠班次
    CONSTRAINT no_overlapping_shifts
        EXCLUDE USING gist (
            employee_id WITH =,
            shift_time WITH &&
        )
);

-- 插入排班数据
INSERT INTO employee_schedules (employee_id, work_date, shift_time, shift_type) VALUES
(1, '2024-03-15', tsrange('2024-03-15 06:00', '2024-03-15 14:00'), '早班'),
(1, '2024-03-15', tsrange('2024-03-15 14:00', '2024-03-15 22:00'), '中班'),
(2, '2024-03-15', tsrange('2024-03-15 06:00', '2024-03-15 14:00'), '早班');

-- 查询某个时间点谁在上班
SELECT employee_id, shift_type
FROM employee_schedules
WHERE shift_time @> '2024-03-15 10:00'::timestamp;

-- 查询某员工某天的排班
SELECT shift_type, shift_time
FROM employee_schedules
WHERE employee_id = 1
  AND shift_time && tsrange('2024-03-15'::timestamp, '2024-03-16'::timestamp);
```

### 示例6：自定义范围类型

```sql
-- 创建自定义范围类型：浮点数范围
CREATE TYPE floatrange AS RANGE (
    subtype = float8,
    subtype_diff = float8mi  -- 用于 GiST 索引优化
);

-- 使用自定义范围类型
CREATE TABLE sensor_thresholds (
    sensor_id INTEGER PRIMARY KEY,
    normal_range floatrange,
    warning_range floatrange,
    critical_range floatrange
);

INSERT INTO sensor_thresholds VALUES
(1, floatrange(20.0, 30.0), floatrange(15.0, 35.0), floatrange(10.0, 40.0));

-- 检查传感器读数是否在正常范围
SELECT
    sensor_id,
    normal_range @> 25.5 AS is_normal,
    normal_range @> 33.0 AS is_normal_high
FROM sensor_thresholds
WHERE sensor_id = 1;
```

## 常见问题

### Q1: 排他约束和唯一约束有什么区别？

**A:** 唯一约束（UNIQUE）是排他约束的一个特例。唯一约束确保指定列的值不重复，使用等值（`=`）比较。排他约束更通用，可以使用任意操作符（如 `&&` 重叠检查、`=` 等值检查）的组合来确保约束条件。排他约束使用 GiST 索引实现，而唯一约束使用 B-tree 索引。排他约束的典型用例是防止范围重叠，这是唯一约束无法实现的。

### Q2: 为什么需要 btree_gist 扩展？

**A:** 排他约束底层使用 GiST 索引。当排他约束中混合使用范围操作符（如 `&&`）和等值操作符（如 `=`）时，等值比较的列也需要支持 GiST 索引。但 integer、text 等标量类型默认只有 B-tree 索引支持。`btree_gist` 扩展为这些标量类型提供了 GiST 索引支持，使得它们可以在排他约束中与范围类型一起使用。

### Q3: 范围类型的 GiST 索引和 B-tree 索引怎么选？

**A:** 范围类型同时支持 GiST 和 B-tree 索引，但用途不同：
- **GiST 索引**：支持 `@>`、`<@`、`&&`、`<<`、`>>` 等范围操作符，是范围查询的首选。
- **B-tree 索引**：只支持 `=`、`<`、`>` 等比较操作，不支持包含和重叠查询。
绝大多数使用范围类型的场景应该选择 GiST 索引。
