## 理论讲解

### 什么是 PostgreSQL 函数？

PostgreSQL 函数（也称为用户自定义函数，UDF）是一段可以被重复调用的数据库逻辑。函数接收输入参数，执行一系列操作，并返回结果。与应用层代码不同，函数在数据库内部执行，可以减少网络传输、提高性能，并集中管理业务逻辑。

PostgreSQL 支持多种语言编写函数：SQL、PL/pgSQL、PL/Python、PL/Perl、C 等。本节重点介绍最基础的 **SQL 语言函数**。

### CREATE FUNCTION 语法

```sql
CREATE [ OR REPLACE ] FUNCTION function_name (
    [ [ argmode ] [ argname ] argtype [ { DEFAULT | = } default_expr ] [, ...] ]
)
RETURNS rettype
AS $$
    -- 函数体（SQL语句）
$$
LANGUAGE lang_name
[ IMMUTABLE | STABLE | VOLATILE ]
[ STRICT | CALLED ON NULL INPUT ]
[ SECURITY DEFINER | SECURITY INVOKER ]
[ PARALLEL { UNSAFE | RESTRICTED | SAFE } ]
[ COST execution_cost ]
[ ROWS result_rows ];
```

**关键子句说明：**

| 子句 | 说明 |
|------|------|
| `OR REPLACE` | 如果函数已存在则替换，避免先 DROP 再 CREATE |
| `RETURNS` | 指定返回类型，可以是标量、复合类型、SETOF 等 |
| `LANGUAGE` | 指定函数使用的语言，如 `sql`、`plpgsql` |
| `IMMUTABLE` | 相同输入永远返回相同输出（可被优化器缓存） |
| `STABLE` | 在单个查询内，相同输入返回相同输出 |
| `VOLATILE` | 默认值，每次调用可能返回不同结果 |
| `STRICT` | 任何参数为 NULL 时直接返回 NULL，不执行函数体 |
| `SECURITY DEFINER` | 以函数创建者权限执行 |
| `SECURITY INVOKER` | 以调用者权限执行（默认） |

### 参数模式详解

PostgreSQL 函数支持四种参数模式：

1. **IN（默认）**：输入参数，调用时传入值，函数内部只读。
2. **OUT**：输出参数，不在调用时传入，用于返回额外结果。
3. **INOUT**：既是输入又是输出，调用时传入值，函数结束后返回修改后的值。
4. **VARIADIC**：可变参数，允许传入任意数量的同类型参数，在函数内部作为数组处理。

### 函数重载

PostgreSQL 允许同名函数拥有不同的参数列表（参数数量或类型不同），这就是**函数重载**。数据库根据调用时传入的参数自动匹配最合适的函数版本。

注意事项：
- 仅返回类型不同不构成重载
- 过度使用重载可能导致歧义调用，应谨慎使用
- 使用 `DROP FUNCTION` 时需要指明参数列表以区分重载版本

## 代码示例

### 示例 1：最简单的 SQL 函数

```sql
-- 创建一个计算两数之和的函数
CREATE OR REPLACE FUNCTION add_numbers(a INTEGER, b INTEGER)
RETURNS INTEGER
AS $$
    SELECT a + b;
$$
LANGUAGE sql
IMMUTABLE;

-- 调用函数
SELECT add_numbers(3, 5);  -- 返回 8

-- 在查询中使用函数
SELECT product_name, price, add_numbers(price::int, 10) AS adjusted_price
FROM products;
```

### 示例 2：返回表行的函数

```sql
-- 创建一个根据薪资范围查询员工的函数
CREATE OR REPLACE FUNCTION get_employees_by_salary(min_sal NUMERIC, max_sal NUMERIC)
RETURNS SETOF employees
AS $$
    SELECT *
    FROM employees
    WHERE salary BETWEEN min_sal AND max_sal
    ORDER BY salary DESC;
$$
LANGUAGE sql
STABLE;

-- 调用函数，结果可当作表使用
SELECT employee_id, first_name, salary
FROM get_employees_by_salary(5000, 10000);
```

### 示例 3：使用 OUT 参数

```sql
-- 使用 OUT 参数返回多个值
CREATE OR REPLACE FUNCTION get_salary_stats(
    IN  dept_id    INTEGER,
    OUT min_salary NUMERIC,
    OUT max_salary NUMERIC,
    OUT avg_salary NUMERIC
)
AS $$
    SELECT MIN(salary), MAX(salary), AVG(salary)
    FROM employees
    WHERE department_id = dept_id;
$$
LANGUAGE sql
STABLE;

-- 调用：OUT 参数自动作为返回列
SELECT * FROM get_salary_stats(10);
-- 结果示例：min_salary | max_salary | avg_salary
--           3000.00   | 12000.00   | 7500.00
```

### 示例 4：使用 INOUT 参数

```sql
-- INOUT 参数示例：对价格打折
CREATE OR REPLACE FUNCTION apply_discount(INOUT price NUMERIC, discount_rate NUMERIC)
AS $$
    SELECT price * (1 - discount_rate);
$$
LANGUAGE sql
IMMUTABLE;

-- 调用
SELECT apply_discount(100.00, 0.15);  -- 返回 85.00
```

### 示例 5：VARIADIC 可变参数

```sql
-- 接受任意数量整数并返回总和
CREATE OR REPLACE FUNCTION sum_all(VARIADIC nums INTEGER[])
RETURNS INTEGER
AS $$
    SELECT COALESCE(SUM(n), 0)::INTEGER
    FROM unnest(nums) AS n;
$$
LANGUAGE sql
IMMUTABLE;

-- 调用时传入任意数量的参数
SELECT sum_all(1, 2, 3);          -- 返回 6
SELECT sum_all(10, 20, 30, 40);   -- 返回 100

-- 也可以用 VARIADIC 关键字传入数组
SELECT sum_all(VARIADIC ARRAY[5, 10, 15]);  -- 返回 30
```

### 示例 6：函数重载

```sql
-- 版本1：两个整数相加
CREATE OR REPLACE FUNCTION calculate(a INTEGER, b INTEGER)
RETURNS INTEGER
AS $$
    SELECT a + b;
$$
LANGUAGE sql IMMUTABLE;

-- 版本2：两个浮点数相加（重载）
CREATE OR REPLACE FUNCTION calculate(a NUMERIC, b NUMERIC)
RETURNS NUMERIC
AS $$
    SELECT a + b;
$$
LANGUAGE sql IMMUTABLE;

-- 版本3：三个整数相加（重载）
CREATE OR REPLACE FUNCTION calculate(a INTEGER, b INTEGER, c INTEGER)
RETURNS INTEGER
AS $$
    SELECT a + b + c;
$$
LANGUAGE sql IMMUTABLE;

-- 调用不同版本
SELECT calculate(1, 2);           -- 调用版本1，返回 3
SELECT calculate(1.5, 2.5);       -- 调用版本2，返回 4.0
SELECT calculate(1, 2, 3);        -- 调用版本3，返回 6
```

### 示例 7：带默认值的参数

```sql
-- 带默认参数的分页查询函数
CREATE OR REPLACE FUNCTION get_recent_orders(
    p_limit  INTEGER DEFAULT 10,
    p_offset INTEGER DEFAULT 0
)
RETURNS SETOF orders
AS $$
    SELECT *
    FROM orders
    ORDER BY order_date DESC
    LIMIT p_limit OFFSET p_offset;
$$
LANGUAGE sql
STABLE;

-- 使用默认值调用
SELECT * FROM get_recent_orders();            -- 默认前10条
SELECT * FROM get_recent_orders(5);           -- 前5条
SELECT * FROM get_recent_orders(5, 10);       -- 跳过10条取5条
-- 命名参数调用
SELECT * FROM get_recent_orders(p_offset => 20);
```

## 常见问题

### Q1：SQL 函数和 PL/pgSQL 函数有什么区别？该如何选择？

SQL 函数更简洁，只能包含 SQL 语句，适用于简单的查询和计算。优势在于优化器可以将 SQL 函数"内联"到调用查询中进行统一优化。PL/pgSQL 函数支持流程控制（条件、循环、异常处理），适用于复杂业务逻辑。

**选择原则：** 如果一条 SQL 能完成的事情，就用 SQL 函数；需要流程控制时再用 PL/pgSQL。

### Q2：IMMUTABLE、STABLE、VOLATILE 分别在什么场景使用？

- **IMMUTABLE**：纯计算函数，如数学运算、字符串拼接。允许在索引表达式中使用。
- **STABLE**：读取数据库数据但在单次查询中结果不变的函数，如查询配置表。
- **VOLATILE**：会修改数据或依赖外部状态的函数，如使用 `random()`、`now()`、插入数据。

错误标记可能导致查询结果不正确或索引失效，务必准确设置。

### Q3：删除重载函数时报错怎么办？

删除重载函数必须指明完整参数列表：

```sql
-- 错误：有多个同名函数时会报错
DROP FUNCTION calculate;

-- 正确：指定参数类型
DROP FUNCTION calculate(INTEGER, INTEGER);
DROP FUNCTION calculate(NUMERIC, NUMERIC);
DROP FUNCTION calculate(INTEGER, INTEGER, INTEGER);
```
