## 理论讲解

### 1. 数值类型概览

PostgreSQL 的数值类型分为三大类：

| 类别 | 类型 | 存储大小 | 范围/精度 |
|------|------|---------|----------|
| 整数 | smallint (int2) | 2 字节 | -32,768 到 32,767 |
| 整数 | integer (int4) | 4 字节 | -2,147,483,648 到 2,147,483,647 |
| 整数 | bigint (int8) | 8 字节 | -9,223,372,036,854,775,808 到 9,223,372,036,854,775,807 |
| 精确数值 | numeric(p,s) | 可变 | 最大 131072 位整数 + 16383 位小数 |
| 精确数值 | decimal(p,s) | 可变 | 与 numeric 完全相同 |
| 浮点数 | real (float4) | 4 字节 | 6 位十进制精度 |
| 浮点数 | double precision (float8) | 8 字节 | 15 位十进制精度 |

### 2. 整数类型选择原则

- **smallint**：适用于值范围确定较小的场景，如年龄、评分（1-5）、月份
- **integer**：最常用的整数类型，适合大多数场景（主键、计数器等）
- **bigint**：当 integer 范围不够时使用，如全局唯一 ID、大型系统的计数

**存储空间对性能的影响**：较小的类型不仅节省磁盘空间，还能让更多数据放入内存缓存，提升查询性能。在数据量大的表中，从 bigint 改为 integer 可能显著减少存储。

### 3. 精确数值 vs 浮点数

这是非常重要的区分：

**numeric/decimal（精确）：**
- 存储的是精确值，不存在精度丢失
- 适用于金融、货币计算
- 运算速度比浮点数慢
- `numeric(10,2)` 表示最多 10 位数字，其中 2 位小数

**real/double precision（近似）：**
- 使用 IEEE 754 浮点数标准
- 存在精度丢失（0.1 + 0.2 != 0.3）
- 运算速度快
- 适用于科学计算、地理坐标等容许微小误差的场景

### 4. 字符串类型

| 类型 | 说明 | 最大长度 | 尾部空格 |
|------|------|---------|---------|
| char(n) / character(n) | 固定长度 | n 个字符 | 自动补空格 |
| varchar(n) / character varying(n) | 可变长度，有上限 | n 个字符 | 不补空格 |
| varchar / character varying | 可变长度，无上限 | 无限制 | 不补空格 |
| text | 可变长度，无上限 | 无限制 | 不补空格 |

**PostgreSQL 中的最佳实践**：

在 PostgreSQL 中，`text` 和 `varchar` 的底层实现完全相同，性能没有差别。`char(n)` 因为要补空格反而会浪费存储空间。PostgreSQL 核心团队的建议是：

> 在大多数情况下，使用 `text` 或 `varchar`。只在需要数据库层面强制长度限制时使用 `varchar(n)`。避免使用 `char(n)`。

## 代码示例

### 整数类型实践

```sql
-- 创建演示表
CREATE TABLE numeric_demo (
    id          integer PRIMARY KEY,
    age         smallint,           -- 年龄用 smallint 足够
    population  bigint,             -- 人口数可能超过 integer 范围
    quantity    integer DEFAULT 0    -- 常规数量用 integer
);

-- 插入数据
INSERT INTO numeric_demo VALUES (1, 28, 1400000000, 500);
INSERT INTO numeric_demo VALUES (2, 35, 330000000, 1200);

-- 查看类型信息
SELECT column_name, data_type, numeric_precision
FROM information_schema.columns
WHERE table_name = 'numeric_demo';

-- 整数溢出演示
-- 以下会报错：smallint 最大值是 32767
INSERT INTO numeric_demo (id, age) VALUES (3, 40000);
-- ERROR: smallint out of range

-- 整数运算
SELECT
    id,
    quantity,
    quantity * 2 AS doubled,
    quantity / 3 AS int_division,      -- 整数除法，结果截断
    quantity / 3.0 AS float_division,  -- 除以浮点数，结果保留小数
    quantity % 7 AS remainder          -- 取模
FROM numeric_demo;
```

### 精确数值类型实践

```sql
-- 金融计算：必须使用 numeric
CREATE TABLE accounts (
    id       serial PRIMARY KEY,
    name     text NOT NULL,
    balance  numeric(15, 2) NOT NULL DEFAULT 0.00,  -- 最大 13 位整数 + 2 位小数
    rate     numeric(5, 4)                            -- 利率：0.0000 - 9.9999
);

INSERT INTO accounts (name, balance, rate) VALUES
    ('张三', 100000.50, 0.0385),
    ('李四', 250000.00, 0.0425),
    ('王五', 50000.75, 0.0350);

-- 精确计算利息
SELECT
    name,
    balance,
    rate,
    ROUND(balance * rate, 2) AS annual_interest,
    ROUND(balance * rate / 12, 2) AS monthly_interest
FROM accounts;

-- 浮点数精度问题演示
SELECT 0.1::real + 0.2::real AS float_result;
-- 结果：0.30000001192092896（不精确！）

SELECT 0.1::numeric + 0.2::numeric AS numeric_result;
-- 结果：0.3（精确）

-- 更明显的精度差异
SELECT
    (0.1::double precision + 0.2::double precision) = 0.3::double precision AS float_equal,
    (0.1::numeric + 0.2::numeric) = 0.3::numeric AS numeric_equal;
-- 结果：float_equal = false, numeric_equal = true

-- numeric 不指定精度时可以存储任意精度
SELECT 123456789.123456789::numeric;  -- 保留完整精度
SELECT 123456789.123456789::numeric(12,2);  -- 截断为 123456789.12
```

### 字符串类型实践

```sql
-- 创建演示表，对比三种字符串类型
CREATE TABLE string_demo (
    id         serial PRIMARY KEY,
    code       char(10),      -- 固定长度
    name       varchar(50),   -- 有长度限制
    bio        text           -- 无长度限制
);

INSERT INTO string_demo (code, name, bio) VALUES
    ('PRD001', '无线蓝牙耳机', '这是一款高品质的蓝牙耳机，支持主动降噪功能。'),
    ('PRD002', '机械键盘', '采用 Cherry MX 轴体的全尺寸机械键盘。');

-- 观察 char 的补空格行为
SELECT
    code,
    length(code) AS code_len,          -- 逻辑长度（去除尾部空格）
    octet_length(code) AS code_bytes,  -- 实际字节长度（包含补的空格）
    '|' || code || '|' AS padded       -- 可视化看到空格
FROM string_demo;

-- varchar 超长测试
INSERT INTO string_demo (name) VALUES ('这个名字特别特别特别长超过了五十个字符限制这个名字特别特别特别长超过了五十个字符限制');
-- ERROR: value too long for type character varying(50)

-- text 可以存储超长内容
INSERT INTO string_demo (bio) VALUES (repeat('PostgreSQL很强大！', 1000));
SELECT id, length(bio) FROM string_demo WHERE id = 3;
```

### 常用字符串函数

```sql
-- 字符串长度
SELECT length('Hello PostgreSQL');       -- 16
SELECT char_length('你好世界');           -- 4（字符数）
SELECT octet_length('你好世界');          -- 12（UTF-8 字节数）

-- 大小写转换
SELECT upper('hello');     -- HELLO
SELECT lower('HELLO');     -- hello
SELECT initcap('hello world postgresql');  -- Hello World Postgresql

-- 截取与拼接
SELECT substring('PostgreSQL' FROM 1 FOR 4);  -- Post
SELECT left('PostgreSQL', 4);                  -- Post
SELECT right('PostgreSQL', 3);                 -- SQL
SELECT 'Hello' || ' ' || 'World';             -- Hello World
SELECT concat('Hello', ' ', 'World');          -- Hello World
SELECT concat_ws('-', '2025', '01', '15');     -- 2025-01-15

-- 查找与替换
SELECT position('SQL' IN 'PostgreSQL');      -- 8
SELECT strpos('PostgreSQL', 'SQL');           -- 8
SELECT replace('Hello World', 'World', 'PG'); -- Hello PG
SELECT translate('Hello', 'Helo', 'Xyzw');   -- Xyzww

-- 填充与修剪
SELECT lpad('42', 5, '0');     -- 00042
SELECT rpad('Hi', 10, '.');   -- Hi........
SELECT trim('  hello  ');      -- hello
SELECT ltrim('  hello  ');     -- 'hello  '
SELECT rtrim('  hello  ');     -- '  hello'
SELECT trim(both 'x' from 'xxxhelloxxx');  -- hello

-- 正则表达式
SELECT 'Hello123' ~ '[0-9]+';              -- true（包含数字）
SELECT regexp_replace('a1b2c3', '[0-9]', 'X', 'g');  -- aXbXcX
SELECT regexp_matches('abc123def456', '[0-9]+', 'g'); -- {123}, {456}
SELECT regexp_split_to_array('one,two,three', ',');   -- {one,two,three}
```

### 类型转换

```sql
-- CAST 语法（SQL 标准）
SELECT CAST('42' AS integer);
SELECT CAST(3.14 AS text);
SELECT CAST('2025-01-15' AS date);

-- :: 语法（PostgreSQL 特有，更简洁）
SELECT '42'::integer;
SELECT 3.14::text;
SELECT '2025-01-15'::date;

-- 隐式转换与显式转换
SELECT 1 + '2';          -- 隐式转换，结果为 3
SELECT 1 || '2';         -- 报错！integer 不能隐式转换为 text 做拼接
SELECT 1::text || '2';   -- 显式转换，结果为 '12'

-- 数值转字符串格式化
SELECT to_char(1234567.89, '9,999,999.99');  -- ' 1,234,567.89'
SELECT to_char(1234567.89, 'FM9,999,999.99'); -- '1,234,567.89'（FM去前导空格）
SELECT to_char(42, '000');   -- ' 042'（前导零）
```

## 常见问题

### Q1: varchar(n) 和 text 性能一样吗？应该用哪个？

**A:** 在 PostgreSQL 中，`varchar(n)`、`varchar`（不带长度）和 `text` 三者的底层存储机制完全相同，性能没有任何差别。`varchar(n)` 只是多了一个长度检查约束。

建议：
- 如果需要数据库层面的长度约束，用 `varchar(n)`
- 如果不需要长度约束，直接用 `text`
- 不要使用 `char(n)`，它既浪费空间又容易引起尾部空格问题

### Q2: 为什么金融系统不能用 float/double？

**A:** 因为浮点数使用二进制表示，无法精确存储某些十进制小数：
```sql
SELECT 0.1::double precision + 0.2::double precision;
-- 结果：0.30000000000000004
```
在金融场景中，即使是 0.01 元的误差累积起来也可能造成严重问题。因此必须使用 `numeric` 或 `decimal` 类型，它们以十进制方式精确存储数值。

### Q3: numeric 不指定精度和指定精度有什么区别？

**A:**
- `numeric(p,s)`：精度受限，超出会四舍五入或报错。存储空间与实际值有关。
- `numeric`（不指定）：可以存储任意精度的数值，最多 131072 位整数部分和 16383 位小数部分。
- 不指定精度更灵活，但指定精度有助于数据校验（如确保金额只有两位小数）。
