# Day 024: 日期时间函数

## 学习目标

- 掌握 PostgreSQL 获取当前日期时间的函数：now()、current_date、current_time、current_timestamp
- 熟练使用 age()、extract()、date_trunc() 进行日期计算与截断
- 理解 interval 类型及其运算规则
- 掌握 to_char() 日期格式化以及 to_date()/to_timestamp() 解析函数
- 能够在实际业务场景中灵活运用日期时间函数

## 理论讲解

### PostgreSQL 的日期时间类型

| 类型 | 存储大小 | 说明 | 示例 |
|------|----------|------|------|
| `date` | 4 字节 | 仅日期 | 2025-06-15 |
| `time` | 8 字节 | 仅时间（无时区） | 14:30:00 |
| `timetz` | 12 字节 | 时间（含时区） | 14:30:00+08 |
| `timestamp` | 8 字节 | 日期+时间（无时区） | 2025-06-15 14:30:00 |
| `timestamptz` | 8 字节 | 日期+时间（含时区） | 2025-06-15 14:30:00+08 |
| `interval` | 16 字节 | 时间间隔 | 1 year 2 months 3 days |

### 获取当前时间

```sql
now()               -- 当前时间戳（含时区），等同于 current_timestamp
current_date         -- 当前日期
current_time         -- 当前时间（含时区）
current_timestamp    -- 当前时间戳（含时区）
localtime            -- 当前本地时间（不含时区）
localtimestamp        -- 当前本地时间戳（不含时区）
clock_timestamp()    -- 实际的实时时间（每次调用都可能不同）
```

> **注意**：`now()` 在同一事务中返回值相同，而 `clock_timestamp()` 每次调用都返回当前实时时间。

### age() 函数

```sql
age(timestamp, timestamp)  -- 计算两个时间之间的差值
age(timestamp)              -- 计算从参数到当前时间的差值
```

返回值是 `interval` 类型，以年、月、日表示。

### extract() 函数

从日期时间值中提取指定字段：

```sql
extract(field FROM source)
-- field 可以是：year, month, day, hour, minute, second,
--              dow(周几0-6), doy(一年中第几天), week, quarter, epoch
```

### date_trunc() 函数

将日期时间截断到指定精度：

```sql
date_trunc('field', source)
-- field: microseconds, milliseconds, second, minute, hour,
--        day, week, month, quarter, year, decade, century, millennium
```

### interval 运算

PostgreSQL 允许对日期时间直接进行算术运算：

```sql
date + interval           -- 日期加间隔
date - interval           -- 日期减间隔
date - date               -- 两个日期相减得到整数（天数）
timestamp - timestamp     -- 两个时间戳相减得到 interval
interval * number         -- 间隔乘以数字
interval / number         -- 间隔除以数字
```

### 格式化函数

- `to_char(datetime, format)` — 将日期时间转为格式化字符串
- `to_date(text, format)` — 将字符串解析为日期
- `to_timestamp(text, format)` — 将字符串解析为时间戳

常用格式化模式：

| 模式 | 说明 | 示例 |
|------|------|------|
| `YYYY` | 四位年份 | 2025 |
| `MM` | 月份(01-12) | 06 |
| `DD` | 日期(01-31) | 15 |
| `HH24` | 24小时制小时 | 14 |
| `MI` | 分钟 | 30 |
| `SS` | 秒 | 00 |
| `Day` | 星期全称 | Monday |
| `Mon` | 月份缩写 | Jun |
| `Q` | 季度 | 2 |

### 示例一：获取当前时间

```sql
SELECT
    now()             AS current_ts,
    current_date      AS today,
    current_time      AS time_now,
    localtimestamp     AS local_ts;
```

### 示例二：age() 计算年龄和工龄

```sql
SELECT
    name,
    birth_date,
    age(current_date, birth_date) AS age_interval,
    -- 提取年份部分作为年龄
    extract(YEAR FROM age(current_date, birth_date)) AS age_years,
    hire_date,
    age(current_date, hire_date) AS tenure,
    extract(YEAR FROM age(current_date, hire_date)) AS tenure_years
FROM employees;
```

### 示例三：extract() 提取日期字段

```sql
SELECT
    name,
    hire_date,
    extract(YEAR FROM hire_date)    AS hire_year,
    extract(MONTH FROM hire_date)   AS hire_month,
    extract(DOW FROM hire_date)     AS hire_dow,       -- 0=周日
    extract(QUARTER FROM hire_date) AS hire_quarter,
    extract(DOY FROM hire_date)     AS day_of_year,
    -- epoch 转秒数
    extract(EPOCH FROM last_login)  AS login_epoch
FROM employees;
```

### 示例四：date_trunc() 截断

```sql
SELECT
    last_login,
    date_trunc('year', last_login)    AS trunc_year,    -- 截断到年初
    date_trunc('month', last_login)   AS trunc_month,   -- 截断到月初
    date_trunc('week', last_login)    AS trunc_week,    -- 截断到周一
    date_trunc('hour', last_login)    AS trunc_hour     -- 截断到整点
FROM employees;
```

### 示例五：interval 运算

```sql
-- 各种 interval 运算示例
SELECT
    current_date + INTERVAL '30 days'           AS thirty_days_later,
    current_date - INTERVAL '1 year 3 months'   AS past_date,
    INTERVAL '1 hour' * 24                      AS one_day,
    INTERVAL '1 year' / 12                      AS one_month_approx;

-- 计算员工入职 5 周年纪念日
SELECT
    name,
    hire_date,
    hire_date + INTERVAL '5 years' AS fifth_anniversary
FROM employees;

-- 计算距离下次生日还有多少天
SELECT
    name,
    birth_date,
    -- 今年生日的日期
    make_date(extract(YEAR FROM current_date)::INT,
              extract(MONTH FROM birth_date)::INT,
              extract(DAY FROM birth_date)::INT) AS this_year_birthday,
    CASE
        WHEN make_date(extract(YEAR FROM current_date)::INT,
                       extract(MONTH FROM birth_date)::INT,
                       extract(DAY FROM birth_date)::INT) >= current_date
        THEN make_date(extract(YEAR FROM current_date)::INT,
                       extract(MONTH FROM birth_date)::INT,
                       extract(DAY FROM birth_date)::INT) - current_date
        ELSE make_date(extract(YEAR FROM current_date)::INT + 1,
                       extract(MONTH FROM birth_date)::INT,
                       extract(DAY FROM birth_date)::INT) - current_date
    END AS days_until_birthday
FROM employees;
```

### 示例六：to_char() 格式化输出

```sql
SELECT
    name,
    to_char(hire_date, 'YYYY年MM月DD日')               AS hire_cn,
    to_char(hire_date, 'YYYY-MM-DD (Day)')             AS hire_with_day,
    to_char(last_login, 'YYYY-MM-DD HH24:MI:SS')      AS login_formatted,
    to_char(last_login, 'FMMonth DD, YYYY HH12:MI AM') AS login_us_format
FROM employees;
```

### 示例七：to_date() 和 to_timestamp() 解析

```sql
-- 将字符串解析为日期
SELECT to_date('2025/06/15', 'YYYY/MM/DD') AS parsed_date;
SELECT to_date('15-Jun-2025', 'DD-Mon-YYYY') AS parsed_date2;

-- 将字符串解析为时间戳
SELECT to_timestamp('2025-06-15 14:30:00', 'YYYY-MM-DD HH24:MI:SS') AS parsed_ts;
```

### 示例八：按月份分组统计

```sql
-- 按入职月份分组统计人数
SELECT
    to_char(hire_date, 'YYYY-MM') AS hire_month,
    COUNT(*) AS headcount
FROM employees
GROUP BY to_char(hire_date, 'YYYY-MM')
ORDER BY hire_month;

-- 等价写法：使用 date_trunc
SELECT
    date_trunc('month', hire_date)::DATE AS hire_month,
    COUNT(*) AS headcount
FROM employees
GROUP BY date_trunc('month', hire_date)
ORDER BY hire_month;
```

## 常见问题

### Q1：now() 和 clock_timestamp() 有什么区别？
**A**：`now()` 返回当前事务开始时的时间，在整个事务中保持不变。`clock_timestamp()` 返回调用时的实际时间，每次调用可能不同。在需要记录精确操作时间时使用 `clock_timestamp()`。

### Q2：timestamp 和 timestamptz 应该用哪个？
**A**：推荐使用 `timestamptz`（带时区的时间戳）。它在存储时统一转换为 UTC，查询时根据会话时区转换显示。这在多时区应用中尤为重要，可以避免时区混乱。

### Q3：两个日期相减为什么得到整数而不是 interval？
**A**：`date - date` 返回整数（天数），这是 PostgreSQL 的设计。如果需要 interval 结果，可以将日期转为 timestamp 后相减：`timestamp1 - timestamp2` 返回 interval。
