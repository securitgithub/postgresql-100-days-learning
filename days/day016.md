## 理论讲解

### 1. 什么是子查询？

子查询是嵌套在另一个查询中的 `SELECT` 语句，用括号包裹。子查询可以出现在 `SELECT`、`FROM`、`WHERE`、`HAVING` 等子句中。

按返回结果分类：
- **标量子查询**：返回单个值（一行一列）
- **行子查询**：返回一行多列
- **表子查询**：返回多行多列
- **列子查询**：返回一列多行

### 2. 标量子查询

返回单个值，可以用在任何需要单个值的地方：

```sql
-- 在 SELECT 中
SELECT name, price,
       (SELECT AVG(price) FROM products) AS avg_price
FROM products;

-- 在 WHERE 中
SELECT * FROM products
WHERE price > (SELECT AVG(price) FROM products);
```

如果标量子查询返回多行，PostgreSQL 会报错。

### 3. IN / NOT IN 子查询

`IN` 检查值是否在子查询返回的集合中：

```sql
SELECT * FROM customers
WHERE id IN (SELECT customer_id FROM orders);
```

**NOT IN 的 NULL 陷阱**：如果子查询结果中包含 NULL，`NOT IN` 将返回空结果集！因为任何值与 NULL 比较都是 UNKNOWN。

```sql
-- 危险！如果 customer_id 有 NULL 值，结果为空
SELECT * FROM customers
WHERE id NOT IN (SELECT customer_id FROM orders);

-- 安全写法：排除 NULL
SELECT * FROM customers
WHERE id NOT IN (SELECT customer_id FROM orders WHERE customer_id IS NOT NULL);
```

### 4. EXISTS / NOT EXISTS

`EXISTS` 检查子查询是否返回至少一行，返回布尔值：

```sql
-- 查找有订单的客户
SELECT * FROM customers c
WHERE EXISTS (
    SELECT 1 FROM orders o WHERE o.customer_id = c.id
);
```

`EXISTS` 的优点：
- 不受 NULL 影响
- 找到第一行匹配就停止扫描（短路求值）
- 子查询中 SELECT 的内容不重要（通常写 `SELECT 1`）

### 5. 相关子查询

相关子查询引用外层查询的列，对外层每一行都执行一次：

```sql
-- 查找每个类别中价格最高的商品
SELECT * FROM products p1
WHERE price = (
    SELECT MAX(price) FROM products p2
    WHERE p2.category = p1.category  -- 引用外层的 p1.category
);
```

相关子查询的执行逻辑：
1. 取外层查询的一行
2. 将该行的值代入子查询执行
3. 根据子查询结果决定该行是否保留
4. 重复以上步骤

### 6. FROM 子句中的子查询（派生表）

子查询可以作为虚拟表放在 `FROM` 中，必须指定别名：

```sql
SELECT category, avg_price
FROM (
    SELECT category, AVG(price) AS avg_price
    FROM products
    GROUP BY category
) AS category_stats
WHERE avg_price > 100;
```

### 7. 子查询优化

- PostgreSQL 优化器可能将 `IN` 子查询转换为 JOIN
- `EXISTS` 通常比 `IN` 更高效（尤其当子查询结果集很大时）
- 避免在 SELECT 列表中使用相关标量子查询（N+1 问题）
- 考虑用 JOIN 或 CTE 替代复杂子查询

### 示例 1：标量子查询

```sql
-- 查询高于平均价格的商品
SELECT name, price
FROM products
WHERE price > (SELECT AVG(price) FROM products);

-- 在 SELECT 中显示与平均价格的差值
SELECT
    name,
    price,
    price - (SELECT AVG(price) FROM products) AS diff_from_avg
FROM products
ORDER BY diff_from_avg DESC;
```

### 示例 2：IN 子查询

```sql
-- 查找下过订单的客户
SELECT name, email
FROM customers
WHERE id IN (SELECT DISTINCT customer_id FROM orders);

-- 查找购买过"书籍"类别商品的客户
SELECT name FROM customers
WHERE id IN (
    SELECT o.customer_id
    FROM orders o
    JOIN order_items oi ON o.id = oi.order_id
    JOIN products p ON oi.product_id = p.id
    WHERE p.category = '书籍'
);
```

### 示例 3：NOT IN（注意 NULL 陷阱）

```sql
-- 查找没有下过订单的客户（安全写法）
SELECT name FROM customers
WHERE id NOT IN (
    SELECT customer_id FROM orders
    WHERE customer_id IS NOT NULL
);
```

### 示例 4：EXISTS / NOT EXISTS

```sql
-- 查找有订单的客户（推荐写法）
SELECT c.name, c.email
FROM customers c
WHERE EXISTS (
    SELECT 1 FROM orders o WHERE o.customer_id = c.id
);

-- 查找没有订单的客户
SELECT c.name, c.email
FROM customers c
WHERE NOT EXISTS (
    SELECT 1 FROM orders o WHERE o.customer_id = c.id
);
```

### 示例 5：相关子查询

```sql
-- 查找每个部门薪资最高的员工
SELECT s1.name, s1.department, s1.salary
FROM staff s1
WHERE s1.salary = (
    SELECT MAX(s2.salary)
    FROM staff s2
    WHERE s2.department = s1.department
);

-- 查找薪资高于所在部门平均薪资的员工
SELECT s1.name, s1.department, s1.salary
FROM staff s1
WHERE s1.salary > (
    SELECT AVG(s2.salary)
    FROM staff s2
    WHERE s2.department = s1.department
);
```

### 示例 6：FROM 中的子查询（派生表）

```sql
-- 查询每个部门的平均薪资，并找出高于全公司平均的部门
SELECT dept_avg.department, dept_avg.avg_salary
FROM (
    SELECT department, AVG(salary) AS avg_salary
    FROM staff
    GROUP BY department
) AS dept_avg
WHERE dept_avg.avg_salary > (SELECT AVG(salary) FROM staff);
```

### 示例 7：子查询在 UPDATE 中的应用

```sql
-- 将薪资低于部门平均值的员工加薪 10%
UPDATE staff s1
SET salary = salary * 1.1
WHERE salary < (
    SELECT AVG(s2.salary)
    FROM staff s2
    WHERE s2.department = s1.department
)
RETURNING name, department, salary;
```

### 示例 8：用 JOIN 改写子查询

```sql
-- 子查询写法：查找有订单的客户
SELECT name FROM customers
WHERE id IN (SELECT customer_id FROM orders);

-- 等价的 JOIN 写法（通常性能更好）
SELECT DISTINCT c.name
FROM customers c
JOIN orders o ON c.id = o.customer_id;

-- 子查询写法：查找没有订单的客户
SELECT name FROM customers
WHERE NOT EXISTS (SELECT 1 FROM orders o WHERE o.customer_id = customers.id);

-- 等价的 LEFT JOIN 写法
SELECT c.name
FROM customers c
LEFT JOIN orders o ON c.id = o.customer_id
WHERE o.id IS NULL;
```

## 常见问题

### Q1: IN 和 EXISTS 有什么区别？该用哪个？

**A**：语义上都可以实现"判断是否存在匹配"。关键区别：
- `IN` 先执行子查询得到完整结果集，再逐一比较
- `EXISTS` 找到第一个匹配就短路返回，不需要完整扫描
- 当子查询结果集很大时，`EXISTS` 通常更快
- 当外层表很大、子查询结果集很小时，`IN` 可能更快
- PostgreSQL 优化器通常能互相转换，性能差异不大。但考虑到 NULL 安全性，推荐优先使用 `EXISTS`。

### Q2: 相关子查询是不是很慢？

**A**：理论上相关子查询对外层每一行都执行一次子查询，看似 O(M×N)。但 PostgreSQL 优化器很智能，可能将其转换为 JOIN 或使用哈希等策略。不要过早优化——先用 `EXPLAIN ANALYZE` 查看实际执行计划，再决定是否改写。

### Q3: 子查询可以嵌套多少层？

**A**：PostgreSQL 没有硬性的嵌套层数限制，但过深的嵌套（超过 3-4 层）会导致代码难以阅读和维护。建议使用 CTE（`WITH` 子句）将复杂子查询拆分为命名步骤，提高可读性。
