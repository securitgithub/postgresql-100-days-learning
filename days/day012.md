## 理论讲解

### 1. 聚合函数概述

聚合函数对一组行进行计算，返回单个结果值。常用聚合函数：

| 函数 | 说明 |
|------|------|
| `COUNT(*)` | 计算行数（包括 NULL） |
| `COUNT(列名)` | 计算该列非 NULL 的行数 |
| `SUM(列名)` | 求和（忽略 NULL） |
| `AVG(列名)` | 求平均值（忽略 NULL） |
| `MIN(列名)` | 求最小值 |
| `MAX(列名)` | 求最大值 |

**重要特性**：所有聚合函数（除 `COUNT(*)`）都会忽略 NULL 值。

### 2. GROUP BY 分组

`GROUP BY` 将结果集按照一个或多个列分组，每组返回一行汇总数据。

```sql
SELECT 分组列, 聚合函数(列)
FROM 表名
WHERE 条件
GROUP BY 分组列;
```

**规则**：`SELECT` 中的列要么出现在 `GROUP BY` 中，要么被聚合函数包裹。

### 3. HAVING 过滤分组

`WHERE` 在分组前过滤行，`HAVING` 在分组后过滤组。

```sql
SELECT category, COUNT(*) AS cnt
FROM products
GROUP BY category
HAVING COUNT(*) > 2;
```

**执行顺序**：`FROM` → `WHERE` → `GROUP BY` → `HAVING` → `SELECT` → `ORDER BY`

### 4. DISTINCT 与聚合

在聚合函数内使用 `DISTINCT` 可以去除重复值后再计算：

```sql
-- 计算不同类别的数量
SELECT COUNT(DISTINCT category) FROM products;
```

### 5. FILTER 子句（PostgreSQL 扩展）

`FILTER` 子句允许为聚合函数指定额外条件，比 `CASE WHEN` 写法更简洁：

```sql
SELECT
    COUNT(*) AS total,
    COUNT(*) FILTER (WHERE price > 100) AS expensive_count,
    AVG(price) FILTER (WHERE category = '书籍') AS avg_book_price
FROM products;
```

等价的 `CASE WHEN` 写法：

```sql
SELECT
    COUNT(*) AS total,
    COUNT(CASE WHEN price > 100 THEN 1 END) AS expensive_count,
    AVG(CASE WHEN category = '书籍' THEN price END) AS avg_book_price
FROM products;
```

`FILTER` 更加清晰、可读性更好，且是 SQL 标准的一部分。

### 6. 多列分组

可以按照多个列进行分组，形成更细粒度的汇总：

```sql
SELECT category, EXTRACT(YEAR FROM created_at) AS year, SUM(price)
FROM products
GROUP BY category, EXTRACT(YEAR FROM created_at);
```

### 示例 1：基本聚合函数

```sql
-- 统计总行数
SELECT COUNT(*) AS total_records FROM sales;

-- 求总销售金额（amount * quantity）
SELECT SUM(amount * quantity) AS total_revenue FROM sales;

-- 求平均单价
SELECT AVG(amount) AS avg_price FROM sales;

-- 求最高和最低单价
SELECT MIN(amount) AS min_price, MAX(amount) AS max_price FROM sales;
```

### 示例 2：GROUP BY 分组统计

```sql
-- 按类别统计销售数量和总金额
SELECT
    category,
    COUNT(*) AS order_count,
    SUM(quantity) AS total_qty,
    SUM(amount * quantity) AS total_revenue
FROM sales
GROUP BY category
ORDER BY total_revenue DESC;
```

### 示例 3：多列分组

```sql
-- 按类别和地区统计
SELECT
    category,
    region,
    COUNT(*) AS order_count,
    SUM(amount * quantity) AS total_revenue
FROM sales
GROUP BY category, region
ORDER BY category, total_revenue DESC;
```

### 示例 4：HAVING 过滤

```sql
-- 查找总销量超过 5 件的类别
SELECT
    category,
    SUM(quantity) AS total_qty
FROM sales
GROUP BY category
HAVING SUM(quantity) > 5;

-- 查找平均单价超过 100 的类别
SELECT
    category,
    ROUND(AVG(amount), 2) AS avg_price
FROM sales
GROUP BY category
HAVING AVG(amount) > 100;
```

### 示例 5：DISTINCT 在聚合中的使用

```sql
-- 统计有多少个不同的产品
SELECT COUNT(DISTINCT product) AS unique_products FROM sales;

-- 每个类别有多少个不同的产品
SELECT
    category,
    COUNT(DISTINCT product) AS unique_products
FROM sales
GROUP BY category;

-- 有多少个不同的地区购买了书籍
SELECT COUNT(DISTINCT region) AS regions
FROM sales
WHERE category = '书籍';
```

### 示例 6：FILTER 子句

```sql
-- 使用 FILTER 同时计算多个条件的聚合
SELECT
    COUNT(*) AS total_orders,
    COUNT(*) FILTER (WHERE category = '书籍') AS book_orders,
    COUNT(*) FILTER (WHERE category = '电子产品') AS electronics_orders,
    SUM(amount * quantity) FILTER (WHERE region = '华东') AS east_revenue,
    SUM(amount * quantity) FILTER (WHERE region = '华北') AS north_revenue
FROM sales;

-- 按月统计各类别销售额（结合 FILTER）
SELECT
    TO_CHAR(sale_date, 'YYYY-MM') AS month,
    SUM(amount * quantity) AS total,
    SUM(amount * quantity) FILTER (WHERE category = '书籍') AS book_total,
    SUM(amount * quantity) FILTER (WHERE category = '电子产品') AS elec_total
FROM sales
GROUP BY TO_CHAR(sale_date, 'YYYY-MM')
ORDER BY month;
```

### 示例 7：综合查询

```sql
-- 找出每个地区销售额最高的类别
SELECT DISTINCT ON (region)
    region,
    category,
    SUM(amount * quantity) AS revenue
FROM sales
GROUP BY region, category
ORDER BY region, revenue DESC;
```

## 常见问题

### Q1: WHERE 和 HAVING 有什么区别？

**A**：`WHERE` 在分组之前过滤行，不能使用聚合函数；`HAVING` 在分组之后过滤组，可以使用聚合函数。例如不能写 `WHERE COUNT(*) > 5`，但可以写 `HAVING COUNT(*) > 5`。性能上，尽量用 `WHERE` 先过滤数据以减少分组计算量。

### Q2: COUNT(*) 和 COUNT(列名) 有什么区别？

**A**：`COUNT(*)` 计算所有行数，包括含 NULL 的行；`COUNT(列名)` 只计算该列非 NULL 的行数。例如表中有 10 行，其中某列有 3 个 NULL，`COUNT(*)` 返回 10，`COUNT(该列)` 返回 7。

### Q3: GROUP BY 中可以使用列别名吗？

**A**：PostgreSQL **允许**在 `GROUP BY` 中使用 `SELECT` 列表中定义的列别名，这是 PostgreSQL 的扩展行为。但标准 SQL 不支持此写法，若需要跨数据库兼容，建议在 `GROUP BY` 中使用原始表达式。
