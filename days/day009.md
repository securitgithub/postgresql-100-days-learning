## 理论讲解

### 1. INSERT 语法概览

```sql
INSERT INTO table_name [ (column1, column2, ...) ]
    VALUES (value1, value2, ...),
           (value1, value2, ...),
           ...
    [ ON CONFLICT conflict_target conflict_action ]
    [ RETURNING * | column1, column2, ... ];
```

### 2. INSERT 的性能考虑

| 方式 | 适用场景 | 速度 |
|------|---------|------|
| 单行 INSERT | 少量数据、应用层逐条插入 | 慢 |
| 多行 INSERT | 一次插入几十到几百行 | 较快 |
| INSERT ... SELECT | 从查询结果插入 | 快 |
| COPY | 大批量数据导入（万行以上） | 最快 |
| \copy (psql) | 从客户端文件导入 | 最快（客户端） |

### 3. UPSERT（INSERT ON CONFLICT）

UPSERT = Update + Insert，即"有则更新，无则插入"。PostgreSQL 9.5 引入了 `ON CONFLICT` 子句，提供了原子性的 UPSERT 操作。

两种冲突处理方式：
- **DO NOTHING**：冲突时忽略，不做任何操作
- **DO UPDATE**：冲突时更新指定列

### 4. COPY vs \copy

| 特性 | COPY（服务器端） | \copy（客户端） |
|------|-----------------|----------------|
| 执行位置 | PostgreSQL 服务器 | psql 客户端 |
| 文件路径 | 服务器文件系统 | 客户端文件系统 |
| 权限要求 | 需要超级用户权限 | 普通用户即可 |
| 性能 | 略快（本地文件） | 略慢（网络传输） |
| 常用场景 | 服务器本地数据导入 | 远程导入、开发环境 |

## 代码示例

### 准备演示表

```sql
-- 创建演示表
CREATE TABLE users (
    id integer GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    username varchar(30) NOT NULL UNIQUE,
    email varchar(100) NOT NULL UNIQUE,
    display_name text,
    age integer CHECK (age > 0 AND age < 200),
    is_active boolean NOT NULL DEFAULT true,
    created_at timestamptz NOT NULL DEFAULT now(),
    updated_at timestamptz NOT NULL DEFAULT now()
);

CREATE TABLE products (
    id integer GENERATED ALWAYS AS IDENTITY PRIMARY KEY,
    sku varchar(20) NOT NULL UNIQUE,
    name text NOT NULL,
    price numeric(10,2) NOT NULL CHECK (price >= 0),
    stock integer NOT NULL DEFAULT 0 CHECK (stock >= 0),
    category varchar(50),
    created_at timestamptz NOT NULL DEFAULT now(),
    updated_at timestamptz NOT NULL DEFAULT now()
);
```

### 单行插入

```sql
-- 指定所有列（推荐明确列名）
INSERT INTO users (username, email, display_name, age)
VALUES ('alice', 'alice@example.com', 'Alice Wang', 28);

-- 省略可选列（使用默认值）
INSERT INTO users (username, email)
VALUES ('bob', 'bob@example.com');

-- 使用 DEFAULT 关键字显式使用默认值
INSERT INTO users (username, email, display_name, age, is_active)
VALUES ('charlie', 'charlie@example.com', 'Charlie Li', 35, DEFAULT);
```

### INSERT RETURNING

```sql
-- RETURNING 返回插入的行数据（非常实用！）

-- 返回所有列
INSERT INTO users (username, email, display_name, age)
VALUES ('iris', 'iris@example.com', '爱丽丝', 26)
RETURNING *;

-- 只返回特定列（常用于获取自动生成的 ID）
INSERT INTO users (username, email)
VALUES ('jack', 'jack@example.com')
RETURNING id, username, created_at;

-- 多行插入也可以使用 RETURNING
INSERT INTO products (sku, name, price, stock, category) VALUES
    ('SSD-001', '固态硬盘 1TB', 499.00, 60, '存储'),
    ('SSD-002', '固态硬盘 2TB', 899.00, 35, '存储')
RETURNING id, sku, name;

-- RETURNING 配合表达式
INSERT INTO products (sku, name, price, stock, category)
VALUES ('HDD-001', '机械硬盘 4TB', 599.00, 45, '存储')
RETURNING id, name, price * stock AS total_value;

-- new_id = result.rows[0].id
```

### INSERT ... ON CONFLICT（UPSERT）

```sql
-- ===== DO NOTHING：冲突时跳过 =====

-- 尝试插入已存在的用户名
INSERT INTO users (username, email, display_name)
VALUES ('alice', 'alice_new@example.com', 'New Alice')
ON CONFLICT (username) DO NOTHING;
-- 不报错，也不做任何事

-- 结合 RETURNING 查看是否成功插入
INSERT INTO users (username, email, display_name)
VALUES ('alice', 'alice_new@example.com', 'New Alice')
ON CONFLICT (username) DO NOTHING
RETURNING id, username;
-- 如果冲突，不返回任何行；如果成功，返回新行

-- ===== DO UPDATE：冲突时更新 =====

-- 商品库存更新：有则更新库存和价格，无则插入
INSERT INTO products (sku, name, price, stock, category)
VALUES ('MOUSE-001', '无线静音鼠标 V2', 109.00, 300, '配件')
ON CONFLICT (sku) DO UPDATE SET
    name = EXCLUDED.name,          -- EXCLUDED 代表"想要插入的那行数据"
    price = EXCLUDED.price,
    stock = EXCLUDED.stock,
    updated_at = now();

-- 验证更新结果
SELECT sku, name, price, stock, updated_at FROM products WHERE sku = 'MOUSE-001';

-- 累加库存（而非覆盖）
INSERT INTO products (sku, name, price, stock, category)
VALUES ('MOUSE-001', '无线静音鼠标 V2', 109.00, 50, '配件')
ON CONFLICT (sku) DO UPDATE SET
    stock = products.stock + EXCLUDED.stock,  -- 累加
    updated_at = now()
RETURNING sku, stock AS new_stock;

-- 带条件的 UPSERT（只在满足条件时更新）
INSERT INTO products (sku, name, price, stock, category)
VALUES ('LAPTOP-001', '轻薄笔记本 Pro Max', 7999.00, 20, '电脑')
ON CONFLICT (sku) DO UPDATE SET
    name = EXCLUDED.name,
    price = EXCLUDED.price
WHERE EXCLUDED.price > products.price;  -- 只在新价格更高时才更新
-- 如果新价格不高于原价格，则不更新

-- 使用约束名指定冲突目标
INSERT INTO users (username, email, display_name, age)
VALUES ('alice', 'alice_v2@example.com', 'Alice Updated', 29)
ON CONFLICT ON CONSTRAINT users_username_key DO UPDATE SET
    display_name = EXCLUDED.display_name,
    age = EXCLUDED.age,
    updated_at = now()
RETURNING *;
```

### INSERT ... SELECT

```sql
-- 从查询结果插入（数据迁移、数据转换常用）

-- 创建归档表
CREATE TABLE products_archive (LIKE products INCLUDING ALL);

-- 将低库存商品归档
INSERT INTO products_archive
SELECT * FROM products WHERE stock < 30;

-- 带转换的 INSERT SELECT
CREATE TABLE product_summary (
    category varchar(50) PRIMARY KEY,
    product_count integer,
    avg_price numeric(10,2),
    total_stock integer,
    snapshot_at timestamptz DEFAULT now()
);

INSERT INTO product_summary (category, product_count, avg_price, total_stock)
SELECT
    category,
    count(*),
    round(avg(price), 2),
    sum(stock)
FROM products
GROUP BY category;

SELECT * FROM product_summary;

-- 从 CTE 结果插入
WITH new_products AS (
    SELECT
        'NEW-' || sku AS sku,
        name || ' (新版)' AS name,
        price * 1.1 AS price,
        0 AS stock,
        category
    FROM products
    WHERE category = '配件'
    LIMIT 3
)
INSERT INTO products (sku, name, price, stock, category)
SELECT sku, name, round(price, 2), stock, category FROM new_products
RETURNING id, sku, name, price;
```

### COPY 批量导入导出

```sql
-- ========== 导出数据 ==========

-- 导出为 CSV（服务器端 COPY，需要超级用户权限）
COPY products TO '/tmp/products.csv' WITH (FORMAT csv, HEADER true);

-- psql 客户端 \copy（推荐，无需超级用户权限）
\copy products TO '/tmp/products_client.csv' WITH (FORMAT csv, HEADER true)

-- 导出指定列
\copy products (sku, name, price) TO '/tmp/products_partial.csv' WITH (FORMAT csv, HEADER true)

-- 导出查询结果
\copy (SELECT sku, name, price FROM products WHERE category = '电脑') TO '/tmp/computers.csv' WITH (FORMAT csv, HEADER true)

-- 导出为 TSV（Tab 分隔）
\copy products TO '/tmp/products.tsv' WITH (FORMAT csv, DELIMITER E'\t', HEADER true)

-- ========== 导入数据 ==========

-- 准备导入表
CREATE TABLE products_import (
    sku varchar(20),
    name text,
    price numeric(10,2),
    stock integer,
    category varchar(50)
);

-- 从 CSV 导入
\copy products_import FROM '/tmp/products_client.csv' WITH (FORMAT csv, HEADER true)

-- 指定列顺序（CSV 列和表列不同时）
\copy products_import (sku, name, price) FROM '/tmp/products_partial.csv' WITH (FORMAT csv, HEADER true)

-- ========== COPY 选项 ==========

-- 常用选项
\copy products TO '/tmp/products_full.csv' WITH (
    FORMAT csv,           -- 格式：csv, text, binary
    HEADER true,          -- 是否包含表头
    DELIMITER ',',        -- 分隔符（默认 CSV 为逗号）
    QUOTE '"',            -- 引号字符
    ESCAPE '"',           -- 转义字符
    NULL 'NULL',          -- NULL 值表示
    ENCODING 'UTF8'       -- 编码
)

-- 导入时忽略错误行（PostgreSQL 17+ 支持 ON_ERROR）
-- \copy products_import FROM '/tmp/data.csv' WITH (FORMAT csv, HEADER true, ON_ERROR stop)
```

### 性能对比与优化

```sql
-- 大批量插入的性能优化技巧

-- 技巧1：使用多行 INSERT 代替多条单行 INSERT
-- 差：逐行插入（每条都是独立事务）
INSERT INTO products (sku, name, price, stock, category) VALUES ('T-001', 'Test 1', 10, 1, 'test');
INSERT INTO products (sku, name, price, stock, category) VALUES ('T-002', 'Test 2', 20, 2, 'test');
INSERT INTO products (sku, name, price, stock, category) VALUES ('T-003', 'Test 3', 30, 3, 'test');

-- 好：多行插入（一条语句）
INSERT INTO products (sku, name, price, stock, category) VALUES
    ('T-001', 'Test 1', 10, 1, 'test'),
    ('T-002', 'Test 2', 20, 2, 'test'),
    ('T-003', 'Test 3', 30, 3, 'test');

-- 技巧2：在事务中批量插入
BEGIN;
INSERT INTO products (sku, name, price, stock, category) VALUES ('B-001', 'Batch 1', 10, 1, 'test');
INSERT INTO products (sku, name, price, stock, category) VALUES ('B-002', 'Batch 2', 20, 2, 'test');
-- ... 更多插入
COMMIT;

-- 技巧3：临时禁用索引和约束（仅限于初始数据加载）
-- 先删除索引 → 批量插入 → 重新创建索引
-- 对于大型初始数据加载可以提速数倍

-- 技巧4：调整参数加速大批量导入
-- SET maintenance_work_mem = '1GB';
-- SET synchronous_commit = off;  -- 注意：可能在崩溃时丢失少量数据
-- SET wal_level = minimal;       -- 需要重启，仅用于初始化导入

-- 性能对比（示意）：
-- 10万行数据插入：
-- 逐行 INSERT：约 60 秒
-- 多行 INSERT（每次1000行）：约 5 秒
-- COPY：约 1 秒
```

## 常见问题

### Q1: ON CONFLICT 中的 EXCLUDED 是什么？

**A:** `EXCLUDED` 是一个特殊的表引用，代表"因为冲突而没有被插入的那行数据"（即 VALUES 中提供的值）。在 `DO UPDATE SET` 中：
- `EXCLUDED.column_name`：引用试图插入的新值
- `table_name.column_name`：引用表中已存在的旧值

```sql
-- 例如：取新旧价格中较高的那个
ON CONFLICT (sku) DO UPDATE SET
    price = GREATEST(products.price, EXCLUDED.price)
```

### Q2: COPY 和 INSERT 什么时候选哪个？

**A:**
- **INSERT**：少量数据（几百行以内）、需要 RETURNING、需要 ON CONFLICT、应用程序中使用
- **COPY**：大量数据（千行以上）、初始数据加载、数据迁移、ETL 流程
- 经验法则：如果超过 1000 行，优先考虑 COPY

### Q3: 多行 INSERT 一次可以插入多少行？

**A:** PostgreSQL 没有硬性限制，但需要注意：
- 每条 SQL 语句有大小限制（默认 1GB，由 `max_query_length` 控制）
- 过大的 INSERT 语句会占用大量内存
- 实践建议：每次 1000-10000 行比较合适
- 如果超过 10000 行，建议使用 COPY 或分批插入
