# Day 027: 窗口函数（二）进阶

## 学习目标

- 掌握 LAG() 和 LEAD() 函数访问前后行数据
- 熟练使用 FIRST_VALUE()、LAST_VALUE()、NTH_VALUE() 获取窗口中指定位置的值
- 理解 NTILE() 函数实现数据分桶
- 深入理解窗口帧（Window Frame）的 ROWS BETWEEN 语法
- 能够结合窗口帧定义精确控制窗口函数的计算范围

## 理论讲解

### LAG() 和 LEAD()

这两个函数用于访问当前行前面或后面的行数据，常用于计算同比、环比等：

```sql
LAG(expression [, offset [, default]])  OVER (...)
LEAD(expression [, offset [, default]]) OVER (...)
```

- `expression`：要返回的列或表达式
- `offset`：偏移量，默认为 1（前/后一行）
- `default`：当偏移超出范围时返回的默认值，默认为 NULL

### FIRST_VALUE / LAST_VALUE / NTH_VALUE

| 函数 | 说明 |
|------|------|
| `FIRST_VALUE(expr)` | 返回窗口帧内第一行的值 |
| `LAST_VALUE(expr)` | 返回窗口帧内最后一行的值 |
| `NTH_VALUE(expr, n)` | 返回窗口帧内第 n 行的值 |

> **重要陷阱**：`LAST_VALUE()` 的默认窗口帧是 `ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW`，这意味着"最后一行"其实就是当前行！要获取分区真正的最后一行，需要显式指定 `ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING`。

### NTILE()

`NTILE(n)` 将分区内的行尽可能均匀地分成 n 组（桶），并为每行分配组号（1 到 n）：

```sql
NTILE(4) OVER (ORDER BY amount DESC)  -- 分成4组
```

如果行数不能被 n 整除，前面的组会多分配一行。

### 窗口帧（Window Frame）

窗口帧精确定义了窗口函数计算时包含哪些行：

```sql
{ ROWS | RANGE | GROUPS } BETWEEN frame_start AND frame_end
```

帧边界选项：

| 边界 | 说明 |
|------|------|
| `UNBOUNDED PRECEDING` | 分区的第一行 |
| `n PRECEDING` | 当前行的前 n 行 |
| `CURRENT ROW` | 当前行 |
| `n FOLLOWING` | 当前行的后 n 行 |
| `UNBOUNDED FOLLOWING` | 分区的最后一行 |

默认帧行为：
- 有 ORDER BY 时：`RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW`
- 无 ORDER BY 时：`RANGE BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING`

### ROWS vs RANGE vs GROUPS

- **ROWS**：基于物理行位置，精确到具体行
- **RANGE**：基于 ORDER BY 列的值范围，相同值的行被视为同一位置
- **GROUPS**：基于 ORDER BY 值的分组（PostgreSQL 11+）

### 示例一：LAG — 上期对比

```sql
-- 每个员工按月查看销售额及与上月的变化
SELECT
    employee,
    sale_date,
    amount,
    LAG(amount) OVER (
        PARTITION BY employee ORDER BY sale_date
    ) AS prev_amount,
    amount - LAG(amount) OVER (
        PARTITION BY employee ORDER BY sale_date
    ) AS change,
    -- 环比增长率
    ROUND(
        (amount - LAG(amount) OVER (PARTITION BY employee ORDER BY sale_date))
        / LAG(amount) OVER (PARTITION BY employee ORDER BY sale_date) * 100,
        1
    ) AS growth_pct
FROM sales
ORDER BY employee, sale_date;
```

### 示例二：LAG 带默认值

```sql
-- 使用默认值 0 避免第一行出现 NULL
SELECT
    employee,
    sale_date,
    amount,
    LAG(amount, 1, 0) OVER (
        PARTITION BY employee ORDER BY sale_date
    ) AS prev_amount
FROM sales
ORDER BY employee, sale_date;
```

### 示例三：LEAD — 查看下一期

```sql
-- 显示下一笔销售的金额和日期间隔
SELECT
    employee,
    sale_date,
    amount,
    LEAD(amount) OVER w AS next_amount,
    LEAD(sale_date) OVER w AS next_sale_date,
    LEAD(sale_date) OVER w - sale_date AS days_between
FROM sales
WINDOW w AS (PARTITION BY employee ORDER BY sale_date)
ORDER BY employee, sale_date;
```

### 示例四：LAG 偏移多行

```sql
-- 查看两期前的数据（季度对比场景）
SELECT
    employee,
    sale_date,
    amount,
    LAG(amount, 2) OVER (
        PARTITION BY employee ORDER BY sale_date
    ) AS two_periods_ago
FROM sales
ORDER BY employee, sale_date;
```

### 示例五：FIRST_VALUE 和 LAST_VALUE

```sql
-- 每个部门的最高和最低销售额
SELECT
    department,
    employee,
    amount,
    FIRST_VALUE(amount) OVER (
        PARTITION BY department ORDER BY amount DESC
    ) AS dept_max,
    -- 注意：必须指定完整的帧范围才能获取真正的最后一行
    LAST_VALUE(amount) OVER (
        PARTITION BY department ORDER BY amount DESC
        ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    ) AS dept_min,
    -- 不指定帧范围的 LAST_VALUE（错误示范）
    LAST_VALUE(amount) OVER (
        PARTITION BY department ORDER BY amount DESC
    ) AS last_value_wrong  -- 这里返回的是当前行！
FROM sales
ORDER BY department, amount DESC;
```

### 示例六：NTH_VALUE

```sql
-- 获取每个部门销售额排名第 2 的记录
SELECT
    department,
    employee,
    amount,
    NTH_VALUE(amount, 2) OVER (
        PARTITION BY department ORDER BY amount DESC
        ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    ) AS second_highest
FROM sales
ORDER BY department, amount DESC;
```

### 示例七：NTILE 分桶

```sql
-- 将所有销售记录按金额分成 4 个等级
SELECT
    employee,
    amount,
    NTILE(4) OVER (ORDER BY amount DESC) AS quartile,
    CASE NTILE(4) OVER (ORDER BY amount DESC)
        WHEN 1 THEN '第一梯队 (Top 25%)'
        WHEN 2 THEN '第二梯队 (25%-50%)'
        WHEN 3 THEN '第三梯队 (50%-75%)'
        WHEN 4 THEN '第四梯队 (Bottom 25%)'
    END AS tier
FROM sales
ORDER BY amount DESC;
```

### 示例八：ROWS BETWEEN 自定义帧

```sql
-- 移动平均：当前行及前后各一行的平均值
SELECT
    employee,
    sale_date,
    amount,
    ROUND(AVG(amount) OVER (
        PARTITION BY employee
        ORDER BY sale_date
        ROWS BETWEEN 1 PRECEDING AND 1 FOLLOWING
    ), 2) AS moving_avg_3
FROM sales
ORDER BY employee, sale_date;

-- 累计到当前行的总和
SELECT
    employee,
    sale_date,
    amount,
    SUM(amount) OVER (
        PARTITION BY employee
        ORDER BY sale_date
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS running_total
FROM sales
ORDER BY employee, sale_date;
```

### 示例九：综合应用 — 销售趋势分析

```sql
-- 综合使用多种窗口函数分析销售趋势
SELECT
    employee,
    sale_date,
    amount,
    -- 与上期比较
    LAG(amount) OVER w AS prev_amount,
    amount - COALESCE(LAG(amount) OVER w, 0) AS change,
    -- 部门内排名
    RANK() OVER (PARTITION BY department ORDER BY amount DESC) AS dept_rank,
    -- 员工历史最高
    FIRST_VALUE(amount) OVER (
        PARTITION BY employee ORDER BY amount DESC
        ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    ) AS personal_best,
    -- 是否为个人最佳
    CASE WHEN amount = FIRST_VALUE(amount) OVER (
        PARTITION BY employee ORDER BY amount DESC
        ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
    ) THEN '是' ELSE '否' END AS is_best
FROM sales
WINDOW w AS (PARTITION BY employee ORDER BY sale_date)
ORDER BY employee, sale_date;
```

## 常见问题

### Q1：为什么 LAST_VALUE 经常返回当前行的值？
**A**：因为默认窗口帧是 `RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW`，在这个帧里"最后一行"就是当前行。要获取分区中真正的最后一行值，必须显式指定 `ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING`。

### Q2：LAG/LEAD 是否受窗口帧影响？
**A**：不受影响。LAG 和 LEAD 始终基于整个分区进行偏移计算，它们会忽略窗口帧的定义。只有 FIRST_VALUE、LAST_VALUE、NTH_VALUE 以及聚合窗口函数才受窗口帧影响。

### Q3：NTILE 分组不均匀时怎么处理？
**A**：当行数不能被 n 整除时，前面的组会多分配一行。例如 10 行分 3 组，结果是 4+3+3。PostgreSQL 会自动处理，无需额外干预。
