# Day 013: UPDATE 与 DELETE

> **PostgreSQL 百日学习计划——第 13 天：UPDATE、DELETE、TRUNCATE 深入讲解**

## 学习目标

- 掌握 `UPDATE` 的基本语法与条件更新
- 学会使用 `UPDATE ... FROM` 进行跨表更新
- 理解 `UPDATE ... RETURNING` 获取更新后的数据
- 掌握 `DELETE` 语法及 `DELETE ... USING` 跨表删除
- 了解 `TRUNCATE` 与 `DELETE` 的区别

## 理论讲解

### 1. UPDATE 基本语法

`UPDATE` 用于修改表中已存在的行：

```sql
UPDATE 表名
SET 列1 = 值1, 列2 = 值2, ...
WHERE 条件;
```

**重要提醒**：如果省略 `WHERE` 子句，将更新表中的**所有行**！这是一个极其危险的操作。

### 2. 条件更新

可以使用 `CASE WHEN` 实现不同条件下的不同更新：

```sql
UPDATE products
SET price = CASE
    WHEN category = '书籍' THEN price * 0.9
    WHEN category = '电子产品' THEN price * 0.85
    ELSE price
END
WHERE price IS NOT NULL;
```

### 3. UPDATE ... FROM（跨表更新）

PostgreSQL 支持 `UPDATE ... FROM` 语法，可以根据其他表的数据来更新当前表：

```sql
UPDATE orders o
SET status = '已发货'
FROM shipping s
WHERE o.id = s.order_id AND s.shipped = true;
```

这是 PostgreSQL 的扩展语法，标准 SQL 中需要使用子查询实现同样效果。

### 4. UPDATE ... RETURNING

`RETURNING` 子句可以返回被更新的行，避免额外的查询：

```sql
UPDATE products
SET price = price * 1.1
WHERE category = '书籍'
RETURNING id, name, price;
```

`RETURNING` 可以返回任意列或表达式，甚至返回更新前后的值（使用子查询技巧）。

### 5. DELETE 基本语法

`DELETE` 从表中删除行：

```sql
DELETE FROM 表名 WHERE 条件;
```

同样，省略 `WHERE` 会删除所有行。

### 6. DELETE ... USING（跨表删除）

类似 `UPDATE ... FROM`，`DELETE ... USING` 可以根据其他表的数据来删除行：

```sql
DELETE FROM order_items oi
USING orders o
WHERE oi.order_id = o.id AND o.status = '已取消';
```

### 7. DELETE ... RETURNING

与 `UPDATE` 相同，`DELETE` 也支持 `RETURNING`：

```sql
DELETE FROM products WHERE stock = 0
RETURNING id, name;
```

### 8. TRUNCATE 与 DELETE 的区别

| 特性 | DELETE | TRUNCATE |
|------|--------|----------|
| 语法 | `DELETE FROM 表名` | `TRUNCATE TABLE 表名` |
| 过滤条件 | 支持 WHERE | 不支持 |
| 速度 | 逐行删除，较慢 | 直接释放数据页，极快 |
| 事务安全 | 可以回滚 | PostgreSQL 中可以回滚 |
| 触发器 | 触发行级触发器 | 只触发 TRUNCATE 触发器 |
| MVCC | 生成死行，需要 VACUUM | 不产生死行 |
| 序列重置 | 不重置 | 可选 RESTART IDENTITY |
| 外键约束 | 逐行检查 | 需要 CASCADE 或无外键引用 |

```sql
-- TRUNCATE 示例
TRUNCATE TABLE logs;

-- 重置序列并级联截断引用表
TRUNCATE TABLE orders RESTART IDENTITY CASCADE;
```

## 代码示例

### 准备测试数据

```sql
-- 创建商品表
CREATE TABLE products (
    id        SERIAL PRIMARY KEY,
    name      VARCHAR(100) NOT NULL,
    category  VARCHAR(50),
    price     NUMERIC(10,2),
    stock     INT DEFAULT 0,
    status    VARCHAR(20) DEFAULT '在售'
);

-- 创建折扣规则表
CREATE TABLE discount_rules (
    category   VARCHAR(50) PRIMARY KEY,
    discount   NUMERIC(3,2)  -- 如 0.90 表示打九折
);

-- 插入数据
INSERT INTO products (name, category, price, stock) VALUES
('PostgreSQL 实战', '书籍', 89.00, 120),
('机械键盘', '电子产品', 399.00, 45),
('Python 入门', '书籍', 59.00, 200),
('无线鼠标', '电子产品', 129.00, 0),
('显示器', '电子产品', 1999.00, 15),
('数据结构教程', '书籍', 69.00, 0),
('USB 数据线', '配件', 19.90, 500),
('蓝牙耳机', '电子产品', 299.00, 60);

INSERT INTO discount_rules (category, discount) VALUES
('书籍', 0.90),
('电子产品', 0.85),
('配件', 0.95);
```

### 示例 1：基本 UPDATE

```sql
-- 更新单个产品的价格
UPDATE products SET price = 79.00 WHERE name = 'PostgreSQL 实战';

-- 更新多列
UPDATE products
SET price = 109.00, stock = 150
WHERE name = '无线鼠标';

-- 使用表达式更新：所有商品涨价 10%
UPDATE products SET price = price * 1.10;
```

### 示例 2：条件更新（CASE WHEN）

```sql
-- 根据库存状态批量更新：库存为 0 标记为"缺货"，库存低于 20 标记为"低库存"
UPDATE products
SET status = CASE
    WHEN stock = 0 THEN '缺货'
    WHEN stock < 20 THEN '低库存'
    ELSE '在售'
END;

-- 验证结果
SELECT name, stock, status FROM products ORDER BY stock;
```

### 示例 3：UPDATE ... FROM 跨表更新

```sql
-- 根据折扣规则表更新价格
UPDATE products p
SET price = p.price * d.discount
FROM discount_rules d
WHERE p.category = d.category;

-- 验证结果
SELECT name, category, price FROM products;
```

### 示例 4：UPDATE ... RETURNING

```sql
-- 将缺货商品下架，并返回被修改的记录
UPDATE products
SET status = '已下架'
WHERE stock = 0
RETURNING id, name, status;
```

### 示例 5：DELETE 基本操作

```sql
-- 删除已下架的商品
DELETE FROM products WHERE status = '已下架';

-- 验证
SELECT * FROM products;
```

### 示例 6：DELETE ... USING 跨表删除

```sql
-- 创建一个黑名单表
CREATE TABLE blacklist (product_name VARCHAR(100));
INSERT INTO blacklist VALUES ('USB 数据线');

-- 根据黑名单删除商品
DELETE FROM products p
USING blacklist b
WHERE p.name = b.product_name
RETURNING p.id, p.name;
```

### 示例 7：DELETE ... RETURNING

```sql
-- 删除价格低于 50 的商品并记录删除的信息
DELETE FROM products
WHERE price < 50
RETURNING id, name, price;
```

### 示例 8：安全更新模式

```sql
-- 先用 SELECT 验证将要影响的行
SELECT id, name, price FROM products WHERE category = '书籍';

-- 确认无误后再执行 UPDATE
UPDATE products SET price = price * 0.95 WHERE category = '书籍';

-- 在事务中执行更新（可以回滚）
BEGIN;
UPDATE products SET price = 0 WHERE category = '电子产品';
-- 发现错误，回滚！
ROLLBACK;
```

## 常见问题

### Q1: UPDATE 不加 WHERE 会怎样？

**A**：将更新表中的**所有行**。这是非常危险的操作。建议养成习惯：先用 `SELECT` 加相同的 `WHERE` 条件查看将被影响的行，确认无误后再将 `SELECT` 改为 `UPDATE ... SET`。在生产环境中，务必在事务中操作，出问题可以 `ROLLBACK`。

### Q2: TRUNCATE 可以回滚吗？

**A**：在 PostgreSQL 中，`TRUNCATE` 是**事务安全**的，可以回滚。这与 MySQL、Oracle 不同（在那些数据库中 TRUNCATE 是 DDL 操作，会自动提交）。但 TRUNCATE 仍然需要获取 `ACCESS EXCLUSIVE` 锁，会阻塞其他所有操作。

### Q3: UPDATE ... FROM 中如果关联到多行会怎样？

**A**：如果 `FROM` 子句中的关联导致一行匹配到多行，PostgreSQL 会使用其中**任意一行**的值进行更新，结果是不确定的。因此必须确保关联条件能唯一匹配。可以先用 `SELECT` 验证关联结果。
